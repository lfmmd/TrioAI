TC_MbMemMapSet

Type

System Function.

Function

Set the memory mapping for Modbus TCP communication.

Inputs

Execute: BOOL;Rising edge requests execution.
ReqAddr: UINT;Modbus request address.
BaseAddr: UINT;Base address on the controller memory area to which the request address is mapped.
NumEntries: UINT;Number of Modbus registers in the mapped region. Minimum value is 1. See Address Halving.
MemArea: TC_MEM_MAPPING_AREA_T;Memory area to map Modbus data to: tcMapToVR (VR) or tcMapToTable (TABLE).
DataType: TC_MB_DATATYPE_T;Data type of the mapped region: tcMbInt16, tcMbInt32, or tcMbFloat32.
Slot: SINT;Slot number, the default value is -1.

Outputs

Done: BOOL;TRUE when function has completed normally.
Error: BOOL;TRUE if a program error is detected.
ErrorID: UINT;Returned error number.

Description

This function allows the user to configure Modbus TCP at a low level. The default values allow a master to connect without any configuration on the Controller side. In case a different initial configuration is needed, the system parameter IP_PROTOCOL_CTRL must be configured in the MC_CONFIG file in order to disable the protocol and set the configuration required within a IEC program.

Address Halving

This function automatically applies address halving, so the number of mapped controller memory registers depends on the data type.

For 16-bit data types, each Modbus register is mapped to one controller address, so the mapped address area is from the base address to (base address + num_entries – 1).

For 32-bit data types, each two Modbus registers are mapped to one controller address, so the mapped address area is from the base address to (base address + (num_entries/2) – 1).

ST Language

MyTC_MbMemMapSet(Execute:= Execute, ReqAddr:= ReqAddr, BaseAddr:= BaseAddr, NumEntries:= NumEntries, MemArea:= MemArea, DataType:= DataType, Slot:= Slot);

Done:=MyTC_MbMemMapSet.Done;

Error:=MyTC_MbMemMapSet.Error;

ErrorID:=MyTC_MbMemMapSet.ErrorID;
