程序组织单元

应用程序是程序列表。根据以下模型，程序在目标周期内按顺序执行：

Begin cycle

| exchange I/Os | execute first program | ... | execute last program | wait for cycle time to elapse

End Cycle

程序按照用户定义的顺序执行。所有SFC程序都必须分组（不可能在两个SFC程序之间的FBD、LD或STEN中插入程序）。应用程序中的程序数限制为32767。使用创建程序时选择的语言输入每个程序。
可能的语言有：

  • 顺序功能图 (SFC),
  • 功能框图 (FBD),
  • 梯形图 (LD),
  • 结构化文本 (ST)

程序必须有唯一的名称。名称不能是编程语言的保留关键字，也不能与标准或“C”函数或函数块同名。程序不应与声明的变量同名。程序的名称应该以字母或下划线（“_”）开头，后跟字母、数字或下划线标记。不允许在一个名称中放置两个连续的下划线。命名不区分大小写。两个不同情况的名字被认为是相同的。

子SFC程序

可以定义SFC程序的层次结构，在程序列表中作为树输入。子程序控制在父SFC程序的操作块中。

用户定义的功能块

程序列表可以由用户定义的功能块（UDFBs）完成。UDFBs使用SFC、FBD、LD或ST语言描述，并且可以用作应用程序中的其他功能块。输入和输出参数加上UDFB的私有变量在变量编辑器中声明为UDFB的局部变量。

在UDFB中使用任何操作都没有限制。UDFB可以调用标准函数和函数块。

一个UDFB可以调用另一个UDFB。被调用的UDFB必须在程序列表中的调用方之前声明。

每次实例化UDFB时，都会为声明的实例复制其私有变量。UDFB的代码在父程序中的每个调用上都是重复的。这会在运行时带来更高的性能，但会消耗代码空间。建议将小算法打包到UDFBs中。大部分代码应该在程序中管理。

UDFB不能有超过32个输入参数或32个输出参数。

子程序

程序列表可以由子程序完成。子程序使用FBD、LD或ST语言描述，并可由应用程序调用。输入和输出参数加上子程序的局部变量在变量编辑器中声明为子程序的局部变量。

一个子程序可以调用另一个子程序或UDFB。

与UDFB不同，子程序的局部变量没有实例化。这意味着子程序总是在同一组局部变量上工作。子程序的局部变量在各种调用中保持其值。子程序的代码在被父程序多次调用时不会重复。

一个子程序不能有超过32个输入参数或32个输出参数。
