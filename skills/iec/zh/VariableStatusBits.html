变量状态位

工作台允许您将状态位与声明的变量相关联。除了实时值外，每个变量还可以有：

64位状态位

日期和时间戳

状态位和时间戳通常由负责硬件输入的输入驱动程序设置，但也可以与某些网络协议上的变量值一起传输。此外，IEC 61131-3程序可以访问变量的状态位。

状态位管理在某些目标上可能不可用。有关可用功能的更多详细信息，请参阅OEM说明。

状态位管理占用CPU和内存，可能会降低应用程序的性能。

为了启用运行时对状态位和时间/日期戳的管理，必须从项目设置向导的编译器选项列表中选中以下选项：

为具有嵌入属性的变量分配状态标志

只有定义了某些属性（附加配置文件或嵌入符号）的变量才能获得状态位。状态位仅适用于具有单个数据类型（不能是数组或结构）的全局范围变量（全局、保留、I/Os…）。

程序的读写状态

以下功能可用于管理程序中的状态信息：
名称描述
vsiGetBit获取变量的状态位
vsiGetDate获取变量的日期戳
vsiGetTime获取变量的时间戳
vsiSetBit设置变量的状态位
vsiSetDate设置变量的日期戳
vsiSetTime设置变量的时间戳
vsiStamp根据当前时间更新变量的戳记

句法

bBit := vsiGetBit ( variable, bitID );
iDate := vsiGetDate ( variable );
iTime := vsiGetTime ( variable );
bOK := vsiSetBit ( variable, bitID, bBit );
bOK := vsiSetDate ( variable, iDate );
bOK := vsiSetTime ( variable, iTime );
bOK := vsiStamp ( variable );

函数使用以下参数：
参数描述
variable具有嵌入外形或符号的变量。
bitID : DINT状态位的ID（请参阅ID列表）。
bBit : BOOL状态位的值。
iDate : DINT根据实时时钟功能约定的日期戳。
iTime : DINT根据实时时钟功能约定的时间戳。
bOK : BOOL如果成功，则为TRUE。

有关时间和日期戳的更多信息，请参阅实时时钟功能的说明。

支持状态位的驱动程序

以下是处理状态位和日期/时间戳的运行时驱动程序：
驱动描述
Variable binding (ETHERNET)绑定（自发协议）用于在以太网上的运行时之间实时交换变量值。协议负责携带状态位。协议更新由网络更新的变量的日期和时间戳。
MODBUS MasterMODBUS主协议（RTU/TCP/UDP）负责更新网络更新的所有变量的日期和时间戳。MODBUS堆栈还根据交换错误状态设置接收变量的_VSB_I_位状态位。
MODBUS SlaveMODBUS从协议（RTU/TCP/UDP）负责更新网络更新的所有变量的日期和时间戳。
IEC 60870-5 SlaveIEC 60870-5-101 和 IEC 60870-5-104从机协议发送点和量的协议报文中的_VSB_I_BIT, _VSB_OV_BIT, _VSB_BL_BIT, _VSB_SP_BIT and _VSB_NT_BIT，包括日期/时间戳的更新。
IEC 61850 ServerIEC 61850服务器不支持可变状态位。
IEC 61850 ClientIEC 61850客户端可以从iec61850服务器读取VSB-I-u位。包括日期/时间戳的更新。
zenon RT to straton connectionzenon RT到straton连接可以读取所有64个状态位。包括日期/时间戳的更新。
