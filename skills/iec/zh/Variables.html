变量

程序中使用的所有变量必须首先在变量编辑器中声明。每个变量都属于一个组，并且必须由其组内的唯一名称标识。

组

组是一组变量。一个组要么引用变量的物理类，要么标识程序或用户定义的功能块的局部变量。以下是可能的组：
组描述
GLOBAL所有程序都已知的内部变量。
RETAIN所有程序已知的非易失性内部变量。
%I...输入板通道-与物理输入设备链接的具有相同数据类型的变量
%Q...输出板通道-与物理输出设备链接的具有相同数据类型的变量
PROGRAMxxx所有程序局部的内部变量。
（组的名称是程序的名称）
UDFBxxx用户定义的功能块的所有局部内部变量加上其输入和输出参数。
（组的名称是程序的名称）

数据类型和维度

每个变量都必须具有有效的数据类型。它可以是基本数据类型，也可以是功能块。在这种情况下，变量是功能块的一个实例。物理I/O必须具有基本数据类型。功能块的实例可以引用标准或“C”嵌入块，也可以引用用户定义的功能块。

如果选定的数据类型是String，则必须指定最大长度，不能超过255个字符。

有关更多信息，请参阅可用数据类型列表。有关如何使用函数实例的更多信息，请参阅描述功能块的部分。

此外，可以为内部变量指定维度，以便声明数组。数组最多有3维。所有索引都是基于0的。例如，对于一维数组，第一个元素总是由Array名称[0]标识。数组中的项总数（合并所有维度）不能超过65535。

命名变量

变量必须由其父组中的唯一名称标识。变量名不能是编程语言的保留关键字，也不能与标准或“C”函数或功能块同名。变量不应与程序或用户定义的功能块具有相同的名称。

变量的名称应以字母或下划线（“\u”）标记开头，后跟字母、数字或下划线标记。不允许在变量名中放置两个连续的下划线。命名不区分大小写。两个不同情况的名字被认为是相同的。

命名物理I/O

每个I/O通道都有一个反映其物理位置的预定义符号。此符号以输入的%I和输出的%Q开头，后跟标识数据物理大小的字母。然后是电路板的位置，用1或2个数字表示，最后是电路板内通道的0索引。所有数字都用点隔开。以下是I/O符号的可能前缀：
前缀描述
%IX1位输入- BOOL 或 SINT
%QX1 byte output - BOOL 或 SINT
%IW2 位输入 - INT
%QW2 位输出 - INT
%ID4 位输入 - DINT 或 REAL
%QD4 位输入 - DINT 或 REAL
%IL8 位输入 - LINT 或 LEAL
%QL8 位输出 - LINT 或 LEAL
%ISSTRING 输入
%QSSTRING 输出


此外，还可以为每个I/O通道提供别名（可读名称）。在这种情况下，“%”名称或别名可以在程序中使用，没有区别。别名必须与变量名符合相同的规则。

变量的属性

物理I/O标记为输入或输出。输入是只读变量。对于每个内部变量，可以选择只读。

用户定义的功能块和子程序的参数标记为IN或OUT。

子程序和UDFBs的参数

子程序和UDFB可能在输入或ou输出上有参数。输出参数不能是数据结构的数组，只能是单个数据。当数组作为inupt参数传递给UDFB时，它被视为INOUT，这样UDFB就可以在其中读或写。复杂数据类型对输入参数的支持可能取决于选定的编译选项。
