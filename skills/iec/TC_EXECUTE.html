TC_EXECUTE

Type:

Command

Function:

Allows any TrioBASIC command, specified as STRING input parameter, to be executed.

Inputs:

  

EN : BOOL;

Set TRUE to enable the function

Command : STRING;

TrioBASIC command line to be executed

Outputs: 

ENO : BOOL;

TRUE if function is enabled

Error : BOOL;

TRUE if a program error is detected

ErrorID : UINT;

Returned error number

Description:

When the EN input is TRUE, the function block executes the TrioBASIC command STRING specified in the Command input parameter.

The request is initiated on the rising edge of the EN input parameter.

A programming error will set the Error output and return an error ID number.  For the Error ID reference, see TrioBASIC Error List .

ST Language:

Inst_TC_EXECUTE( EN (*BOOL*) , Command (*STRING*) );

Example 1:

At rising edge of ‘print_status_Cmd’ variable, the Command STRING is executed. In this case, it is printing ‘OK’ onto terminal 7.

Local Variables

Text Mode

VAR
 print_status_Cmd : STRING ( 255 ) ;
END_VAR

Grid Mode

 

Inst_TC_EXECUTE(print_status_Cmd, 'PRINT #7,"OK"' );

Example 2:

Using a loop, a command string is constructed and used for TC_EXECUTE command. In this example there is only 1 instance of TC_EXECUTE created to be re-used for different commands. ‘EN’ input needs to be reset to false since TC_EXECUTE is executed at the rising edge of ‘EN’ input.

Local Variables

Text Mode

VAR
 cmd : STRING ( 255 ) ;
 i : INT ;
 Inst_TC_EXECUTE : TC_EXECUTE ; 
END_VAR

Grid Mode

 

FOR i := 0 TO 5 DO
         // Construct string as command
        cmd:= any_to_string (i);
        cmd:= concat ( 'INVERT_IN(' ,cmd);
        cmd:= concat (cmd, ',ON)' );
        
         // Execute command
        Inst_TC_EXECUTE( TRUE ,cmd);
        
         // Reset execution state so that instance can be reused for other inputs
        Inst_TC_EXECUTE( FALSE ,cmd);        
     END_FOR ;

Example 3:

TC_EXECUTE is used to invert 6 inputs as specified in input argument for each instruction. In this example, each TC_EXECUTE instruction is allocated one instance of the instruction.

Local Variables

Text Mode

VAR
 Inst_TC_EXECUTE : TC_EXECUTE ;
 Inst_TC_EXECUTE1 : TC_EXECUTE ;
 Inst_TC_EXECUTE2 : TC_EXECUTE ;
 Inst_TC_EXECUTE3 : TC_EXECUTE ;
 Inst_TC_EXECUTE4 : TC_EXECUTE ;
 Inst_TC_EXECUTE5 : TC_EXECUTE ;
END_VAR

Grid Mode

 

    Inst_TC_EXECUTE( TRUE , 'INVERT_IN(0,ON)' );
    Inst_TC_EXECUTE1( TRUE , 'INVERT_IN(1,ON)' );
    Inst_TC_EXECUTE2( TRUE , 'INVERT_IN(2,ON)' );
    Inst_TC_EXECUTE3( TRUE , 'INVERT_IN(8,ON)' );
    Inst_TC_EXECUTE4( TRUE , 'INVERT_IN(9,ON)' );
    Inst_TC_EXECUTE5( TRUE , 'INVERT_IN(10,ON)' );

FBD Language:

Example 1:

Command 'PRINT #5, "Initialise components..."' is executed one time, i.e. at the rising edge, as soon as ‘print_status_CMD’ is TRUE. If the STRING command is invalid, Err becomes TRUE with ErrId specifying the error type.

LD Language:

Example 1:

At rising edge of ‘print_result’ variable, the command STRING is executed and ‘printed’ becomes TRUE. In this case, it is printing ‘Recipe Added’ onto terminal 7. If there is an error, ‘Err’ becomes TRUE.
