NOP

Type

System Parameter

Description

This parameter returns the number of outputs fitted to the system. The value is set by the firmware taking into consideration the total I/O detected, including module I/O, CAN I/O, Fieldbus I/O and CANopen I/O.

Value

The highest output point + 1 that is available for use.

Example

There are 64 external Output points in addition to the 8 built-in I/O points on the controller.

Printing NOP in the terminal:

>>?NOP

80

>>

Note; in this case the last output point addressable is OP(79, state) and READ_OP(79). The outputs start at OP(8, state) so the NOP value is not the total output points, it is the number at which the output map has as the highest available.
