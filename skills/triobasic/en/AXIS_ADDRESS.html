AXIS_ADDRESS

Type

Axis Parameter (MC_CONFIG)

Description

The AXIS_ADDRESS parameter holds the address of the drive or feedback device. For example, can be used to specify the Sercos drive address or AIN channel that is used for feedback on the base axis.

An extension in firmware V2.0232 and later uses bits 8 to 15 as the AOUT address for an analogue output so that a servo axis may use a standard analogue output. This mode requires axis ATYPE 31 and must have FEC 25 enabled in the Motion Coordinator. FEC 25 allows up to 8 axes of this type to be run.

Value

ATYPE in useBits 0 to 7Bits 8 to 15
53 to 59 (Sercos)Node addressN/A
30 (Ain servo)AIN addressN/A
31 (Aout servo)AIN address (optional)AOUT address

ATYPE 31 is for EtherCAT systems using the FlexSlice modules only.

You may require additional Feature Enable Codes before using the remote axis functionality.

Examples

Example 1

Set up a servo axis in MC403/MC405 to use an analogue input instead of the encoder. Feedback comes from a linear potentiometer connected to AIN(32) and the speed command to the drive comes out of axis 1 DAC output.

BASE(1)

AXIS_ADDRESS = 32

ATYPE = 30

P_GAIN = 1.2

VFF_GAIN = 14.8

Example 2

Set up a servo axis in the Flex-6 Nano to use an analogue input for feedback and analogue output for the speed command to the drive. Feedback comes from an LVDT via its signal processing card and is connected to AIN(4) and the speed command to the drive is at AOUT(7).

BASE(14)

AXIS_ADDRESS = (7 << 8) + 4

ATYPE = 31

P_GAIN = 0.78

VFF_GAIN = 8.26

Example 3

Assigning the Sercos drive with the node address 4 to axis 8 in the controller. Then starting it in position mode with drive registration.

BASE(8)

AXIS_ADDRESS = 4

ATYPE = 58
