FORCE_SPEED

Type

Axis Parameter

Description

This parameter sets the main speed for a motion command that supports the advanced speed control (commands ending in SP). The VP_SPEED will accelerate or decelerate so that the profile is run at FORCE_SPEED

FORCE_SPEED is loaded into the buffer at the same time as the move so you can set different speeds for subsequent moves.

With VP_MODE = 5 and VP_MODE = 6 with VP_OPTIONS.1 = 1, the forced demands (those loaded with the move) are bound to the axis demands such that the axis demands are primed with the forced demands of a move when it becomes active. This binding allows the axis demands to be adjusted, and the changes made influence the current move’s profiling.

FORCE_SPEED is also used to provide the speed demand to MOVEADD.

Value

The speed at which the SP motion command will execute, in user UNITS. The default value is 1000 when the UNITS are 1.

Examples

Example 1

In this example the controller will ramp the speed up to a speed of 30 at the end of the MOVE. Then for the duration of the MOVESP(20) the speed will be 30, after which it will ramp back to a speed of 15.

SPEED = 15

MOVE(100)

FORCE_SPEED = 30

ENDMOVE_SPEED = -1 'Disable ENDMOVE_SPEED

MOVESP(20)

MOVE(100)

Example 2

Use FORCE_SPEED to slow the profile speed down during a corner move

FORCE_SPEED = 100

ENDMOVE_SPEED = 50

MOVESP(100, 0)

FORCE_SPEED = 50

MOVECIRCSP(100, 100, 100, 0, 1)

FORCE_SPEED = 100

ENDMOVE_SPEED = 100

MOVESP(0, 100)
