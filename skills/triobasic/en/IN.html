IN

Type

I/O command

Syntax

value = IN[(input_no[, final_input_no])]

Description

IN is used to read the state of the Digital Inputs. Digital I/O normally refers to the 24 V inputs and outputs attached to the Motion Coordinator. I/O points that do not have hardware attached can be used as "Virtual" outputs and inputs that can be set with OP and read back with IN.

If called with no parameters, IN returns the binary sum of the first 32 inputs. If called with one parameter, it returns the state (1 or 0) of that specific input channel. If called with 2 parameters IN returns the binary sum of the group of inputs.

For the 2 parameter case, the inputs should be less than 32 apart.

IN is equivalent to IN(0, 31)

Parameters

value:The state of the selected input or range of inputs
none:Returns the binary sum of the first 32 inputs
input_no:Input number to return the value of/start of input group
final_inpu_not:Last input number of group

Examples

Example 1

In this example a single input is tested:

WAIT UNTIL IN(4) = ON

GOSUB place

Example 2

Move to the distance set on a thumb wheel multiplied by a factor. The thumb wheel is connected to inputs 4, 5, 6, 7 and gives output in binary coded decimal.

The move command is constructed in the following order:

Step 1: IN(4, 7) will get a number 0 to 15

Step 2: multiply by 1.5467 to get required distance

Step 3: absolute MOVE to this position

WHILE TRUE

  MOVEABS(IN(4, 7) * 1.5467)

  WAIT IDLE

WEND

Example 3

Test if either input 2 or 3 is ON.

If (IN AND 12) <> 0 THEN GOTO start

'(Bit 2 = 4 and Bit 3 = 8) so the mask is 12
