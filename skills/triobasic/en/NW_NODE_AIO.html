NW_NODE_AIO

Type

System Array (MC_CONFIG)

Syntax

NW_NODE_AIO(slot, alias_address) = offset_value

Description

This 2D array can be used to set the start address of any EtherCAT node analogue I/O channels. This can be used to define a user specific I/O map to fix I/O points from different sources in place.

The array is 2-dimensional, the first dimension is the master slot identifier, the second dimension is the Configured Station Alias address of the node within that master network.

When using multiple I/O types, it is best to configure the I/O order with the I/O base modifier parameters. See NODE_IO.

Value

slot:The slot number of the EtherCAT port. (0 on MC4N, MC6N and Flex-6 Nano)
alias_address:The configured alias address of the slave device.
offset_value0EtherCAT I/O allocated automatically (default)
>= 1EtherCAT I/O for the defined Node starts at this I/O point address

Example

A system with MC4N-ECAT, with built-in EtherCAT master (slot 0) and 10 servo drives with 2 analogue inputs per drive. It is required that if one or more drives are removed, the analogue I/O addresses do not change.

NW_NODE_AIO(0, 1) = 10

NW_NODE_AIO(0, 2) = 12

NW_NODE_AIO(0, 3) = 14

NW_NODE_AIO(0, 4) = 16

NW_NODE_AIO(0, 5) = 18

NW_NODE_AIO(0, 6) = 20

NW_NODE_AIO(0, 7) = 22

NW_NODE_AIO(0, 8) = 24

NW_NODE_AIO(0, 9) = 26

NW_NODE_AIO(0, 10) = 28
AIN #Location
0-1Built in analogue inputs (N/A on MC4N)

12-13First servo drive – node 2
14-15First servo drive – node 3
16-17First servo drive – node 4
18-19First servo drive – node 5
20-21First servo drive – node 6
22-23First servo drive – node 7
24-25First servo drive – node 8
26-27First servo drive – node 9

If a node is removed, the Analogue I/O addresses do not change. They are tied to the Configured Station Alias address in the remaining drives.
