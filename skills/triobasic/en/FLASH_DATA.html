FLASH_DATA

Type

Startup Parameter (MC_CONFIG only)

Description

FLASH_DATA controls whether VR or TABLE data is automatically backed up to flash memory.

The default setting (0) will use VR memory as the source for backup. However, by changing this parameter to 1 within MC_CONFIG will cause TABLE data as the source for backup. Please note that regardless of which data source is selected, only the first 4096 elements will be available for automatic backup.

MC664 would use the first 196608 TABLE and all the VR points as a backup (flash memory) by default. When FLASH_DATA=1 all the TABLE and VR points are backed up in flash.

MC508, Flex-6 Nano and MC6N would only backup VR points by default. When FLASHTABLE=1, all VR and the first 64000 TABLE points would be backed up in flash.

Value

MC4X0VR memory selected for automatic backup (default)
1TABLE memory selected for automatic backup
MC6640VR memory selected for automatic backup + first 196608 TABLE points (default)
1VR memory and TABLE memory selected for automatic backup
MC508
Flex-6 Nano
MC6N		0VR memory selected for automatic backup (default)
1VR memory and first 64000 TABLE points selected for automatic backup

Examples

Example 1

FLASH_DATA = 0 'Select VR memory for backup

Example 2

FLASH_DATA = 1 'Select TABLE memory for backup
