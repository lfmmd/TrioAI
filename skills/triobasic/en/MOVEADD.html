MOVEADD

Type

Axis Command

Syntax

MOVEADD(distance)

Description

MOVEADD inserts an additional change of position on top of a move that is already running on the axis. The distance moved is in the same user units as the main move, i.e. if the axis is set up by UNITS to be in mm, then MOVEADD adds distance in mm, speeds in mm/s and accelerations in mm/s2.

MOVEADD uses the demands of the axis it is being added to, but these can be overridden by using FORCE_SPEED, FORCE_ACCEL, FORCE_DECEL and FORCE_JERK.

Note that MOVEADD is not an advanced speed control move, and uses the FORCE_SPEED, FORCE_ACCEL, FORCE_DECEL and FORCE_JERK purely as a convenient means of supplying MOVEADD its own demands in the case that they need to differ from the demands of axis having the additional movement.

Note that MOVEADD does not adhere to the VP_MODE setting of the axis it is being added to, instead using its own S-ramped VPU.

The action of MOVEADD is like loading a MOVE on a virtual axis and connecting it to the main axis using ADDAX, but without the need of a virtual axis.

Prefer using a virtual axis and ADDAX to MOVEADD, as this gives extra flexibility with regard how the move is handled (choice of demands, VP_MODE, cancellation, etc.).

To cancel both the move and the additional movement, use CANCEL(5).

To cancel the additional movement part way through its programmed distance, use CANCEL(6).

Parameters

distance:Relative distance of the added movement. A positive value will advance the axis, and a negative value will retard the axis.

Example

Example 1

With continuous motion running on the axis, make an additional move to advance the axis by 40 mm after setting the speed, accelerations and jerk values for the added movement.

FORWARD

WA(100)



FORCE_SPEED = 50

FORCE_ACCEL = 2000

FORCE_DECEL = 2000

FORCE_JERK = 50000

MOVEADD(40)

Example 2

An adjustment of position is needed after capturing the registration position from a sensor on the product. The calculated registration offset can be used in the MOVEADD to advance or retard the axis relative to the Link Axis.

BASE(10)

MOVELINK(1000, 1000, 0, 0, 11)

WAIT UNTIL MARK OR IDLE

IF MARK THEN

    adjustment_dist = calc_reg_adjust(REG_POS)

    FORCE_SPEED = 150

    FORCE_ACCEL = 2000

    FORCE_DECEL = 2000

    FORCE_JERK = 50000

    MOVEADD(adjustment_dist)

    missed_mark_count = 0

ELSE

    missed_mark_count = missed_mark_count + 1

ENDIF
