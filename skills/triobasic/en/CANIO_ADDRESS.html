CANIO_ADDRESS

Type

I/O Parameter (MC_CONFIG only / FLASH)

Description

CANIO_ADDRESS is used to set the operating mode of the CANbus. You can select between Trio CAN, DeviceNet, CANopen or a user configuration when you want to implement your own CANbus protocol using the CAN command.

For most systems CANIO_ADDRESS does not need to be changed from the default value of 32.

The upper byte of CANIO_ADDRESS is used to set extra functions. For example, the CANbus Baud rate in bits per second and some special modes of operation. Certain special functions may need a firmware update to the Trio CAN I/O module.

The value must be set to 32 to enable connection to Trio CAN I/O

The following Motion Coordinators do not set CANIO_ADDRESS in Flash: MC4N-ECAT, MC4N-RTEX, MC404-Z, MC508, MC6N-ECAT, MC664 and Flex 7. MC_CONFIG must be used.

Motion Coordinators MC403(z), MC405, Euro404 and Euro408 with firmware 2.0255 and below do not support bits 8 and 9.

Examples

Example 1

In MC_CONFIG, set up the CANbus port to use the pre-defined CANopen mode with 2 slave nodes, running at 250 kbps.

CANIO_ADDRESS = $100 + 41

Example 2

In MC_CONFIG, set up the CANbus port to be a DeviceNet slave at 125 kbps. This enables the port to come online at the correct Baud rate (required for some PLC DeviceNet masters).

CANIO_ADDRESS = $200 + 33

In a BASIC program, initialise the DeviceNet slave to the following state:

  • 125 kbps
  • Mac ID = 3
  • Polling data starts at TABLE(20)
  • Polled I/O input data = 2 words
  • Polled I/O output data = 4 words
DEVICENET(-1, 0, 125, 3, 20, 2, 4)
