INTERNALIO_BASE

Type

I/O Parameter (MC_CONFIG only)

Syntax

INTERNALIO_BASE = value

Description

The majority of Motion Coordinators have internal I/O addressed at IN(0) to IN(15) and OP(8) to OP(15). If the Motion Coordinator has more than 16 internal I/O points, the extra I/O are addressed from IN/OP(16) and above. These addresses are usually allocated to CAN I/O. Therefore, to allow application programs to have common functionality between the standard 16 I/O points and Motion Coordinators with more than 16 I/O, the parameter INTERNALIO_BASE can be set to split the internal I/O at the 15/16 boundary.

INTERNALIO_BASE sets the start address for the internal I/O point number 16. The internal I/O is then split into 2 groups, 0 to 15 and INTERNALIO_BASE to INTERNALIO_BASE + number of extra internal I/O points.

It is common to set the INTERNALIO_BASE to the next value above the last CAN I/O point.

This command only has a function on Motion Coordinators with more than 16 internal I/O points.

Value

valueSets the base address of the upper part of the internal I/O. (Default = 16)

Examples

Example 1

A Motion Coordinator has 48 internal I/O points. The user is loading a project that runs well on a MC405 with 16 I/O points, but which also uses one CANIO module., and want to keep the same addresses 16 to 31 for the CAN I/O. The default situation is that the CANIO will start at IN(48) so INTERNALIO_BASE is used to change it.

INTERNALIO_BASE = 32

The first bank of internal I/O is now at 0 to 15, the CAN I/O is at 16 to 31 and the remaining internal I/O is at 32 to 63.
