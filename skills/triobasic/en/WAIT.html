WAIT

Type

Command

Syntax

WAIT UNTIL <expression>

WAIT IDLE [AXIS(<axis-number>)]

WAIT IN_POS [AXIS(<axis-number>)]

WAIT LOADED [AXIS(<axis-number>)]

Description

Suspends program execution until the specified condition is satisfied.

The most generic form of this statement is WAIT UNTIL <expression>, as any valid numeric expression may be used. However, there are some shorthand forms used for common tasks, like WAIT IDLE, WAIT IN_POS and WAIT LOADED.

It is very common to use only WAIT IDLE and WAIT LOADED, and in this situation the UNTIL is optional. However, when IDLE and LOADED are part of a more complex expression, the UNTIL is required.

Note that the shorthand forms used to behave differently to the generic form using UNTIL, as the shorthand form would put the process to sleep until the condition was met, whereas using WAIT UNTIL would constantly poll the axis, which is less efficient. This is no longer the case, and both variants now put the process to sleep until the axis has become IDLE/IN_POS/LOADED.

Examples

Example 1

The program waits until the measured position on axis 0 exceeds 150 then starts a movement on axis 7.

WAIT UNTIL MPOS AXIS(0) > 150

MOVE(100) AXIS(7)

Example 2

Force the program to wait until either the current move has finished, or an input turns ON.

Note that because the expression contains IN(12), the UNTIL is required.

MOVELINK(distance, link_dist, acceldist, deceldist, linkaxis)

WAIT UNTIL IDLE OR IN(12) = ON

Example 3

Start a move and then suspend program execution until the move has finished (note that this does not necessarily imply that the axis is stationary in a servo motor system).

MOVE(100)

WAIT UNTIL IDLE

PRINT "Move Done"

Example 4

Start a move and then suspend program execution until the move satisfies the conditions of IN_POS.

MOVE(100)

WAIT UNTIL IN_POS

PRINT "Move in position"

Example 5

Switch output 45 ON at start of MOVE(350) and OFF at the end of that move.

MOVE(100)

MOVE(350)

WAIT UNTIL LOADED

OP(45, ON)

MOVE(200)

WAIT UNTIL LOADED

OP(45, OFF)

Example 6

This behaves identically to example 3.

MOVE(100)

WAIT IDLE

PRINT "Move Done"

Example 7

This behaves identically to example 4.

MOVE(100)

WAIT IN_POS

PRINT "Move in position"

Example 8

This behaves identically to example 5.

MOVE(100)

MOVE(350)

WAIT LOADED

OP(45, ON)

MOVE(200)

WAIT LOADED

OP(45, OFF)
