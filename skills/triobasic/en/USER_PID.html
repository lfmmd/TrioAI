USER_PID

Type

System Function

Syntax

USER_PID(mode, input data VR index, output data VR index)

Description

The USER_PID is a PID function for general purpose use. The function mirrors the PID found in a typical IEC 61131-3 PLC programming system. The function uses global variables VR to pass input, output and control values to the USER_PID.

In mode 1, the USER_PID runs one cycle each time it is called from a BASIC program. It is designed to be used in a program loop with a consistent repeatable period. This is usually achieved by putting a time wait at the end of the loop, using WA(time_in_ms) or TICKS.

In mode 2, the USER_PID is called once only and runs in the Kernel at the SERVO_PERIOD rate.

Function diagram

Parameters

Mode:1 = Run one pass of the PID control per call.
2 = Run continuously and automatically in the controller at the SERVO_PERIOD rate.
Input_data_index:The base address of a block of VRs that contain the input data.
Output_data_index:The base address of a block of VRs that contain the output data.

Input data

Input values (offset from VR(input_data))

0AUTOTRUE = auto / FALSE = manual

3XOUT_MANUoutput value for manual mode

10I_SELIf FALSE the integral term is ignored

12I_ITL_ONTRUE = reset integral sum to I_ITLVAL

Examples

Example 1

Using a virtual axis to simulate the process to be controlled by USER_PID.

input_data_index = 100

GLOBAL "p_auto", 100

GLOBAL "p_pv", 101

GLOBAL "p_sp", 102

GLOBAL "p_xout_manu", 103

GLOBAL "p_kp", 104

GLOBAL "p_ti", 105

GLOBAL "p_td", 106

GLOBAL "p_ts", 107

GLOBAL "p_xmin", 108

GLOBAL "p_xmax", 109

GLOBAL "p_i_sel", 110

GLOBAL "p_int_hold", 111

GLOBAL "p_i_itl_on", 112

GLOBAL "p_i_itlval", 113

GLOBAL "p_deadb_err", 114

GLOBAL "p_ffd", 115



output_data_index = 120

GLOBAL "p_xout", 120

GLOBAL "p_er", 121

GLOBAL "p_xout_p", 122

GLOBAL "p_xout_i", 123

GLOBAL "p_xout_d", 124

GLOBAL "p_xout_hlm", 125

GLOBAL "p_xout_llm", 126



'Initialise PID settings

p_xout_manu = 0

p_kp = 70.0

p_ti = 300

p_td = 2

p_ts = 1

p_xmin = -2048

p_xmax = 2048

p_i_sel = TRUE

p_int_hold = FALSE

p_deadb_err = 0

p_ffd = 0



'Start the virtual "target process"

GOSUB set_test_target



'Reset the USER_PID

p_auto = FALSE

p_i_itlval = 0

p_i_itl_on = TRUE

USER_PID(1, input_data_index, output_data_index)



p_auto = TRUE

p_i_itl_on = FALSE



WHILE TRUE

  BASE(10)

  p_pv = MPOS

  USER_PID(1, input_data_index, output_data_index)

  SPEED = p_xout

  ACCEL = ABS(SPEED) * 40

  DECEL = ACCEL

  WA(10)

WEND



set_test_target:

'Set up a virtual axis to act as the process value being controlled

BASE(10)

DEFPOS(0)

ATYPE = 0

UNITS = 1000

SERVO = ON

WDOG = ON

SPEED = 0 'Initial speed is 0

ACCEL = 10000000 'Instant accel

DECEL = ACCEL

IF IDLE THEN FORWARD

'The PID output will set the speed value

'The MPOS will be the feedback PV value

RETURN

Example 2

Uses AIN(0) as the feedback from the external process and AOUT(2) as the control output.

input_data_index = 100

GLOBAL "p_auto", 100

GLOBAL "p_pv", 101

GLOBAL "p_sp", 102

GLOBAL "p_xout_manu", 103

GLOBAL "p_kp", 104

GLOBAL "p_ti", 105

GLOBAL "p_td", 106

GLOBAL "p_ts", 107

GLOBAL "p_xmin", 108

GLOBAL "p_xmax", 109

GLOBAL "p_i_sel", 110

GLOBAL "p_int_hold", 111

GLOBAL "p_i_itl_on", 112

GLOBAL "p_i_itlval", 113

GLOBAL "p_deadb_err", 114

GLOBAL "p_ffd", 115



output_data_index = 120

GLOBAL "p_xout", 120

GLOBAL "p_er", 121

GLOBAL "p_xout_p", 122

GLOBAL "p_xout_i", 123

GLOBAL "p_xout_d", 124

GLOBAL "p_xout_hlm", 125

GLOBAL "p_xout_llm", 126



'Initialise PID settings

p_xout_manu = 0

p_kp = 5.0

p_ti = 40

p_td = 0.5

p_ts = 1

p_xmin = -2048

p_xmax = 2048

p_i_sel = TRUE

p_int_hold = FALSE

p_deadb_err = 0

p_ffd = 0



'Reset the USER_PID

p_auto = FALSE

p_i_itlval = 0

p_i_itl_on = TRUE

USER_PID(1, input_data_index, output_data_index)



p_auto = TRUE

p_i_itl_on = FALSE



WHILE TRUE

  TICKS = 20 'Set cycle time

  p_pv = AIN(0)

  USER_PID(1, input_data_index, output_data_index)

  AOUT(2) = p_xout

  REPEAT

    WA(1) 'Sleep the task for remainder of cycle time

  UNTIL TICKS <= 0

WEND

Example 3

Using the same GLOBAL initialisation as example 2, the USER_PID is started in mode 2, after which it runs in the kernel at the SERVO_PERIOD rate.

'Reset the USER_PID

p_auto = FALSE

p_i_itlval = 0

p_i_itl_on = TRUE

USER_PID(1, input_data_index, output_data_index)



p_auto = TRUE

p_i_itl_on = FALSE



USER_PID(2, input_data_index, output_data_index)



'VR(101) has the process value

'VR(102) has the setpoint value

'VR(120) contains the output value
