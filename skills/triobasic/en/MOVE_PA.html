MOVE_PA

Type

Axis Parameter

Description

MOVE_PA is a special axis parameter which pass through the motion buffers and become available during the motion for processing.

This and MOVE_PB are two axis parameters which can be freely set by BASIC. Every time a move is loaded by BASIC a copy is made of both these parameters into the move queue buffers.

This parameter is used to switch outputs linked to moves. For example switching ON a certain output once a move command is executed.

Every time a move is loaded by BASIC a copy is made of both these parameters into the move queue buffers:

Parameters

MOVE_PA_CONT = 1 or 3

Bits 0…11:Set the output number to be switched
Bit 12:Sets ON/OFF

MOVE_PA_CONT = 2

Bits 0…31:The value in MOVE_PA is placed in AXIS_DEBUG_A synchronised with the loading of the move.

MOVE_PA_CONT = 4

Bits 0…11:Set the first output index
Bits 12…19:The number of outputs to be set/cleared
Bits 20…51:The output bit pattern

Mode 4 is available in firmware V2.0284 and later

MOVE_PA_CONT = 5 or 6

Bits 0…31:Analogue 32-bit value
Bits 32…39:Analogue channel number

Modes 5 and 6 are available in firmware v2.0339 and later

Examples

Example 1

Switching output number 10 during a move once the command is executed:

SPEED = 15

FORCE_SPEED = 15

ENDMOVE_SPEED = 15

MERGE = ON

LIMIT_BUFFERED = 8

ACCEL = SPEED * 20

DECEL = SPEED * 20

TRIGGER



MOVE_PA = 10

MOVE_PA_CONT = 1

MOVE_PA.12 = ON

MOVESP(200)

MOVE_PA.12 = OFF

MOVESP(20) 

MOVE_PA.12 = ON

MOVESP(100)

Example 2

Set a bit pattern on outputs 8 to 15 to coincide with the start of the next move.

SPEED = 15

FORCE_SPEED = 15

ENDMOVE_SPEED = 15

MOVE_PA_CONT = 4

MOVE_PA = ($AA << 20) + (8 << 12) + 8

MOVE(200)
