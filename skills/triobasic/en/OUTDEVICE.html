OUTDEVICE

Type

Process Parameter

Description

The value of this parameter determines the default output device for the process it is issued within, allowing the channel number to be set for all statements that generate output, like PRINT, AXISLIST, DIR, etc.

This command is process specific so other processes will use the default channel.

This command is available for backward compatibility, it is currently recommended to use #channel, instead. However, it remains useful for commands that do not accept a channel number, like DIR.

Value

The channel number to use for output

For a full list of communication channels see #

Examples

Example 1

Set up a program to print all data to channel 5.

OUTDEVICE = 5



IF error THEN

  PRINT "Error Detected"

ENDIF

Example 2

Change the output device to enable the output from a command to go into a string variable and subsequent conversion to a value.

DIM old_outdevice AS INTEGER

DIM my_str AS STRING(20)

DIM crc AS INTEGER



old_outdevice = OUTDEVICE

OPEN #40 AS "STRING:my_str" FOR OUTPUT(0)

OUTDEVICE = 40

FILE "CRC16" "FILE1.TXT"

OUTDEVICE = old_outdevice

CLOSE #40



crc = VAL(my_str)

PRINT "CRC-16 of file 'FILE1.TXT' is "; crc

Will display:

CRC-16 of file 'FILE1.TXT' is 51612
