Robot Selection and Program Types

RPS can configure up to 8 robots in the same system. To let the program knows what Robot should direct all subsequent motion commands and other parameters to read/write, a robot must be selected first.

Select and Release a Robot

When a Robot is selected, the robot will stay with that process until the movements produced in that process are finished and the program stops, i.e. the process has been terminated. Stopping a program will not automatically release a robot. Only until move buffers are all idle will the robot be released when a program stops (its process finishes).

Syntax

ROBOT(function | identity)
function: INTEGER-1 = deselect/release a robot from its process. This function can only be executed from the same process as where the robot is selected.
identity: INTEGERRobot ID. Value: 0 to 7.

Robot will only be released from the process where it is selected, i.e. ROBOT(-1) is executed in the same program where the robot is selected.

Examples

Example 1

Select Robot 2 and move joint to a target.

DIM jtgt1 AS TARGETJ

jtgt1 = 10,10,10,0,0,0



ROBOT(2)

MOVEJ jtg1

Example 2

Release Robot 2 after moving to a target.

DIM jtgt1 AS TARGETJ

jtgt1 = 10,10,10,0,0,0



ROBOT(2)

MOVEJ jtg1

WAIT IDLE



ROBOT(-1)

Example 3

Check what process hold Robot 2 and store it in a variable called “my_robot”.

my_robot = ROBOT(2)

Stop a Robot

It is possible to stop a robot from another process. This enables controlling a robot from a main program.

When a robot is stopped using STOP_ROBOT command, all the moves are cancelled, buffers are cleared, and the process that owns the robot is stopped.

Syntax

STOP_ROBOT(identity)
identity: INTEGERRobot ID. Value: 0 to 7.

Disable a Robot

It is possible to inhibit any robot programs from running. Any running programs will be stopped, and buffers will be cleared.

Syntax

DISABLE_ROBOT(flag)
flag: BOOLEANDisable flag. Value: TRUE to disable running robot programs; FALSE to enable running robot programs.

Program Types

There are two types of program to control robots: robot programs and robot Basic programs. Both can be edited in Motion Perfect.

Robot Program

Robot programs contain only instructions that TPS can handle, among others, MOVE instructions, Structures (While, If, Repeat, For structures), function calls, and labels. Robot programs will have a ‘.ROB’ file extension.

Below is a list of commands that can be used in Robot Programs:
AOUTGTAMOVELSTRINGWORLD_ORI_SPEED
ASGTAJMOVELRELTARGETWORLD_POS_SPEED
BASEIDLEOBJECT_FRAMETARGETJ
BOOLEANIF-ELSEREPEAT-UNTILTOOL_OFFSET
DIMINTEGERRETURNVR
FLOATMOVECROBOTWA
FOR-NEXTMOVEJSTOPWAIT
GOSUBMOVEJRELSTOP_ROBOTWHILE-WEND

Robot Basic Program

Robot Basic Programs can contain all Trio instructions as in BASIC programs).

Robot Basic Program can be edited by TPS only for user level 1. The commands that are not in guided form will be inserted by direct form.

Users can write complex programs as Robot Basic programs to have more control over their application. It is important to use this type of program and NOT a BASIC program as the controllers can terminate this type of program in case of an emergency.

This type of program will have ‘.RBS’ file extension

Robot Library

Robot Library contains user functions for the robot application. TPS can read this file and list all the functions in the Teach pendant. Commands allowed in Robot Library files are the same as in Basic Library files.

This type of library will have a ‘.ROBLIB’ file extension.
