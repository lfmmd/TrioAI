SET_EIP_TAG

Type

System Function

Syntax

SET_EIP_TAG(name, mem_area, address, data_type)

Description

Define a new CIP tag entry (a symbol segment data type) or update the configuration of an existing tag if the tag name has already been defined.

All the currently defined tags will be deleted if this command is called without any parameters.

Parameters

name:Name of the tag (of data type string)
mem_area:1: VR, 2: TABLE
address:Address within the memory area of the tag
data_type:The datatype of the tag, using the values as defined in the CIP Spec volume 1, section C-6.1.
Data types currently supported include:
Data Type NameData Type CodeDescription
SINT$C2Signed 8-bit integer
INT$C3Signed 16-bit integer
DINT$C4Signed 32-bit integer
USINT$C5Unsigned 8-bit integer
UINT$C6Unsigned 16-bit integer
UDINT$C7Unsigned 32-bit integer

Return value

TRUE (-1) : Successful, FALSE (0) : Failed

Examples

Example 1

Set up tag names for x and y position located in VR(200) and VR(201)

SET_EIP_TAG("x_pos", 1, 200, $c3)

SET_EIP_TAG("y_pos", 1, 201, $c7)

Display tags by command in the terminal:

>>GET_EIP_TAG

x_pos: Memory: 1 (Vr), Address: 200, Datatype: 0xC3

y_pos: Memory: 1 (Vr), Address: 201, Datatype: 0xC7

Example 2

Change the address of the ‘y_pos’ tag in example 1

>>SET_EIP_TAG("y_pos", 1, 150, $c7)

Display tags

>>GET_EIP_TAG                   

x_pos: Memory: 1 (Vr), Address: 200, Datatype: 0xC3

y_pos: Memory: 1 (Vr), Address: 150, Datatype: 0xC7

>>

Example 3

Delete all tags ready to start configuring tags again.

SET_EIP_TAG

Display the tag store to confirm that it is empty.
>>GET_EIP_TAG
>>
