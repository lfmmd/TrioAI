INTEGER_READ

Type

Mathematical Function

Syntax

INTEGER_READ(source, least_significant, most_significant)

Description

INTEGER_READ performs a low-level read of a 64-bit source, splitting it into two 32-bit values, where each value represents the bit pattern of the source’s least significant and most significant 32-bit words. The source may be integer or floating-point, as it is the bit pattern (i.e. internal representation) that is read, not its value or a cast of its value.

This can be used to read the position from high resolution encoders

Parameters

source:64-bit value that will be read, can be VR, TABLE, or system variable.
least_significant:The variable to store the least significant (rightmost) 32 bits, this may be local variable, VR or TABLE
most_significant:The variable to store the most significant (leftmost) 32 bits, this may be local variable, VR or TABLE

Examples

Example 1

Read the floating-point number stored in VR(1) as two integer values and display them hexadecimal format.

VR(1) = 0.9999

INTEGER_READ(VR(1), VR(4), VR(5))

PRINT VR(1); "    ";

PRINT "MSW "; HEX(VR(5)), "LSW "; HEX(VR(4))

Would display:

0.9999    MSW 48E8A71E     LSW 3FEFFF2E

Example 2

Read the ENCODER value as an integer.

INTEGER_READ(ENCODER AXIS(2), VR(4), VR(5))

PRINT "Encoder Value "; VR(5)[0]
