Errors

RPS programs can generate errors due to errors in the program structure, or type of variables used, or as the result of a command being unsuccessfully executed.

Most errors are reported using a channel 9 status message which will be displayed as an error in Motion Perfect. Some commands return a value which indicates the error status.

In addition to error codes, supplementary information may be given in the axis status and the robot status, which are accessed using the AXISSTATUS and ROBOTSTATUS keywords.

Axis Status

The axis status is a 64-bit mask where each bit has a distinct meaning.

See AXISSTATUS for details of each bit’s meaning.

Robot Status

The robot status is a 32-bit mask where each bit has a distinct meaning.

See ROBOTSTATUS for details of each bit’s meaning.

Robot Log

A logging system is provided to store events in flash that can happen at any time. Up to 2048 entries are stored in a cyclic buffer.

A logged event comprises the index, the date and time it occurred, the controller up-time, and a description of the event.

Syntax

ROBOT_LOG
ROBOT_LOG(function)
ROBOT_LOG(start_index[, end_index])
ROBOT_LOG(name)
(optional) functionLess than or equal to zero
-1Delete all entries
(optional) start_indexStarting log index to read (indexed from 1)
(optional) end_indexEnding log index to read
(optional) nameName of destination file where to store robot log messages to.

Examples

Example 1

Display all robot log messages on Terminal.

Terminal

>>robot_log    

   1 23/Jan/2018 13:15:10 [    4618.272] Wrist Singularity axis [4]

   2 23/Jan/2018 13:15:12 [    4620.961] Blend max radius reached: 89 Units

   3 23/Jan/2018 13:15:13 [    4620.963] Blend max radius reached: 121 Units

   4 23/Jan/2018 13:15:13 [    4621.130] Encompassed point: 350,150,550,-120,16,-100

   5 23/Jan/2018 13:15:14 [    4622.739] Wrist Singularity axis [4]

   6 23/Jan/2018 13:15:14 [    4622.739] Max speed reached: 356.445313 Units/s in axis 5 

<END OF LOG>
