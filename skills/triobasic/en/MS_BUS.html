MS_BUS

Type

System Command

Syntax

return code = MS_BUS(function, slot[, parameters...])

Description

The command MS_BUS is used to perform advanced operations on the MS-Bus network. In normal use the MS-Bus network will start automatically without the need for any commands in a startup program. MS_BUS command functions may be useful when debugging and setting up a MS-Bus system.

Parameters

functionFunction to be performed
$00Start MS-Bus network
$01Stop MS-Bus network
$02Check slice online
$03Check number of slices
$04Print slice modules
$05Read errors on the network
$06Returns status of network
$07Print network error information
$08Asynchronous read from slice register
$09Asynchronous write to slice register
$0ARead slice serial number
$0BRead slice type
$0CRead slice status register
$0DRead slice warning bitmap
slotSet to MS-Bus port slot number
1MotionPLC

Function = $00: Start MS-Bus network

Syntax

error = MS_BUS(0, slot[, silent])

Description

Initialise MS-Bus network and put it onto operational mode.

Parameters

slotSet to MS-Bus port slot number
1MotionPLC
silentLimits the detail printed to the terminal (optional)
omittedPrint full detail to the device specified by OUTDEVICE
0Suppress printing (silent)
1 (bit-0)Prints list of slices found with FPGA version number
2 (bit-1)Prints list of slices found with hardware version number
4 (bit-2)Prints list of slices found with serial number

Note: ‘silent’ bit can be combined to print additional data

Returns

TRUE if the network initialised successfully and FALSE if errors occurred during initialisation.

Example

Check the MS-Bus state and if not in Operational State, restart the MS-Bus and set an output to indicate that a re-start is in progress.

'--Init MS-Bus if needed.

chk = MS_BUS($06, 1) 'Test state



IF chk <> 3 THEN

  OP(9, ON)

  WA(1500) 'wait 1.5sec for drive to power up

  MS_BUS(0, 1, 0) 'Initialise MS-Bus with no terminal output

ENDIF

Function = $02: Check slice online

Syntax

state = MS_BUS(2, slot, slice_id)

Description

Check whether given slice is online. Returns TRUE if a slice with a given slice ID is online.

If there is an MS-Bus error, then the online status of all slices is returned as FALSE.

Parameters

slotSet to MS-Bus port slot number
1MotionPLC
slice_idSlice number on MS-Bus Network

Returns

TRUE if a slice is online.

FALSE if a slice is offline.

Example

Check if the slices at network positions 0 to 7 have gone offline.

FOR posn = 0 TO 7

  IF MS_BUS(2, 1, posn) = FALSE THEN

    PRINT "MS I/O Slice at "; posn[0]; " is offline"

  ENDIF

NEXT posn

Function = $05: Read error type

Syntax

error_bitmap = MS_BUS(5, slot)

Description

In the case of an error on the MS-BUS, SYSTEM_ERROR will show bit-21 ($00200000). Additional detail about the MS-BUS error can be found by reading the MS-BUS error type.

A value of zero indicates no error.
Error bitValueDescription
Bit 01CRC error has occurred on the network.
Bit 12Frame error has occurred on the network. This indicates a slice did not respond with the correct frame.
Bit 24Break error has occurred on the network. This indicates a physical break in the connection between slices
Bit 38Timeout error has occurred on the network. This indicates a slice has failed to respond within its time deadline.
Bit 416Reserved
Bit 532Slice ID Incorrect.
Bit 664Slice failed to reset.
Bit 7128Too many slices
Bit 8256Slice config invalid. Check MC_CONFIG.
Bit 9512Slice info invalid.
Bit 101028Command timeout.
Bit 112046Empty FIFO.

Parameters

slot:Set to MS-Bus port slot number
1MotionPLC

Returns

Error bitmap.

Example

Print network error. This error indicates there are too many slices on the network.

>>PRINT MS_BUS(5,1) 

128

Function = $08: Asynchronous read from Slice register

Syntax

value = MS_BUS($8, slot, slice_id, reg_addr[, vr_index])

This function operates differently depending on the optional parameter passed.

  • If vr_index is omitted, then the returned value is the async read data.
  • If vr_index is -1, then the returned value is the async read data.
  • If vr_index is 0 or above, then the returned value is the error code.

Description

Read a register from a slice.

Parameters

slot:Set to MS-Bus port slot number
1MotionPLC
slice_id:Slice number on MS-Bus Network
reg_addr:Address of the register in a Slice
vr_index:0 to VRSIZE - 1Result is placed in the selected VR variable.
-1Result is returned.

Returns

If using VR for read data:

TRUE if the command completed successfully.

FALSE if the command failed to complete successfully.

If not using VR for read data:

Value read from slice.
AddressDescription
$70Slice Information
Bits 0 to 7: Slice Type
Bits 8 to 11: Reserved
Bits 12 to 15: Hardware version
Bits 16 to 23: FPGA version
Bits 24 to 31: Firmware version
$44Bitmap of Digital Output state when MS-Bus is not operational, see MS_BUS_CONFIG
$43Analogue Input range selection, see MS_BUS_CONFIG
$47Digital Input filter time, see MS_BUS_CONFIG

Example:

Example below reads data and uses a VR location to store the data.

'Check for valid read of reg $47 from slice 12, store in VR(2)

If MS_BUS(1, 8, 12, $47, 2) = TRUE then

  PRINT "Slice data "; VR(2)

ELSE

  PRINT "ASYNC read failed"

ENDIF

Example below reads data and uses a VR location to store the data.

'Read of reg $47 from slice 12, store in VR(2)

rd_value = MS_BUS(1, 8, 12, $47)

PRINT "Slice data "; rd_value

Function = $0C: Read slice status

Syntax

slice_status = MS_BUS($0C, slot, slice_id, [mode])

Description

Returns slice status. This value is read from every slice to monitor any warnings or errors on the slice.

The 8-bit slice status register is defined as:
BitDigital Slice bitmapAnalogue Slices bitmap
0Sync manager status [0]Sync manager status [0]
1Sync manager status [1]Sync manager status [1]
2Sync manager status [2]Sync manager status [2]
3Channel 0 or Channel 1 CRC errorChannel 0 or Channel 1 CRC error
4Channel 0 or Channel 1 Frame errorChannel 0 or Channel 1 Frame error
5Channel 0 or Channel 1 Break errorChannel 0 or Channel 1 Break error
6Output Bank 1 fault or temperature warningUnused
7Output Bank 0 fault or temperature warningUnused

The 16-bit slice status register is defined as:
BitDescription
0Channel 0 RX CRC Error
1Channel 0 RX Frame Error
2Channel 0 RX Break Error
3Channel 0 RX Timeout Error
4Channel 1 RX CRC error
5Channel 1 RX Frame error
6Channel 1 RX Break error
7Channel 1 RX Timeout error
8Sync Timeout Error
9Temperature Monitor Flag (Active Low) (Supported in FPGA Version >= 11)
10Channel 0 Output driver flag (Supported in FPGA Version >= 11)
11Channel 1 Output driver flag (Supported in FPGA Version >= 11)
12 - 31Unused

Parameters

ModeDescription
0 [Default]Returns slice’s status register read during cyclic transfer (8-bit)
1Asynchronous read of extended slice status register (16-bit)

Example 1

Read the cyclic slice status register from slice 2.

slice_status = MS_BUS($c, 1, 2)

IF slice_status AND 64 THEN

   PRINT "Channel 1 warning on slice 2"

ENDIF

Example 2

Read extended slice status register from slice 5.

extended_status = MS_BUS($c, 1, 5, 2)

IF enteded_status AND 512 THEN

   PRINT "Temperature fault on slice 5"

ENDIF

Function = $0D: Read slice status bitmap

Syntax

warning_slices = MS_BUS($0D, slot)

Description

Returns a bitmap of slices that caused SYSTEM_WARNING to be set.

Example

Pring slices that raised a SYSTEM_WARNING.

IF SYSTEM_WARNING AND $40 THEN

  warning_map = MS_BUS($d, 1)

  FOR i = 0 to 15 

    IF warning_map AND (1 << i) THEN

      PRINT "Slice "; i; "raised a warning"

    ENDIF

  NEXT i

ENDIF
