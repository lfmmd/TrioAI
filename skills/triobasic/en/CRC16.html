CRC16

Type

Mathematical Function

Syntax

result = CRC16(mode,{parameters})

Description

Calculates a 16 bit Cyclic Redundancy Check (CRC) of data stored in contiguous Table Memory or VR Memory locations.

Parameters

mode:0Initialise the polynomial
1Calculate the CRC (legacy mode)
2Calculate the CRC

CRC16 must be initialised at least once by Mode 0 before running the Mode 1 calculations.

Mode = 0

Syntax

result = CRC16(0, poly[, reverse_input [,reverse_output [, bits]]])

Description

Initialises the command with the Polynomial

Parameters

result:Always returns -1
poly:Polynomial used as seed for CRC check range 0 to 65535 (0 to $FFFF)
reverse_input:If reverse_input is 0 then the most significant bit of each character in the input is on the left, otherwise it is on the right. This parameter defaults to 0.
reverse_output:If reverse_output is 0 then the most significant bit of the CRC is on the left, otherwise it is on the right. This parameter defaults to 0.
bits:The number of bits in the CRC calculation this must be in the range 8 to 32. This parameter defaults to 16.

Mode = 1

Syntax

result = CRC16(1, source, start, end, initial)

Description

Calculates the CRC in legacy mode. This mode has the following restrictions:

  • The optional parameters are ignored and should not be specified in the CRC16(0, ...) command.
  • The calculated CRC value must be “flushed” by either appending to two ASCII NUL (numeric 0) characters at the end of the data to be processed, or performing an additional CRC16(1,...) on 2 ASCII NUL characters
  • Once the CRC has been flushed it cannot be used to carry on processing more data.

Parameters

result:Returns the result of the CRC calculation. Returns 0 if the calculation fails or the CRC16 was not initialised.
source:Defines where the data is loaded
0Table Memory
1VR Memory
start:Start location of first byte
end:End Location of last byte
initial:Initial CRC value. Normally $0 to $FFFF

Mode = 2

Syntax

result = CRC16(2, source, start, end, initial)

Description

Calculates the CRC. The reverse_input and reverse_output flags are respected, and no additional processing is required. As no flush is required, we can add more data to the CRC calculation at any time.

Parameters

result:Returns the result of the CRC calculation. Returns 0 if the calculation fails or the CRC16 was not initialised.
source:Defines where the data is loaded
0Table Memory
1VR Memory
start:Start location of first byte
end:End Location of last byte
initial:Initial CRC value. Normally $0 to $FFFF

Examples

Examlpe 1

Calculate the CRC using Table Memory (legacy mode):

poly = $8005

CRC16(0, poly) 'Initialise internal CRC table memory



'Load data into TABLE memory location 0-7

'Load flush data into TABLE memory at location 8-9

TABLE(0, 1, 2, 3, 4, 5, 6, 7, 8, 0, 0)

reginit = 0

calc_crc = CRC16(1, 0, 0, 9, reginit) 'Source Data = TABLE(0..7)

Example 2

Calculate the CRC using TABLE and flush from VR (legacy mode):

poly = $8005

CRC16(0, poly) 'Initialise internal CRC table memory



'Load data into TABLE memory location 0-7

TABLE(0, 1, 2, 3, 4, 5, 6, 7, 8, 0, 0)

'Load flush data into VR memory at location 0

VR(0) = 0

reginit = 0

calc_crc = CRC16(1, 0, 0, 7, reginit) 'Source Data = TABLE(0..7)

calc_crc = CRC16(1, 1, 0, 0, calc_crc) 'Flush first byte

calc_crc = CRC16(1, 1, 0, 0, calc_crc) 'Flush second byte
