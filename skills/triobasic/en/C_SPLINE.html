C_SPLINE

Type

Command

Syntax

C_SPLINE(mode, index_in, number_points, dimensions, expand_ratio, index_out{, tension, bias})

Description

The C_SPLINE command operates in two modes using Cubic Hermite splines to extend the source points into a smoother higher resolution profile. This can be used to smooth motion profiles which can be run in a sequence using MOVEABS, MOVABSSEQ or similar.

The expand_ratio parameter gives the number of points that each position is expanded by, so the total number of output points is (((number_points - 1) * expand_ratio) + 1)

Parameters

mode:1 and 3Catmull Rom Spline
2 and 4Full cardinal spline
index-inThe location of the input points in TABLE memory (grouped for example x1, y1, z1, x2, y2, y3, …)
number_pointsThe number of input points in TABLE memory (Each point has 2 to 8 dimensions)
dimensionsThe number of dimensions of the data (2 to 8)
expand_ratioThe number of intermediate points generated between each input point (2 to 100)
index_outThe location of the output points in TABLE memory (Format depends on “mode”)
tensionUsed for Cardinal splines only. (-1 to 1) -1 is “loosest”, 1 is “tightest”
biasUsed for Cardinal splines only. (-1 to 1, normally 0)

Mode = 1; Mode = 3

Syntax

C_SPLINE(mode, index_in, number_points, dimensions expand_ratio, index_out)

Description

Catmull-ROM (modes 1 and 3) is a commonly used particular case of the full Cardinal spline equations (modes 2 and 4). If the “tension” and “bias” parameters are set to zero the Cardinal spline will be the same as a Catmull-Rom.

Mode 3 generates points where the outputs in the different dimensions are grouped together. Mode 1 generates points where the outputs in the same dimensions are grouped together. For example, for 3 dimensions XYZ and points 1, 2, 3, 4.

Mode 3 will therefore output data in sequence X1, Y1, Z1, X2, Y2, Z2, X3, Y3, Z3, X4, Y4, Z4, …

Modes 1 will therefore output data in sequence X1, X2, X3, X4, Y1, Y2, Y3, Y4, Z1, Z2, Z3, Z4, …

Parameters

index_in:The location of the input points in TABLE memory (grouped for example x1, y1, z1, x2, y2, y3, …)
number_points:The number of input points in TABLE memory (each point has 2 to 8 dimensions)
dimensions:The number of dimensions of the data (2 to 8)
expand_ratio:The number of intermediate points generated between each input point (2 to 100)
Total output points = (Number of points-1) * expansion + 1
(i.e. if the source profile is 100 points and the expansion ratio is set to 10 the resulting profile will be 1010 point ((100 - 1) * 10 + 1).
index_out:The location of the output points in TABLE memory (format depends on “mode”)

Example

Expands a 2-dimensional profile, defining 7 points in the controller TABLE memory at location 1000:

npoints = 7 

TABLE(1000, 0, 0) 

TABLE(1002, 100, 0) 

TABLE(1004, 100, 100) 

TABLE(1006, 200, 100) 

TABLE(1008, 300, 100) 

TABLE(1010, 300, -100) 

TABLE(1012, 150, -100)



inp = 1000 

outp = 4000 

expand = 20 'Max 100, min 2

C_SPLINE(3, inp, npoints, 2, expand, outp)

Mode = 2; Mode = 4

Syntax

C_SPLINE(mode, index_in, number_points, dimensions expand_ratio, index_out, tension, bias)

Description

Mode 2 and 4 will perform a Cubic Hermite spline over a multidimensional (from 2 to 8 dimensions) set of points, obtaining a smoother path depending on the expansion rate.

This is ideally used where the source of points MOVEABS or MOVEABSSEQ profile is too coarse and needs to be extrapolated into a greater number of points

Mode 4 generates points where the outputs in the different dimensions are grouped together. Mode 2 generates points where the outputs in the same dimensions are grouped together. For example, for 3 dimensions XYZ and points 1,2,3,4.

Mode 4 will therefore output data in sequence X1, Y1, Z1, X2, Y2, Z2, X3, Y3, Z3, X4, Y4, Z4, …

Modes 2 will therefore output data in sequence X1, X2, X3, X4, Y1, Y2, Y3, Y4, Z1, Z2, Z3, Z4, …

Parameters

index-inThe location of the input points in TABLE memory (grouped for example x1, y1, z1, x2, y2, y3, …)
number-pointsThe number of input points in TABLE memory (each point has 2 to 8 dimensions)
dimensionsThe number of dimensions of the data (2 to 8)
expand-ratioThe number of intermediate points generated between each input point (2 to 100)
Total output points = (Number of points-1) * expansion + 1
(i.e. if the source profile is 100 points and the expansion ratio is set to 10 the resulting profile will be 1010 point ((100 - 1) * 10 + 1).
index-outThe location of the output points in TABLE memory (format depends on “mode”)
tensionUsed for Cardinal splines only. (-1 to 1) -1 is “loosest”, 1 is “tightest”
biasUsed for Cardinal splines only. (-1 to 1, normally 0)

Example

Generate a C-spline from control points in TABLE data, outputting the spline in TABLE data for use with moves that accept position in TABLE data, such as MOVEABSSEQ.

TABLE(1000, 136, 170)

TABLE(1002, 136, 206)

TABLE(1004, 150, 220)

TABLE(1006, 152, 220)



npoints = 4 'VR(1)

inp = 1000

outp = 2000



expand = 40 'Max 100, min 2



PRINT "output x start = "; outp

PRINT "output y start = "; outp + (npoints - 1) * expand + 1

PRINT "number of output points = "; (npoints - 1) * expand + 1
