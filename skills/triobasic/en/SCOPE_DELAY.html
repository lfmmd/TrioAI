SCOPE_DELAY

Type

System Parameter

Description

This parameter enables pre-triggering in the SCOPE in milliseconds.

If the SCOPE_DELAY is less than 0 then the SCOPE will capture SCOPE_DELAY milliseconds of pre-trigger data.

If the SCOPE_DELAY is greater than 0 then the SCOPE will delay the start of capture by SCOPE_DELAY milliseconds.

When SCOPE_DELAY is negative and the time between the SCOPE command and the TRIGGER is less than -SCOPE_DELAY, only the points captured so far will be saved and the SCOPE_TRIGGER_POS will be set according to the amount of data actually captured.

Value

The scope trigger delay in milliseconds.

Examples

Example 1

Set Scope for delayed trigger of 20 ms. Print a message to show scope capture completed.

'Capture 200 points on 4 channels

'Delay trigger by 20 ms

SCOPE_DELAY = 20

SCOPE(ON, 1, 10001, 10800, DPOS AXIS(2), MPOS AXIS(2), FE AXIS(2), VP_SPEED AXIS(2))

WAIT UNTIL IN(60) = OFF

WAIT UNTIL IN(60) = ON 'Wait for rising edge on IN(60)

TRIGGER

WAIT UNTIL SCOPE_POS = 200

PRINT #5, "Scope capture complete"

PRINT #5, "Trace 1 starts at TABLE(10001)"

PRINT #5, "Trace 2 starts at TABLE(10201)"

PRINT #5, "Trace 3 starts at TABLE(10401)"

PRINT #5, "Trace 4 starts at TABLE(10601)"

Example 2

Set Scope to pre-trigger at 500 ms. Print a message with scope completed and the trigger position to terminal #5.

'Capture 1000 points on 2 channels

'pre-trigger to capture 500 points before the TRIGGER 

'and 500 after TRIGGER

SCOPE_DELAY = -500

SCOPE(ON, 1, 10001, 12000, MPOS AXIS(0), MPOS AXIS(1))

WA(320)

TRIGGER

WAIT UNTIL SCOPE_POS = 1000

PRINT #5, "Scope capture complete"

PRINT #5, "Actual Trigger at TABLE "; 10000 + SCOPE_TRIGGER_POS

This deliberately does not leave enough time to capture 500 points. It is to show the use of SCOPE_TRIGGER_POS.
