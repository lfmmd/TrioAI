RUN_ERROR

Type

Process Parameter (Read Only)

Description

Contains the integer number of the last run time error that stopped the program on the specified process.

RUN_ERROR = 31 is a normal completion of a program.

Example

Example 1

Use the command line to check why a program that was running on process 5 has stopped. The result of 9 indicates a divide by zero error.

>>?RUN_ERROR PROC(5)

9

>>

Example 2

In a program, test the error value and print the result to the Motion Perfect terminal #5. Routine is called from ON BASICERROR GOTO error_report.

error_report:

SELECT_CASE RUN_ERROR

  CASE 1

    PRINT #5, "Command not recognised"

  CASE 9

    PRINT #5, "Divide by zero"

  CASE 181

    PRINT #5, "Device error: No SD card inserted"

  CASE ELSE

    PRINT #5, "Error occurred, number "; RUN_ERROR

END_CASE

STOP
