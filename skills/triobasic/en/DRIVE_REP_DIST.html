DRIVE_REP_DIST

Type

Axis Parameter

Description

Sets the wrap-around value that the axis is expecting from an EtherCAT or RTEX servodrive. The default axis roll-over is at 232 as this is the data size for cyclic position transfers over the fieldbus, but if the drive or encoder wrap at a lower number than this, it will create an axis error if the Motion Coordinator is expecting the axis to continue counting up.

The DRIVE_REP_DIST should be set in cases where the motor has a multi-turn absolute encoder fitted and the drive has been set up to roll-over to 0 at some position lower than 232. The value of DRIVE_REP_DIST must be the same as the position roll-over that is set in the drive or fieldbus encoder.

This parameter enables the axis in the controller to coordinate the actual position reset to 0 with the remote device.

Do not use DRIVE_REP_DIST when the actual position value sent in the cyclic data from the drive or encoder counts up to the full 32 bit value. Check the drive manual and settings.

Value

The value at which the position actual value will reset to 0. DRIVE_REP_DIST works with the ENCODER value and is always in encoder counts.

Example

Example1

An absolute encoder on the motor has 23 bits per turn and the motor is connected to a turntable through a 10:1 gearbox. The drive has been set up to reset the position actual value at 10 turns of the motor.

BASE(4)

' One turn 2^23 = 8388608

' 10 turns = 83886080

DRIVE_REP_DIST = 83886080



' Now set up units to scale the axis in degrees of the

' turntable.

UNITS = 83886080 / 360



' Set the axis repeat distance and mode to match the

' position roll-over. (these parameters affect MPOS)

REP_DIST = 360

REP_OPTION = 1
