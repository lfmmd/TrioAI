GTAE

Type

Robotics Parameter

Syntax

GTAE(index) = x, y, z, u, v, w, name, ptype, rnum, rconf, revs, of, tool, rf, userdata

Description

GTAE is the extended version of GTA (Global Targets Array) and GTAJ (Global Targets Array – Joint) which contains an extended array of globally available positions for use in Robotics applications.

GTAE, like GTA and GTAJ, are global to all programs and their values are retained. For local targets, see TARGETE.

A GTAE variable is an aggregate of multiple values, each value being stored in a named member as shown below.
KeywordDescription
x, y, z, u, v, w64-bit floating point number
id32-character string
activeBoolean
ptype8-bit unsigned integer
rnum8-bit unsigned integer
rconf8-bit unsigned integer
revs8-bit unsigned integer
of8-bit unsigned integer
tool8-bit unsigned integer
rf8-bit unsigned integer
userdata64-bit floating point number

Global targets may be activated or deactivated using the Boolean member “active”. Assigning ‘FALSE’ will erase the target (all values to 0 and empty name) and deactivate it (not visible in any program). When a target has been deactivated, it is like it had never been used, so deactivating a target can be done with any of GTA, GTAJ or GTAE and the result will be the same.

GTAE(10).active = FALSE

If it is required to reset all elements of the global target array, then the following command can be used.

GTAE(0, GTASIZE - 1).active = FALSE

A range of GTAJs can be requested or assigned by specifying the start and end index separated by comma.

GTAE(555, 559) = 90, -90, 0, 0, 0, 0

PRINT GTE(555, 559)

Global target members can be read from and written to using the ‘dot’ notation (“.”).

my_variable = GTAE(10).x

GTAE10).x = (GTAE(23).x + GTAE(24).x) / 2

If an inactive global target array element is assigned to using GTAE, then that element becomes a GTAE, i.e. one that can take any point type, and must be manipulated with GTAE otherwise a run-time error may be generated. The reason that an error may be generated is that if you know that the point type of a GTAE is 0, then it is possible to access it with GTA, and likewise, if you know that the point type of the GTAE is 1, then it is possible to access it with GTAJ. However, it is a good general rule to always use GTAE to access a GTAE, regardless of the point type it contains.

Global target array elements may be assigned to each other and to and from local target variables.

GTAE(10) = GTAE(11)

my_target = GTAE(10)

GTAE(12) = my_target

GTAE requires the RPS runtime FEC.

GTA, GTAJ, and GTAE are sharing the same allocated location of the memory. This means they cannot share indices. Maximum number of targets is controller dependent.

For example: 
GTA(1) = X1, Y1, Z1, U1, V1, W1, “pt1”

GTAE(2) = 154.694, -440.071, -27.495, 180, 0, 200, "test", 0, 0, 1, 0, 0, 0, 0, 69.420

In this example, the GTAE cannot have index 1 and the GTA cannot have index 2.

Parameters

NameDescription
x, y, z, u, v, w: FloatPositions (mm) and/or orientations (degrees) in Cartesian space, Joint space, or user-defined space.
(optional) “name”: StringUnique name for the target. Duplicated names generate a compile error.
ptype: IntegerPoint type.
0 = Cartesian, 1 = Joint, 2+ = User defined.
rnum: IntegerRobot number used for the target.
rconf: IntegerRobot configuration setup for the target.
revs: IntegerNumber of revolutions from the calibration point when target was taken.
of: IntegerObject frame used for the target.
tool: IntegerTool offset used for the target.
rf: IntegerRobot frame used for the target.
userdata: FloatAny useful numeric value.

Name is optional but if specified must be unique (not used by any other GTA elements of any type).

Name is truncated if more than 32 characters long.

GTAE cannot be declared using a name; instead, it must use an id number. If so, the run time error “ID not defined” will be thrown.

Examples

Example 1

Define an extended global target named “p1”.

GTAE(1011) = 100, 0, 50, 180, 0, 180, "p1", 0, 0, 1, 10, 0, 0, 0, 69.420
