TARGETE

Type

Robotics Type

Syntax

DIM <var-list> AS TARGETE

<target-variable> = x, y, z, u, v, w, ptype, rnum, rconf, revs, of, tool, rf, userdata

<target-variable> = <target-variable>

Description

TARGETE is an extended version of TARGET datatype able to store additional data, and is used in robotics move commands, such as MOVEC, MOVEJ and MOVEL. TARGETE variables are for local use only and can be defined just as other variables in a program. For global use, use the global target array (see GTAE).

It is possible to have an array of local targets by defining the dimensions it in the declaration statement:

DIM targetArray AS TARGETE(12)

It is possible read from and write into the target’s elements using the ‘dot’ notation (“.”) as shown in the table below:

my_variable = my_target.x

my_target.y = 12.2

Assignment between targets, both local and global, is possible:

GTAE(20) = GTAE(14)

GTAE(10).x = GTAE(11).x

my_target_1 = GTAE(10)

GTAE(11) = my_target_2

TARGETE requires the RPS runtime FEC.

Parameters

NameDescription
x, y, z, u, v, w: FloatCartesian position (mm) and orientation (degrees), or joint positions (mm or degrees as required by the robot type).
ptype: IntegerPoint type.
Value: 0 = cartesian, 1 = joint.
rnum: IntegerRobot number used for the target.
rconf: IntegerRobot configuration setup for the target.
revs: IntegerLast axis revolution value from calibration point when target was taken.
of: IntegerObject frame used for the target.
tool: IntegerTool offset used for the target.
rf: IntegerRobot frame used for the target.
userdata: FloatAny useful numeric value can be stored here.

The naming convention x, y, z, u, v, w was chosen to reflect Cartesian coordinates, as this was originally their sole purpose. While joint targets expanded this capability (see TARGETJ), the TARGETE data-type extends it further to encompass Cartesian, joint and user-defined point types, while retaining the same naming convention for the positions.

Examples

Example 1

Define an array of 2 extended targets.

DIM targets AS TARGETE(2)

targets(0) = 104, -400, -29, 180, 0, -68, 0, 2, 0, 10, 3, 13, 23, 69.420

targets(1) = 114, -410, -29, 180, 0, -68, 0, 2, 0, 10, 3, 13, 23, 79.420
