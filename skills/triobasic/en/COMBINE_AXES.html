COMBINE_AXES

Type

Axis Command

Syntax

COMBINE_AXES(axis_a, axis_b)

Description

The COMBINE_AXES command takes the position inputs from 2 axes and performs a mathematical operation on them. The result is output as the DPOS of the axis on which the COMBINE_AXES is run.

The operation performed by COMBINE_AXES is set up in the COMBINE_CONTROL function. There are 4 modes;

  • Sum the axis position values
  • Difference the axis position values
  • Perform SQR ( axis_a2 + axis_b2 )
  • Average the axis position values

Either MPOS (actual positon) or DPOS (demand position) can be selected as the axis position source. Mode 2 requires the input values to be MPOS only.

DPOS = Fn[MPOS axis_a, MPOS axis_b]

The specified axis can be any axis, including a virtual axis, and does not have to physically exist in the system.

Parameter

axis_a:First axis input (MPOS or DPOS)
axis_b:Second axis input (MPOS or DPOS)

Examples

Example 1

Using COMBINE_AXES to evaluate the position on the hypotenuse of a pair of orthogonal axes.

axis_a = 2

axis_b = 3

ca_axis = 10

UNITS AXIS(axis_a) = 100

UNITS AXIS(axis_b) = 100

UNITS AXIS(ca_axis) = 100

COMBINE_AXES(<axis_a>, <axis_b>)

COMBINE_CONTROL(2, 0, 0) 'Mode 2, a & b are combined to generate DPOS using sqrt(sqr(MPOS axis a) + sqr(MPOS axis b))

DPOS of axis 10 is the Pythagoras result of the positions on axes 2 and 3.

In Mode 2, the offsets are added to the MPOS values of the axes. Use negative values to subtract an offset.

COMBINE_CONTROL(2, 500, -100)
