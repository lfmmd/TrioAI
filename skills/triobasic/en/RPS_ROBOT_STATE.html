ROBOT_STATE

Type

Robotics Parameter (Read Only)

Syntax

ROBOT_STATE(group)

Description

Allows the reading of the RPS 2.0 kernel’s state.

ROBOT_STATE requires the RPS runtime FEC

RPS 2.0 kernel must be activated using the RPS_KERNEL_V2 keyword to enable its use.

Value

StateNameDescription
0InitialiseSet RPS kernel internal data to initial values
1InitialisedInitial values set
2ConfigureWaiting for the kinematic group and its I/O to be set
3ConfiguredKinematic group fully specified
4ValidateCheck the kinematic group for errors
5ValidatedKinematic group is free from errors
6Activate kinematic groupAsk the firmware to start frame transformations
7Kinematic group activatedFrame transformation have started
8Awaiting operating modeKey switch is being sampled
9Operating mode setSampling is complete and the operating profile set
10OperationalThe operating profile is being executed
11Error arbiterDecide the nature of the error and set ERO
12Minor errorJump to state ‘Reset errors’
13Major errorNeed to do an emergency stop
14Emergency stopAbort motion and robot programs
15Monitor stopWait for motion to stop
16StoppedMotion has stopped
17Reset errorsWait for REI to be pulsed high and low (falling edge trigger
18Errors resetClear ERO and jump to state ‘Configure’
19ShutdownSets outputs low and jump to state ‘Initialise’

Examples

Example 1

Detect if robot 0 motion has stopped.

IF ROBOT_STATE(0) = 16 THEN

   'Robot motion has stopped. Action here.

ENDIF
