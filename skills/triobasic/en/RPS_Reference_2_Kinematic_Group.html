Kinematic Groups

KINEMATIC_GROUP command activates RPS and links the required data to the specified robot. This command must be executed to have RPS set.

Although 8 kinematics can be initialised on a single controller, it may not be possible to process all 8 within a given SERVO_PERIOD. The total number that can be run depends on many factors such as, type of kinematic is selected, drive connection method, if OBJECT_FRAME, ROBOT_FRAME, TOOL_OFFSET or COLLISION_OBJECT is enabled and additional factory communications.

The number of axes in the group must match the number of axes used by the KINEMATICS. The axes must also be ascending order though they do not have to be contiguous.

Syntax

To display information about a kinematic group.
KINEMATIC_GROUP(index)
index: INTEGERGroup index. Up to 8 robots can be set in the same controller. Valid index: 0..7

To perform an action of some sort on a kinematic group (currently the only action is disabling).
KINEMATIC_GROUP(index, function)
index: INTEGERGroup index. Up to 8 robots can be set in the same controller. Valid index: 0..7
function: INTEGER-1 = disable the kinematic group specified by index. All the axes in the group will be released.

When disabling a kinematic group, all axes of the group must be IDLE. If not, runtime error “Operation cannot be performed until IDLE” will be thrown.

To define a kinematic group.
KINEMATIC_GROUP(index, table offset, kinematic number, axis0, ..., axisn)
index: INTEGERGroup index. Up to 8 robots can be set in the same controller. Valid index: 0..7
table offset: INTEGERLocation in TABLE where the robot parameters are stored.
kinematic number: INTEGERRobot type as defined by FRAME. See TrioBasic command FRAME for more information.
axis0…axisn: INTEGERAxes belonging to the robot in ascending order

The number of axes in KINEMATIC_GROUP must match the number of axes used by the kinematic number selected. The axes must be in ascending order but not necessarily contiguous.

When displaying information on a kinematic group, the output takes the form:

Group index [table_offset] [kinematic number]: axes TO={index [name] : TOOL_OFFSET parameters}  RF={index [name] : ROBOT_FRAME parameters}  OB={index [name] : OBJECT_FRAME parameters}  CO{active collision objects}

Where the elements are as follows:
TOActive Tool Offset (specified by index, name, and parameters that follow)
RFActive Robot Frame (specified by index, name, and parameters that follow)
OBActive Object Frame (specified by index, name, and parameters that follow)
COTotal active collision objects in binary index wise. For example, if there are 4 collision objects (index 0, 1, 2, 3) and only collision object index 0 and 3 is active, CO will be of value 21 + 23 = 2 + 8, or {10}

Examples

Example 1

Check if RPS is set by displaying kinematic group 0.

Terminal

>>kinematic_group(0)

0 [0] [19] : 10, 11, 12, 13, 14, 15 TO={0[TO_default]:0.00000, 0.00000, 0.00000, 0.00000, 0.00000, 0.00000} RF={0[RF_default]:0.00000, 0.00000, 0.00000, 0.00000, 0.00000, 0.00000} OF={0[OF_default]:0.00000, 0.00000, 0.00000, 0.00000, 0.00000, 0.00000} CO{3}

Example 2

Disable kinematic group 0.

Terminal

>> kinematic_group(0, -1)
