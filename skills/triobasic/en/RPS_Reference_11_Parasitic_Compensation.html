Parasitic Compensation

Some robots have a mechanical link between 2 axes such that one axis moves when the other linked axis moves, causing parasitic motion. To compensate this type of motion, COMP_RATIO can be used.

This compensation should be done in ROBOT_DEFINITIONS file, using Motion Perfect robot setup tool.

Syntax

COMP_RATIO(function)
COMP_RATIO(driving_axis, ratio_numerator, ratio_denominator)
function-1 = delete any existing compensation applied to base axis
driving_axisAxis to link to.
ratio_numeratorNumber of edges the base axis is required to move per increment of the driving axis. The value can be either positive or negative. It is always specified as an encoder edge ratio.
ratio_denominatorNumber of edges the base axis is required to move per increment of the driving axis. The value can be either positive or negative. It is always specified as an encoder edge ratio.

Description

This function links the DEMAND_EDGES of base axis to the measured movements of the driving axes to compensate mechanical parasitic motion. It will add the compensation pulses to demand edges. When this function is called, an initial compensation amount will be added to AXIS_DPOS to account for previous compensation. No motion will occur during this.

Examples

Example 1

master_axis = 3

slave_axis = 2

compensation_value = -18

num = UNITS AXIS(slave_axis) * compensation_value

denom = UNITS AXIS(master_axis) * 360



COMP_RATIO(master_axis, num, denom) AXIS(slave_axis)
