FLASH_LOG

Type

System Function

Syntax

FLASH_LOG([start[, finish]])

Description

Records up to 8192 of the most recent controller system events. Most controllers support 4096 messages but some record 2048 or 8192 events.

Each event is recorded with a timestamp of the current RTC value and the time since power-up of the controller.

Each event is also tagged with a checksum for verification.

When executing the command, either in the terminal or a Trio BASIC program, it will print the recorded event messages to the terminal.

Note: Before displaying a log entry the checksum for the log is validated, if it is invalid then the message '<INVALID LOG ENTRY>' will be displayed instead.

Parameters

start:Displays all flash log entries starting from line number 'start'.
-1Special case, erases log from flash memory (second parameter is not needed).
-2Special case, will display the most recent ‘n’ entries where ‘n’ is the second parameter.
finish:Displays all flash log entries starting from line number 'start' until line number 'finish'.
