NODE_AIO

Type

System Array (MC_CONFIG)

Syntax

NODE_AIO(slot, node_address) = offset_value

Description

This 2D array can be used to set the start address of any EtherCAT node analogue I/O channels. This can be used to define a user specific I/O map to fix I/O points from different sources in place.

The array is 2-dimensional, the first dimension is the master slot identifier, the second dimension is the position of the node within that master network.

Value

slot:The slot number of the EtherCAT port. (0 on MC4N, MC6N and Flex-6 Nano)
node_address:Address determined by the position of the node on the EtherCAT network. The first node is address 0.
offset_value0EtherCAT I/O allocated automatically (default)
>= 8EtherCAT analogue I/O for the defined Node starts at this I/O point address

To allocate I/O offsets based on the EtherCAT configured address, use NW_NODE_AIO

Example

A system with MC464, an EtherCAT module (slot 0) and a 16 inputs EtherCAT analogue I/O module at node 5 will normally have the following I/O assignment:

NODE_AIO(0, 5) = 0
0-15EtherCAT analogue inputs
32-33Built-in analogue inputs

If the NODE_IO is changed, then the EtherCAT analogue I/O can be moved to higher values.

NODE_AIO(0, 5) = 40
0-1Built in inputs (copy of 32-33)
32-33Built-in inputs
40-55P876 EtherCAT analogue inputs
