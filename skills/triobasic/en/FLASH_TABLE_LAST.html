FLASH_TABLE_LAST

Type

System Parameter (MC_CONFIG)

Syntax

FLASH_TABLE_LAST = value

Description

FLASH_VR_FIRST, FLASH_VR_LAST, FLASH_TABLE_FIRST and FLASH_TABLE_LAST are system parameters that allow user configuration of the range of VR/TABLE locations that are preserved in flash memory. VR and TABLE locations outside the defined range will not be preserved when the power is removed.

Requires firmware V2.0313 and later.

Parameters

Value: the TABLE index where the storing of global Table values in flash will end.

Note 1: MC403, MC405, MC4N do not autosave TABLE values. FLASH_DATA can be used to configure the Motion Coordinator for TABLE instead of VR. In that case FLASH_TABLE_LAST has the same max value as FLASH_VR_LAST.

Example

In a MC508, set up auto-save for the first 32000 indexes (range of TABLE values from 0 to 31999) to Flash memory. TABLE(32000) to TABLE(511999) will therefore be volatile and therefore set to 0 on next power up.

FLASH_TABLE_FIRST = 0

FLASH_TABLE_LAST = 31999
