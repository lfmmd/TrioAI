ETHERNETIP

Type

System Function

Syntax

ETHERNETIP(function, slot, ...)

Description

Controls some of the Ethernet/IP functionality of the controller. The function argument specifies the specific function to be performed, and the other arguments depend on the function.

Parameters

FunctionDescription
$02Get connection status
$10Get an error log entry
$11Get the number of entries in the error log
$12Clear the error log
$13Set the printed message level (0-3)
$14Get the printed message level
$31Set a CIP tag
$32Get a CIP tag
$33Reset CIP tags
$34Display CIP tags
slotDescription
-1The internal Ethernet device.

Functions

Get connection status

The connection status is the number of active connections to the controller on the given connection id. The controller can have up to 3 simulteneous TCP connections, and up to 10 UDP connections (Depending on controller load).

ETHERNETIP($02, -1, connection_id, vr)
ParameterDescription
connection_idThe connection to read0-2: TCP connection3: UDP connection
vrThe VR to store the connection status, or -1 to print the status to the current output device

Example

DEFCONST ethercat_slot -1

DIM i,s AS INTEGER



' Count the number of active TCP connections

s=0

FOR i=0 TO 2

  ' Get the status of this connection.

  ETHERNETIP(2,ethercat_slot,i,0)

  IF VR(0)<>0 THEN

    s=s+1

  ENDIF

NEXT i

PRINT "Active TCP connections: ";s[0]



ETHERNETIP(2,ethercat_slot,3,0)

PRINT "Active UDP connections: ";VR(0)[0]

This program will generate output similar to the following, depending on the number of current connections.

Active TCP connections: 0

Active UDP connections: 1

Error log

The controller stores a log of Ethernet/IP errors that have occurred. This can be useful for troubleshooting issues with Ethernet/IP communications. The log is stored in a first in first out (FIFO) manner, and can store up to 256 entries. When the log is full, new entries will overwrite the oldest entries. The log is volatile and Initialised at restart.

Error log entry 0 is the newest entry, and entry n-1 is the oldest entry, where n is the number of entries in the log (up to 256).

Get an error log entry

Each error log entry reflects an Ethernet/IP error that was generated by the controller. Ethernet/IP errors have values greater than 0. Some Ethernet/IP errors have additional values as well. To support this, each error log entry is made up of a sequence of 16 bit values. The first value is the error code, and the following values depend on the additional values. To extract the full error entry the different 16 bit values must be extracted until we get to a value of 0, which indicates the end of the entry.

Example

DEFCONST ethercat_slot -1

DIM i AS INTEGER



' Get the number of log entries into VR(0)

ETHERNETIP($11,ethercat_slot,0)

PRINT "Error Count: ";VR(0)[0]



' If we have log entries then print them

IF VR(0)>0 THEN

    FOR i=0 TO VR(0)-1

        ETHERNETIP($10,ethercat_slot,i,1)

        ' Print the first value of the entry, which is the error code



        PRINT "->Error: 0x";HEX(VR(1) AND $FFFF)



        ' Remove the first value from the entry

        VR(1)=VR(1)>>16



        ' While the entry has more values, print them as additional information for the error

        WHILE VR(1)<>0

            ' Print the next value of the entry, which is additional information for the error

            PRINT "-->Additional: 0x";HEX(VR(1) AND $FFFF)



            ' Remove the value we just printed from the entry

            VR(1)=VR(1)>>16

        WEND

    NEXT i

ENDIF

Printed error messages

Under certain circumstances, the controller will print error messages related to Ethernet/IP. The messages are categorized as ERROR, WARNING, INFORMATION, DEBUG, and TRACE. The user can select the message level to print. By default, the message level is set to ERROR and WARNING.

Set the printed message level

ETHERNETIP($13, -1, level)
ParameterDescription
levelA bitmap that represents the message level to print1: ERROR2: WARNING4: INFORMATION8: DEBUG16: TRACE

Get the printed message level

ETHERNETIP($14, -1, vr)
ParameterDescription
vrThe VR to store the current message level

Print a CIP tag. If the tag does not exist then nothing is printed.

Reset CIP tags

Clear the CIP tag table.

ETHERNETIP($32, -1)

Display CIP tags

Print all CIP tags.

ETHERNETIP($33, -1)
