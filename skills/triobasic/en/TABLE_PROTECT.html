TABLE_PROTECT

Type

System Command

Syntax

TABLE_PROTECT(first[, last/lock[, lock]])

Description

Sets a range of TABLE values to be write protected (i.e. read only). Can be used to define parts of the TABLE where setup data has been stored that must not be changed by any programs, data connections or direct user entry.

TABLE_PROTECT is not available for Euro404/408, MC403, MC403-X, MC403-Z, MC404-Z, MC405, MC405-Y and P157.

Parameters

first:First TABLE address to be locked
last:Last TABLE address to be locked
lock:Lock/unlock TABLE values as read-only
0Read and write is allowed (default)
1Read-only

Example 1

Set TABLE value number 10 as a read- only value in the terminal:

>>TABLE(10, 222) 

>>?TABLE(10)

222.0000

>>TABLE_PROTECT(10, 1)

>>?TABLE(10)         

222.0000

>>TABLE(10, 111)      

%[COMMAND 224] TABLE location write protected  ((10))

Example 2

Set TABLE values from 10 to 20 as read- only values, then check that they are in the terminal:

'Program sets the protected area

TABLE_PROTECT(10, 20, 1)

In Motion Perfect terminal 0, check that it is read-only:

>>?TABLE(10)         

222.0000

>>?TABLE(13)         

125.0000



>>TABLE(13, 111)      

%[COMMAND 224] TABLE location write protected  ((13))

>>TABLE(16, 111)      

%[COMMAND 224] TABLE location write protected  ((16))
