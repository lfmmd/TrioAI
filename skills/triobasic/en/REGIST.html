REGIST

Type

Axis Command

Syntax

REGIST(mode[, parameters])

Description

The REGIST command initiates a capture of an axis position when it sees a registration input or the Z mark on the encoder. Once a registration event is captured MARK is set and the position and speed at the event can be read back.

See the Hardware Chapter of the manual to understand which registration mode your hardware supports.

Filtering can be applied to the input as well as defining a window of where to capture.

Hardware registration captures the encoder count against the registration input in hardware.

Time based registration captures the time of the registration event and interpolates the position values being sent back from the drive against it.

Although all modes are available for backwards compatibility it is recommended to use modes 20-22 for new applications. Other modes have been provided for compatibility with older products.

The REGIST command must be re-issued for each position capture.

The captured registration position may be outside REP_DIST. You should always check the captured registration position to ensure it is within your application’s usable range.

Parameters

mode:0Cancel all Registration operations on the axis
1 to 4Single channel hardware registration
5Reserved
6 to 13Dual channel hardware registration
14 to 17Single channel hardware registration
20Single channel hardware registration including Touch Probe
21Single channel time interpolated registration
228 channel hardware registration (Flexaxis module only)
23Sets 2.4 µs minimum pulse width
24Sets 0.15 µs minimum pulse width (default)
32 to 39Rising edge on time-based registration (use mode 21)
64 to 71Falling edge on time-based registration (use mode 21)

Mode = 0

Syntax

REGIST(0)

Description

Cancels the previously set REGIST commands for the axis. REGIST(0) cancels all channels. If separate control for channels A and B are required, then use REGIST_CONTROL.

Mode = 1 to 4

Syntax

REGIST(mode)

Where mode = 1 to 4

Description

It is recommended that you use mode 20 for all new applications

Modes 1 to 4 work with the first channel or Z mark of hardware-based registration.

You can add 256 or 768 to enable windowing.

This mode works with MARK, REG_POS and REGIST_SPEED

Parameters

mode:1Z Mark rising into REG_POS
2Z Mark falling into REG_POS
3RA Input rising into REG_POS
4RA Input falling into REG_POS
mode + 256Position must be inside OPEN_WIN to CLOSE_WIN
mode + 768Position must be outside OPEN_WIN to CLOSE_WIN

Example

A disc used in a laser printing process requires registration to the Z marker before printing can start. This routine locates to the Z marker, then sets that as the zero position.

BASE(0)

REGIST(1)           'Initialise to Z mark

FORWARD             'start movement

WAIT UNTIL MARK

CANCEL              'stops movement after Z mark

WAIT IDLE

MOVEABS (REG_POS)   'relocate to Z mark

WAIT IDLE

DEFPOS(0)           'set zero position

Mode = 6 to 13

Syntax

REGIST(mode)

Where mode = 6 to 13

Description

It is recommended that you use mode 20 for all new applications

Modes 6 to 13 work with hardware-based registration but enable you to arm 2 registration registers at once.

You can add 256 or 768 to enable windowing.

The first channel will use MARK, REG_POS and REGIST_SPEED and the second will use MARKB, REG_POSB and REGIST_SPEEDB

Parameters

mode:6RA Input rising into REG_POS & Z Mark rising into REG_POSB
7RA Input rising into REG_POS & Z Mark falling into REG_POSB
8RA Input falling into REG_POS & Z Mark rising into REG_POSB
9RA Input falling into REG_POS & Z Mark falling into REG_POSB
10RA Input rising into REG_POS & RB Input rising into REG_POSB
11RA Input rising into REG_POS & RB Input falling into REG_POSB
12RA Input falling into REG_POS & RB Input rising into REG_POSB
13RA Input falling into REG_POS & RB Input falling into REG_POSB
mode + 256Position must be inside OPEN_WIN to CLOSE_WIN
mode + 768Position must be outside OPEN_WIN to CLOSE_WIN

Example 1

A machine adds glue to the top of a box by switching output 8. It must detect the rising edge (appearance) of and the falling edge (end) of a box. Additionally, it is required that the MPOS be reset to zero on the detection of the Z position.

reg = 6 'Select registration mode 6 (rising edge R, rising edge Z)

REGIST(reg)

FORWARD

WHILE IN(2) = OFF

  IF MARKB THEN  'On a Z mark MPOS is reset to zero

    OFFPOS = -REG_POSB

    REGIST(reg)

  ELSEIF MARK THEN  'On R input output 8 is toggled

    IF reg = 6 THEN

      'Select registration mode 8 (falling edge R, rising edge Z)

      reg = 8

      OP(8, ON)

    ELSE

      reg = 6

      OP(8, OFF)

    ENDIF

    REGIST(reg)

  ENDIF

WEND

CANCEL

Mode = 14 to 17

Syntax

REGIST(mode)

Where mode = 14 to 17

Description

It is recommended that you use mode 20 for all new applications

Modes 14 to 17 work with the second channel or Z mark of hardware-based registration.

You can add 256 or 768 to enable windowing.

This mode works with MARKB, REG_POSB and REGIST_SPEEDB

Parameters

mode:14ZB Mark rising into REG_POSB
15ZB Mark falling into REG_POSB
16RB Input rising into REG_POSB
17RB Input falling into REG_POSB
mode + 256Position must be inside OPEN_WIN to CLOSE_WIN
mode + 768Position must be outside OPEN_WIN to CLOSE_WIN

Example 2

It is required to detect if a component is placed on a flighted belt, so windowing is used to avoid sensing the flights. The flights are at a pitch of 120 mm and the component will be found between 30 and 90 mm. If a component is found, then an actuator is fired to push it off the belt.

REP_DIST = 120         'sets repeat distance to pitch of belt flights

REP_OPTION = ON 

OPEN_WIN = 30          'sets window open position

CLOSE_WIN = 90         'sets window close position

REGIST(17 + 256)       'RB input registration with windowing

FORWARD                'start the belt

box_seen = 0

REPEAT

  WAIT UNTIL MPOS < 60 'wait for centre point between flights

  WAIT UNTIL MPOS > 60 'so that actuator is fired between flights

  IF box_seen = 1 THEN 'was a box seen on the previous cycle?

    OP(8, ON)          'fire actuator

    WA(100)

    OP(8, OFF)         'retract actuator

    box_seen = 0

  ENDIF

  IF MARKB THEN box_seen = 1 'set "box seen" flag

  REGIST(17 + 256)

UNTIL IN(2) = OFF

CANCEL                 'stop the belt

WAIT IDLE

Mode = 20

Syntax

REGIST(20, channel, source, edge, window[, quantity, table_start])

Description

Mode 20 is used to set the hardware registration inputs A or B. Alternatively A or B can be replaced with the Z mark. A and B are completely independent.

The FlexAxis and all MC⅘/6 and Euro404/408 Motion Coordinators (except the MC664) use the REG_INPUTS command to select the actual input used for channel A and channel B.

This mode can be used instead of REGIST modes 1 to 4 and 14 to 17

If the optional parameters quantity and table_start are used, then a set of registration positions can be stored in the table. REG_POS and REG_POSB will still store the latest registration position. When capturing positions to the TABLE, the MARK parameter holds the count value to indicate how many registration events have been captured.

Parameters

Analog axis

channel:0Selects channel A.
1Selects channel B.
0 to 511Digital input selection when source set to 4.
16 to 32Bits 16 to 32 to select digital input when Digital Input Window enabled (window option 3 and 4)
source:0Selects the first 24 V input.
1Selects the Z mark.
2Selects the second 24 V input. (if available)
4Selects any digital input as source, used on any axis.
edge:0Rising edge
1Falling edge
window:0No windowing
1Position must be inside OPEN_WIN to CLOSE_WIN.
2Position must be outside OPEN_WIN to CLOSE_WIN.
3Enable window with 24 V/Virtual input (Active High)
4Enable window with 24 V/Virtual input (Active Low)
5Position must be inside OPEN_WINB to CLOSE_WINB.
6Position must be outside OPEN_WINB to CLOSE_WINB.
quantity1 to TSIZEQuantity of registration captures to store in the TABLE.
table_start0 to TSIZE - 1Start position in the TABLE for the registration positions.

If channel = 0 then MARK, REG_POS and REGIST_SPEED are used
If channel = 1 then MARKB, REG_POSB and REGIST_SPEEDB are used

If source = 4 then MARK, REG_POS and REGIST_SPEED are used, but only values at the nearest servo period tick are captured. (not a true hardware registration)

EtherCAT Touch Probe axis

channel:0Selects channel A; Touch Probe 1 or Z
1Selects channel B; Touch Probe 2 or Z
source:0Selects the Touch Probe input; 1 if channel A, 2 if channel B.
1Selects the Z mark.
edge:0Rising edge
1Falling edge
window:0No windowing
1Position must be inside OPEN_WIN to CLOSE_WIN.
2Position must be outside OPEN_WIN to CLOSE_WIN.
3Enable window with 24 V/Virtual input (Active High)
4Enable window with 24 V/Virtual input (Active Low)
5Position must be inside OPEN_WINB to CLOSE_WINB.
6Position must be outside OPEN_WINB to CLOSE_WINB.
quantity1 to TSIZEQuantity of registration captures to store in the TABLE.
table_start0 to TSIZE - 1Start position in the TABLE for the registration positions.

The Z can be selected from either channel A or Channel B. The ability to detect the Z will require the motor to have an incremental encoder and a Z channel on the motor feedback device. Absolute axes do not generally have a Z capability.

The calculation of Touch Probe position requires the motor direction to be positive. It is not possible to reverse the direction using ENCODER_RATIO. Use the direction reversal mode in the Drive.

Using OPEN_WIN and CLOSE_WIN with Touch Probe needs several EtherCAT cycles per position check. High speed axes should use REGIST(21) if available.

Example 3

Configure the windowing which will be used on channel B and then arm both channel B and the Z mark. Look for falling edge on channel B.

OPEN_WIN = 200

CLOSE_WIN = 400

REG_INPUTS = $10 'Input 0 for first reg input, input 1 for second

REGIST(20, 0, 1, 0, 0) 'Arm channel A on Z mark, no window

REGIST(20, 1, 0, 1, 2) 'Arm channel B on 1st input, use exclusive window

Example 4

Configure the EtherCAT axis to arm the Touch Probe 1 input on channel A rising edge and Touch Probe 2 on channel B falling edge. Requires Touch Probe CoE objects $60BA and $60BD to be in the PDO.

REGIST(20, 0, 0, 0) 'Arm channel A on rising edge

REGIST(20, 1, 0, 1) 'Arm channel B on falling edge

REPEAT

  IF MARK THEN

    VR(201) = REG_POS

  ENDIF

  IF MARKB THEN

    VR(202) = REG_POSB

  ENDIF

  WA(10) 'Check every 10 ms

UNTIL MARK AND MARKB

Mode = 21

Syntax

REGIST(21, channel, source, edge, window[, quantity, table_start])

Description

REGIST mode 21 is used to arm the time interpolated registration.

This can be used instead of REGIST modes 32 to 39 and 64 to 71.

This mode operates with the hardware registration capture circuit built into Motion Coordinators that have networked drive ports such as RTEX and EtherCAT. Parameters R_MARK(channel) , R_REGPOS(channel) and R_REGISTSPEED(channel) are used with mode 21. The Channel number also indicates the 24 V input which is to be used for the registration sensor, unless this is altered by the setting in Source.

If the optional parameters quantity and table_start are used, then a set of registration positions can be stored in the table. R_ REGPOS will still store the latest registration position.

Virtual axes may use REGIST 21 if the axis is associated with the SLOT_NUMBER of the network module or built-in hardware. See Example 3 in SLOT_NUMBER.

Parameters

channel:0 to 7The registration channel to be used (range 0 to 7) which by default is also the Input number on the Motion Coordinator
16 to 32Bits 16 to 32 to select digital input for gating the signal when Digital Input Window is enabled (window option 3 and 4)
source:0Use the default input set by Channel
256 + input_numberWhen bit 8 is set, assigns a different physical input to the logical channel (Over-rides the default input)
edge:0rising edge
1falling edge
window:0no windowing
1position must be inside OPEN_WIN to CLOSE_WIN
2position must be outside OPEN_WIN to CLOSE_WIN
3enable window with 24 V/Virtual input (Active High)
4enable window with 24 V/Virtual input (Active Low)
quantity1 to TSIZEQuantity of registration captures to store in the TABLE
table_start0 to TSIZE - 1Start position in the TABLE for the registration positions

Example 5

Set up registration on falling edge for EtherCAT or RTEX axis 2 using input 7.

BASE(2)

REGIST(21, 7, 0, 1, 0)

WAIT UNTIL R_MARK(7)

Example 6

Set up registration on falling edge for EtherCAT or RTEX axis 2 using input 1, and input 3 to open enable the window.

BASE(2)

input_channel = 1

gate_input = 3

'The window control input (gate) number is in bits 16 to 31

REGIST(21, input_channel + (gate_input << 16), 0, 1, 3)

WAIT UNTIL R_MARK(input_channel)

Example 7

Set up registration on rising edge for virtual axis 15 using input 1.

BASE(15)

SLOT_NUMBER = 0 'Slot number of the RTEX network on MC4N

REGIST(21, 1, 0, 0, 0) 'Input 1, rising edge

WAIT UNTIL R_MARK(1)

captured_posiiton = R_REGPOS(1)

Using SLOT_NUMBER in this way is only valid for virtual axes. Do not change the SLOT_NUMBER of a real hardware axis.

Example 8

Set up two registration events for two different axes using the same physical input (input 1) on and EtherCAT system using the MC6N-ECAT.

BASE(4)

'Set up regist channel 1 on default input

REGIST(21, 1, 0, 0, 0)



BASE(5)

'Set up regist channel 2 with Source bit 8 set to indicate that

'the source number is the required input

REGIST(21, 2, 256 + 1, 0, 0) ' Source value is $101 in Hex



WAIT UNTIL R_MARK(1) AND R_MARK(2)

Mode = 22

Syntax

REGIST(22, channel, source, edge, window[, quantity, table_start])

Description

This mode allows up to 8 hardware registration inputs to be assigned to one axis on the P874/P879 Flexaxis module.

If this mode is used then all 8 inputs are assigned to REGIST(22) for axis 0. You cannot mix REGIST(22) and REGIST(20) on the same axis. The setting for axis 0 does not affect the other axes.

This mode operates with the parameters R_MARK(channel) , R_REGPOS(channel) and R_REGISTSPEED(channel).

To use this mode REG_INPUTS must be set to $10 before you call the REGIST command.

If the optional parameters quantity and table_start are used, then a set of registration positions can be stored in the table. R_ REGPOS will still store the latest registration position.

Parameters

channel:0 to 7This is the registration channel to be used (range 0 to 7)
16 to 32Bits 16 to 32 to select digital input when Digital Input Window enabled (window option 3 and 4)
source:0Selects the 24 V registration input.
1Selects the Z mark.
edge:0Rising edge
1falling edge
window:0no windowing
1position must be inside OPEN_WIN to CLOSE_WIN
2position must be outside OPEN_WIN to CLOSE_WIN
3Enable window with 24 V/Virtual input (Active High)
4Enable window with 24 V/Virtual input (Active Low)
quantity1 to TSIZEQuantity of registration captures to store in the TABLE
table_start0 to TSIZE - 1Start position in the TABLE for the registration positions

Mode = 23

Syntax

REGIST(23)

Description

This mode assigns a 2.4 µs minimum pulse width to the axis. This affects any REGIST mode that is used.

The default value is 0.15 µs.

Mode = 24

Syntax

REGIST(24)

Description

This mode assigns a 0.15 µs minimum pulse width to the axis. This affects any REGIST mode that is used.

This is the default value.
