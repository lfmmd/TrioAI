MOVECONTOUR

Type

Axis Command

Syntax

[value = ]MOVECONTOUR(table_pointer, axes, npoints, options, speed_option, output_table_area, short_len, corner_acc)

Description

The MOVECONTOUR command allows a sequence of 5 axis movements to be loaded via a set of TABLE values. A single move is loaded which interpolates the speed and the positions of the 5 axes simultaneously to allow for “contouring control”.

Since the export of software for greater than 4 axes of contouring control is controlled by international treaties, this 5 axis MOVECONTOUR functionality is not built into Trio motion controllers. However, where an export licence is available, the software (product code P980) may be used to purchase 5 axis contouring control firmware.

The 4-axis version can be used without the licence.

MOVE_COUNT is incremented on every move loaded. This parameter may be set by a user program.

The return value indicates various types of error and the number of vectors processed (see table below).

Parameters

value:Bit 0The move was not loaded
Bit 1XYZ angle exceeds STOP_ANGLE
Bit 2Increment exceeds short_len
Bit 3U exceeds STOP_ANGLE
Bit 4V exceeds STOP_ANGLE
Bit 5W exceeds STOP_ANGLE
Bit 6XYZ angle exceeds RAISE_ANGLE
Bit 7Increment is zero
Bits 8 to 31The number of vectors processed
table_pointer:Location of the absolute points in TABLE memory
axes:Number of axes 2 to 5.
npoints:The number of points, each point requires 2-6 table values. 6 are only used when a speed profile along a 5-axis path is also embedded in the TABLE.
options:Bits 0 to 4Reserved
Bit 40 = Process the move and load it
1 = Process the move but without loading it
Bit 5Embed a speed profile together with axis position points in input TABLE data
Bit 6Take initial direction of XYZ from END_DIR_LAST axis parameters (requires AXIS_MODE.2 = ON)
Bit 7UVW in degrees (otherwise radians)
Bits 8 to 13Specifies an offset between data points other than “Axes”. Note that this is ESSENTIAL if Bit 5 is set
Bit 14Apply ACCEL and DECEL limits to the embedded speed profile
Bit 15Calculate 2D XY direction for plotter
Bit 16Calculate point radii and required speed at point (set automatically if corner_acc > 0)
Bit 17Slow down, don’t return from MOVECONTOUR when STOP_ANGLE exceeded
Bit 18Set initial values for (Bit 5) speed profile from SPEED
Bits 19 to 201 = B-spline interpolation
2 = C-spline interpolation
3 = Cubic spline
Bit 21Copy interpolation mode
Bit 22GCODE N field is copied into output table in lieu of W axis position
Bit 23Return on angle change exceeding RAISE_ANGLE
Bit 24Set CORNER_STATE
Bit 25Expand number of output points when length > short_len
Bit 30Copy MOVECONTOUR/MOVEDISPENSE setting
speed_option:Set to 1 to embed an initial speed for the start of the path in the data table
output_table_area:Output of data to TABLE at this area. Controller will write 15 x Npoints of output data
short_len:Optional parameter - Moves greater than this length in the TABLE entries will cause the MOVECONTOUR to finish. They can then normally be processed with MOVEABS.
corner_acc:Optional parameter – If specified, the speed entries in the TABLE can be reduced (not increased) depending on the effective radius between segments

The number of points that may be interpolated is limited by the size of the TABLE area available in the controller and the pre-processing time required for the move. The maximum size of MOVECONTOUR that is feasible given a 512000 TABLE location controller type would be approximately 10,000 lines of G-Code.

Embedded speed profile

The MOVECONTOUR function can optionally have a speed value attached to each target position. To apply this function the speed is entered in the TABLE after each position set. The Options bit 5 must then be set to tell the controller to work in this way.

The data points will need to be spaced further apart to allow for this additional data. The Options bits 8 to 13 are set to specify this quantity.

An example MOVECONTOUR command would be:

stride = 10 

MOVECONTOUR(1000, 5, 361, 32 + (stride * 256) + $4000, 0, 20000)

Note that the MOVECONTOUR sets bit 5 and bit 14 of the “options” word. Bit 5 sets to use the embedded speed values. Bit 14 makes the MOVECONTOUR adjust the speed values to allow for ACCEL and DECEL. Without bit 14 being set the programmer must profile the requested speed values. With bit 14 set the axis ACCEL and DECEL rates will be applied as maximum limit values. Using a small amount of SRAMP the profile can be further smoothed.
