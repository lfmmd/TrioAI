GTA

Type

Robotics Parameter

Syntax

GTA(index) = x, y, z, u, v, w, name

Description

GTA is the Global Targets Array which contains an array of globally available TARGET objects for use in Robotics applications.

GTA variables, like GTAJ and GTAE, are global to all programs and their values are stored in controller’s flash and, therefore, retained after power cycle. For local targets, see TARGET.

A GTA variable is an aggregate of multiple values, each value being stored in a named member as shown below.
KeywordDescription
x, y, z, u, v, w64-bit floating point number
id32-character string
activeBoolean

Global targets may be activated or deactivated using the Boolean member “active”. Assigning ‘FALSE’ will erase the target (all values to 0 and empty name) and deactivate it (not visible in any program). When a target has been deactivated, it is like it had never been used, so deactivating a target can be done with any of GTA, GTAJ or GTAE and the result will be the same.

GTA(10).active = FALSE

If it is required to reset all elements of the global target array, then the following command can be used.

GTA(0, GTASIZE - 1).active = FALSE

A range of GTAs can be requested or assigned by specifying the start and end index separated by comma.

GTA(10, 13) = 250, 0, 150, 180, 0, 180

PRINT GTA(10, 13)

Global target members can be read from and written to using the ‘dot’ notation (“.”).

my_variable = GTA(10).x

GTA(10).x = (GTA(23).x + GTA(24).x) / 2

If an inactive global target array element is assigned to using GTA, then that element becomes a GTA, i.e. has a point type of 0, and must be manipulated with GTA otherwise a run-time error will be generated. It is also permissible to use GTAE for any point type, but it is a good general rule to know what the point type is for an element and to always use the appropriate access keyword.

Global target array elements may be assigned to each other and to and from local target variables.

GTA(10) = GTA(11)

my_target = GTA(10)

GTA(12) = my_target

GTA requires the RPS runtime FEC.

GTA, GTAJ, and GTAE are sharing the same allocated location of the memory. This means they cannot share indices. Maximum number of targets is controller dependent.

For example: 
GTA(1) = X1, Y1, Z1, U1, V1, W1, “pt1”

GTAJ(2) = t1, t2, t3, t4, t5, t6, “pt2”

In this example, GTAJ cannot have index 1 and GTA cannot have index 2.

Parameters

NameDescription
x, y, z, u, v, w: FloatPosition (mm) and orientation (degrees) in Cartesian space.
(optional) “name”: StringUnique name for the target. Duplicated names generate a compile error.

Name is optional but if specified must be unique (not used by any other GTA elements of any type).

Name is truncated if more than 32 characters long.

A GTA cannot be declared using a name; instead, it must use an id number. If so, the run time error “ID not defined” will be thrown.

Examples

Example 1

Define a global target named “Cartesian_point1”.

GTA(10) = 100, 0, 50, 180, 0, 180, "Cartesian_point1"
