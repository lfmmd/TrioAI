SETCOM

Type

Command

Syntax

SETCOM(baudrate, databits, stopbits, parity, port[, mode][, data][, timeout][, linetype])

SETCOM(port)

Description

With multiple parameters, allows the user to configure the serial port parameters and enable communication protocols.

By default, the controller sets the serial ports to 38400 baud, 8 data bits, 1 stop bits and even parity.

The port parameter used in SETCOM does not correspond to that used with the PORT modifier.

The channel values listed on the # page have a one-to-oneness with the SETCOM port number only in the instances listed.

Only one instance of Modbus RTU is available for the serial. This means that you can only run Modbus on Port 1 OR port 2 NOT both UNLESS one of the instances is working as a Modbus RTU server (mode 11) and the other as a client.

With a single parameter, prints the configuration of the port to the selected output channel (the default terminal or that specified in OUTDEVICE).

Examples

Example 1

Set port 1 to 19200 baud, 7 data bits, 2 stop bits even parity and XON/XOFF enabled.

SETCOM(19200, 7, 2, 2, 1, 1)

Example 2

Set port 2 (RS485) to 9600 baud, 8 data bits, 1 stop bit no parity and no XON/XOFF handshake.

SETCOM(9600, 8, 1, 0, 2, 0)

Example 3

The Modbus protocol is initialised by setting the mode parameter of the SETCOM instruction to 4. The ADDRESS parameter must also be set before the Modbus protocol is activated.

ADDRESS = 1

SETCOM(19200, 8, 1, 2, 2, 4)

Example 4

The Modbus protocol is initialised by setting the mode parameter of the SETCOM instruction to 11. Therefore, the protocol is initialised as a Modbus server.

SETCOM(19200, 8, 1, 2, 2, 11, 0, 3, 0)

Example 5

In the Motion Perfect command line terminal, get a report of the settings for serial port 1.

SETCOM(1)

38400,8,1,2,1,0,4,0
