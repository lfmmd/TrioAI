GTAMAP

Type

Robotics Command

Syntax

value = GTAMAP

Description

Returns a string showing which GTA indices represent cartesian coordinates (i.e. were set using GTA, or GTAE with an explicit point type of 0). Each character represents a nybble (half a byte, or 4 bits) in the form of a hexadecimal character (‘0’ to ‘F’), with the low GTA indices being stored in the high order bits, as illustrated below.
GTA01234567891011
Character012
Bit321032103210

It is recommended to use GTAEMAP in preference to GTAMAP, as it handles all point types in a uniform manner, so GTAEMAP(0) is functionally equivalent to GTAMAP, as a point type of 0 represents cartesian coordinates.

Examples

GTA(0) = 1, 2, 3, 4, 5, 6

GTA(1) = 1, 2, 3, 4, 5, 6

GTAJ(2) = 1, 2, 3, 4, 5, 6

GTA(3) = 1, 2, 3, 4, 5, 6

GTAJ(4) = 1, 2, 3, 4, 5, 6

GTA(5) = 1, 2, 3, 4, 5, 6

GTA(6) = 1, 2, 3, 4, 5, 6

GTAJ(7) = 1, 2, 3, 4, 5, 6

GTAJ(8) = 1, 2, 3, 4, 5, 6

GTAJ(9) = 1, 2, 3, 4, 5, 6

GTA(10) = 1, 2, 3, 4, 5, 6

GTAJ(11) = 1, 2, 3, 4, 5, 6



PRINT "GTAMAP = "; LEFT(GTAMAP, 10)

PRINT "GTAJMAP = "; LEFT(GTAJMAP, 10)

PRINT "GTAEMAP(0) = "; LEFT(GTAEMAP(0), 10)

PRINT "GTAEMAP(1) = "; LEFT(GTAEMAP(1), 10)

Displays the following on the terminal:

>>GTAMAP = D620000000

GTAJMAP = 29D0000000

GTAEMAP(0) = D620000000

GTAEMAP(1) = 29D0000000
