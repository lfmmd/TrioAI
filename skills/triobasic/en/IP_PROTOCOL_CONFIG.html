IP_PROTOCOL_CONFIG

Type

System Parameter (MC_CONFIG)

Description

MC464: This Motion Coordinator is limited to 7 communication ports on Ethernet, IP_PROTOCOL_CONFIG allows the user to select which ports are in use and therefore reduce the load on the communications processor. By default, all ports except the transparent protocol text file loader port are enabled. It is recommended to use the MC4xx protocol which is enabled by default.

All other Motion Coordinators in the MC4xx, MC5xx and MC6xx ranges; this parameter takes a value that is the number of Ethernet packets that are read at a time. The default value is 0, which allows the firmware to automatically decide the data size that is appropriate to efficient operation.

Value (MC464 only)

Up to 7 bits can be selected, the default value is 575 ($23F).
BitDescriptionValue
0Motion Perfect (Telnet)1
1PCMotion2
2Modbus4
3EthernetIP8
4IEC61131-3 programming16
5Uniplay32
6Transparent protocol text file loader64
7Reserved bit128
8Reserved bit256
9MC4xx protocol text file loader512

Do not disable bit 0 otherwise the command line and Motion Perfect will not be usable.

Value (All other Motion Coordinators)

The number of incoming Ethernet packets that trigger the firmware to read data from the Ethernet stack. Default value 0 means Automatic.

Example

In the MC464, enable the standard ports using bits 0-5 and the transparent protocol text file loader ports.

IP_PROTOCOL_CONFIG = 1 + 2 + 4 + 8 + 16 + 32 + 64

'or

IP_PROTOCOL_CONFIG = $7F
