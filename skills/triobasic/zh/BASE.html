BASE

类型

系统命令

语法

BASE(axis0[, axis1[, axis2[, ...]]])

替代格式

BA(...)

描述

BASE命令用于将所有后续运动命令和轴参数读/写定向到特定轴或一组轴。 默认设置是一个序列：0、1、2、3 ...

每个过程有其独立的BASE轴组，每个程序能独立设置BASE值，所以BASE数组也许与程序或者命令行中不同。

这些值存储在一个数组中，当您调整BASE时，控制器将通过继续该序列，然后在末尾自动添加的值来填充其余位置。通过输入不带任何参数的BASE，可以在命令行上打印BASE数组。参数

axis numbers:成为新基础轴阵列的一个或多个轴的编号，即，将运动命令发送到的一个或多个轴或多轴命令中的第一个轴。

当Kinematic FRAMEs启用时，BASE数组必须使用升序值，如果轴顺序没有设置为从低到高，则会导致Run-time错误。如果需要非升序BASE数组，例如BASE(3, 0, 1)，则设置 COORDINATOR_DATA 函数16为1.

例

例 1

设置轴1和轴2的校准单元、速度和加速度值。

BASE(1)

UNITS = 2000     '单位转换系数

SPEED = 100      '设置轴1速度 (单位/秒)

ACCEL = 5000     '加速度 (单位/秒/秒)

BASE(2)

UNITS = 2000     '单位转换系数

SPEED = 125      '设置轴2速度

ACCEL = 10000    '加速度

例 2

在轴上设置一个插补移动；0 (x), 6 (y) 和 9 (z)。轴0移动100单位，轴6移动过-23.1单位，轴9移动1250单位。轴将以轴0设置的速度加速度沿着合成路径移动。

BASE(0, 6, 9)

SPEED = 120

ACCEL = 2000

DECEL = 2500

MOVE(100, -23.1, 1250)

例 3

将基数设置为不连续值，并将它们打印回命令行。这个例子使用了一个15轴控制器。

控制器自动以10继续序列，然后在列表末尾填充缺失的值。

>>BASE(1, 5, 9)

>>BASE

(1, 5, 9, 10, 11, 12, 13, 14, 15, 0, 2, 3, 4, 6, 7, 8)

>>

例 4

系统（不使用运动学）需要按降轴顺序设置BASE。必须首先设置COORDINATOR_DATA 以允许它。

在使用此命令之前，请阅读有关COORDIVATOR_DATA的章节。

COORDINATOR_DATA(16, 1) 'Do not check axis order

BASE(3, 2, 1)

SPEED = 50

ACCEL = 500

DECEL = 500

MOVE(z, y, x)
