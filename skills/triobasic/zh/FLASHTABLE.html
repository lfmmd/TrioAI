FLASHTABLE

类型

系统功能

语法

value = FLASHTABLE(function, flashpage[, tablepage])

描述

将RAM中的TABLE数据复制到永久闪存中的功能。

闪存分成32页，从0开始编号，每页可存储16000个Table元素。下表显示了前几个Table页以及它们如何映射到表元素:
表页面0123
表索引范围0 ~ 1599916000 ~ 3199932000 ~ 4799948000 ~ 63999

Table页存储在flash页中，其中有32个(索引0到31)，因此flash页可以用来存储不同的Table页，以便于恢复，比如定义多个行为。

如果正在使用FLASHTABLE，那么就不能使用FLASHVR(-1)

参数

value:TRUEFLASH页面已被使用(当函数设置为3时)。
FALSEFLASH页面未被使用(当函数设置为3时)。
function:指定所需的操作。
1将一页TABLE数据写入闪存，总是返回TRUE。
2将一页闪存读入TABLE数据，总是返回TRUE。
3如果flash页面正在使用(已经有TABLE数据被刷新)，返回TRUE，否则返回FALSE。
4擦除flash页，总是返回TRUE。
flashpage:索引号(0…31)的一个有16000个值的Flash EPROM页面，表数据将被存储到该页面或从中检索。
tablepage:索引号(0…INT(TSIZE/ 16000))Table内存中要从其中复制或恢复数据的页面。(仅与函数1和2一起使用)。

例1

保存(功能1)TABLE第2页位置(TABLE索引32000到47999)的数据到闪存第5页。

FLASHTABLE(1, 5, 2)

例2

从闪存第3页中恢复到(功能2)TABLE第2页的位置(TABLE索引32000到47999)的数据。

FLASHTABLE(2, 3, 2)

例3

检查FLASH内存第5页是否正在被使用(功能3)，如果有数据，将其擦除。

IF FLASHTABLE(3, 5) THEN

    PRINT "FLASH memory page 5 has data"

    IF FLASHTABLE(4, 5) THEN

PRINT "FLASH 闪存第5页已经被删除"

ENDIF

ENDIF
