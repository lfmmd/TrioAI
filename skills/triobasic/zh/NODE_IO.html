NODE_IO

类型

系统数组(MC_CONFIG)

语法

NODE_IO(slot, node_address) = offset_value

描述

这个二维数组可用于设置任何EtherCAT节点I/O通道的起始地址。这可以用来定义一个用户特定的I/O映射，将固定来自不同来源的I/O点。

这个数组是二维的，第一维是主槽标识符，第二维是节点在主网络中的位置。

默认的I/O分配顺序是:控制器I/O，模块I/O, CAN I/O, EtherCAT I/O(驱动器I/O和独立的EtherCAT I/O)

当使用多个I/O类型时，最好使用I/O基本修改参数来配置I/O顺序。见下文。

参数

slot:EtherCAT端口槽位号。(MC4N、MC6N和Flex-6 Nano上为0)
node_address:由节点在EtherCAT网络中的位置决定的地址。第一个节点是地址0。
offset_value0自动分配EtherCAT I/O(默认)
> = 8定义节点的EtherCAT I/O从这个I/O点地址开始

如果需要根据EtherCAT配置的地址分配I/O偏移量，请使用NW_NODE_IO

例

一个具有MC464、EtherCAT模块(插槽0)和节点5的128输入EtherCAT I/O模块的系统通常具有以下I/O分配:

MODULEIO_BASE = 0: NODE_IO(0, 5) = 0: CANIO_BASE = 0
0-7内置输入
8-15内置双向I/O
16-23P876 EtherCAT模块输入
24 - 151EtherCAT输入
152 - 1023虚拟I / O

如果更改了NODE_IO，则可以将EtherCAT I/O移动到更高的值。

MODULEIO_BASE = 0: NODE_IO(0, 5) = 72: CANIO_BASE = 0
0-7内置输入
8-15内置双向I/O
16-23P876 EtherCAT模块输入
24 - 71虚拟I / O
72 - 199EtherCAT输入
199 - 1023虚拟I / O

将EtherCAT I/O放在首位，设置如下

MODULEIO_BASE = 144: NODE_IO(0, 5) = 0: CANIO_BASE = 0
0-7内置输入
8-15内置双向I/O
16 - 143EtherCAT输入
144 - 151P876 EtherCAT模块输入
151 - 1023虚拟I / O
