STICK_READ

类型

系统功能

语法

value = STICK_READ(flash_file, table_start [,format])

描述

从SD卡读取table数据到控制器。

任何现有的TABLE数据都会被覆盖。

Binary格式提供了最好的数据精度。

参数

value:TRUE =函数成功
FALSE =函数不成功
flash_file:一个数字，当附加到字符“SD”将形成数据文件名。
硬编码的字符串或string变量将形成数据文件名。
table_start:TABLE中数据值将被传输到的起始点。(使用format 2时忽略)
format:0 =二进制64位浮点格式，BIN文件(默认)
1 = ASCII码逗号分隔值，文件，无索引CSV
2 = ASCII CSV格式:索引，值<CR>

当以format=0存储时，数据以IEEE浮点二进制格式little-endian存储，即最不重要的字节优先。

例

例1

从SD卡读取ASCII CSV文件SD001984.csv，并将数据复制到从TABLE(16500)开始的table内存中。

success = STICK_READ (1984, 16500, 1)

IF success=TRUE THEN

  PRINT #5,"SD card read OK"

ENDIF

例2

从SD卡读取ASCII CSV文件MYDATALOG2.CSV，并将数据复制到从TABLE(16500)开始的table内存中。

success = STICK_READ ("MYDATALOG2", 16500, 1)

IF success=TRUE THEN

  PRINT #5,"SD card read OK"

ENDIF

例3

从SD卡读取ASCII CSV文件DATA1.CSV，并使用文件中的table索引将数据复制到table内存中。文件格式为Index, value <cr>作为CAD2Motion输出。

DIM flash_file AS STRING(20)

DIM s AS BOOLEAN

flash_file = "DATA1"



s = STICK_READ(flash_file, 0, 2)



IF s = TRUE THEN

  PRINT flash_file; " read OK"

ELSE

  PRINT "Error reading "; flash_file

ENDIF
