VP_MODE

类型

轴参数

语法

VP_MODE AXIS(n) = value

描述

轴参数VP_MODE允许程序员选择不同的速度曲线类型以适应不同的应用。VP_MODE = 0是所有运动控制器的默认设置。

值

0该模式下的S曲线是通过数字滤波对速度曲线进行处理，处理时间(以毫秒为单位) 由SRAMP命令设置
1未使用
2未使用
3未使用
4未使用
5应用JERK控制多轴插补
6应用JERK控制多轴插补，类似于VPMODE = 5

VP_MODE = 0

该模式下的S曲线是通过数字滤波对速度曲线进行处理，处理时间(以毫秒为单位) 由SRAMP命令设置。

优点:

  • 可以适用于任何速度轮廓
  • SPEED/ACCEL/DECEL可随时更改

缺点:

  • 短距离和长距离的运动不适用相同的SRAMP值
  • 数字滤波器使用整数运算，在低分辨率时会导致噪声
  • SRAMP值不能快速更改
  • 加速度和加加速度不是数学函数

例

UNITS = 1000

length = 50000

SPEED = 50000

ACCEL = 140000

DECEL = 140000

SRAMP = 200



DEFPOS(0)

VP_MODE = 0



TRIGGER



MOVE(length)

这个Motion Perfect示波器跟踪显示了SRAMP=200的效果。

VP_MODE = 5

应用固定的JERK设置。由于jerk是加速度的导数，所以它的单位是/sec/sec/sec。

优点:

  • 使用双精度浮点数计算的数学定义轮廓
  • 适合于多轴插补运动
  • 通过应用jerk来调节加速和减速的斜坡变化，从而实现平滑运动

例

描述一个有圆角的正方形，在形状的每个部分快速移动。

LIMIT_BUFFERED = 4



BASE(1, 2) '设置基本数组

SPEED = 300 'mm/s

ACCEL = 6000 'mm/s^2

DECEL = ACCEL

FORCE_SPEED = SPEED

ENDMOVE_SPEED = SPEED

CORNER_MODE.3 = ON '在圆弧半径减速

FULL_SP_RADIUS = 10.0

VP_MODE = 5

JERK = 500000 'mm/s^3



MERGE = ON '在运动间不停顿

MOVEABSSP(0, 5.0)

WAIT IDLE



TRIGGER

WA(100)

MOVESP(0, 100.0)

MOVECIRCSP(5.0, 5.0, 5.0, 0, 1)

MOVESP(100.0, 0)

MOVECIRCSP(5.0, -5.0, 0, -5.0, 1)

MOVESP(0, -100.0)

MOVECIRCSP(-5.0, -5.0, -5.0, 0, 1)

MOVESP(-100.0, 0)

MOVECIRCSP(-5.0, 5.0, 0, 5.0, 1)

WAIT IDLE

VP_MODE = 6

此模式结合了模式0和模式5的功能，使其成为所有运动使用的主要轮廓。由于大多数应用程序使用VP_MODE 0，它的设计方式与模式0相同，但对于以前使用模式5 (PLCopen运动功能块和RPS)的应用程序，其行为可以使用VP_OPTIONS进行调整。

该模式利用均匀的加加速度轮廓，因此不需要使用SRAMP关键字来平滑运动，当以模式5的方式进行编程时，可以增强灵活性，以适应运动进行中的需求变化。

优点:

  • 使用双精度浮点数计算的数学定义曲线
  • 适合于多轴插补运动
  • 通过应用jerk来调节加速和减速的斜坡变化，从而实现平滑运动
  • 可以通过使用VP_OPTIPNS 将其行为设置为类似 VP_MODE 0或 5
