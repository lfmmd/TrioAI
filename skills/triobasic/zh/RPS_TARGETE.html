TARGETE

类型

机器人类型

语法

DIM <var-list> AS TARGETE

<target-variable> = x, y, z, u, v, w, ptype, rnum, rconf, revs, of, tool, rf, userdata

<target-variable> = <target-variable>

描述

TARGETE是TARGET数据类型的扩展版本，能够存储额外的数据，并用于机器人移动命令，如MOVEC, MOVEJ 和 MOVEL。TARGETE变量仅供本地使用，可以像程序中的其他变量一样定义。对于全局使用，请使用全局目标数组(参见GTAE)。

可以通过在声明语句中定义一个局部目标数组的维度来实现:

DIM targetArray AS TARGETE(12)

可以使用“点”表示法(“。”)对目标的元素进行读写，如下表所示:

my_variable = my_target.x

my_target.y = 12.2

局部和全局目标之间的赋值是可能的:

GTAE(20) = GTAE(14)

GTAE(10).x = GTAE(11).x

my_target_1 = GTAE(10)

GTAE(11) = my_target_2

TARGETE需要RPS run-time FEC。

参数

名字描述
x, y, z, u, v , w: Float笛卡尔位置(mm)和方向(度)，或关节位置(根据机器人类型要求的mm或度)。
ptype: Integer点类型。
值:0 =笛卡尔，1 =关节。
rnum: Integer用于目标的机器人编号。
rconf: Integer目标的机器人配置设置。
revs: Integer取目标时从校准点算起的最后一个轴转速值。
of: Integer用于目标的对象框架。
tool: Integer用于目标的工具偏移量。
rf: Integer用于目标的机器人框架。
userdata: Float任何有用的数值都可以存储在这里。

x, y, z, u, v, w的命名惯例是为了反映笛卡尔坐标，因为这是它们最初的唯一目的。虽然关节目标扩展了此功能(请参阅TARGETJ)，但TARGETE数据类型进一步扩展了它，以包含笛卡尔、关节和用户定义的点类型，同时为位置保留相同的命名约定。

例

例1

定义一个包含2个扩展目标的数组。

DIM targets AS TARGETE(2)

targets(0) = 104, -400, -29, 180, 0, -68, 0, 2, 0, 10, 3, 13, 23, 69.420

targets(1) = 114, -410, -29, 180, 0, -68, 0, 2, 0, 10, 3, 13, 23, 79.420
