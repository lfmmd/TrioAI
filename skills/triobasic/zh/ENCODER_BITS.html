ENCODER_BITS

类型

轴参数(MC_CONFIG)

描述

该参数仅用于绝对编码器轴。 它用于设置轴硬件要从编码器输出的数据位数。 该参数支持2种类型的绝对值编码器： SSI和EnDat。

如果要在运行时更改ENCODER_BITS的数量，则必须在输入新值之前先将ATYPE参数设置为零，然后再将其设置为SSI的值。

值

编码器类型位值函数
All:00没有数据从编码器移出(默认)
SSI:位0 - 50-32编码器要输出的位数。
位664设置为Binary，清空为Gray code(默认)
位7128反转方向(反转数据位)
位8 - 10256 * n

0<n<7		数据移位的位数。用于允许忽略状态位。
EnDat:位0 . .70 - 255返回的数据位总数
位8 . . 13256 * n

0<n<64		多圈位的个数
点1416384这是控制器在从编码器位置数据计算出正确的CRC时设置的。

例

例1

设置2个轴的SSI绝对编码器

ENCODER_BITS AXIS(3) = 12

ENCODER_BITS AXIS(7) = 21

例2

使用编码器的绝对值重新初始化MPOS

SERVO=OFF

ENCODER_BITS = 0

ENCODER_BITS = databits

例3

25位EnDat编码器具有12多圈和13位/圈分辨率。(总位数为25位)

ENCODER_BITS = 25 + (256 * 12)

ATYPE = 47

例4

设置21位数据的SSI绝对值编码器有3个位状态分别在第21位，22位和23位。SSI将时钟为24位，但只有0到20位将被传输到编码器和MPOS。

ENCODER_BITS = 21 + (3 * 256)

例5

雷尼绍Biss-C线性RL32BAT050B10A编码器，具有32位数据和50纳米分辨率。连接MC405上的4轴。使用_修改程序 FPGA…_选择包含Biss-C编码器支持的FPGA。

BASE(4)

' 清除当前轴类型

ATYPE = 0

' 清除任何先前的设置并禁用编码器

ENCODER_BITS = 0

WA(2)

' 设置为32位，2进制数据

ENCODER_BITS = 64 + 32

' 编码器输入(92 = biss伺服)

ATYPE = 93

'短延迟，允许编码器初始化和传输有效数据

WA(10)

'校准轴为mm刻度

UNITS = 1 / (50 * 10 ^ -6)
