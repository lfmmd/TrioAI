HLM_COMMAND

类型

Hostlink参数

语法

HLM_COMMAND(command, port[, node[, mc_area/mode[, mc_offset]]])

描述

HLM_COMMAND命令对选定端口上的一个或所有主机链接从设备执行特定的主机链接命令操作。 程序执行将暂停，直到收到响应字符串或超时时间为止。超时时间通过使用HLM_TIMEOUT参数指定。HLM_STATUS可以使用参数监视传输的状态。

参数

command:需要执行的主机链路操作:
HLM_MREAD0执行HostLink PC MODEL READ (MM)命令读取CPU单元型号代码。结果被写入由mc_area和mc_offset指定的MC 单元变量。
hlm_test1通过发送字符串“MCxxx TEST string”并检查返回的字符串来执行HostLink TEST(TS)命令，以检查通信是否正确。检查HLM_STATUS参数的结果。
hlm_abort2执行Host Link ABORT (XZ)命令，以中止当前正在处理的主机链接命令。ABORT命令不会收到响应。
HLM_INIT3执行Host Link INITIALIZE（**）命令来初始化所有从单元的传输控制过程。
HLM_STWR4执行Host Link SC (Host Link 状态写入)命令修改CPU单元的工作模式。
port:指定的串行端口。（有关编号，请参阅特定的控制器规格）
node:（对于HLM_MREAD, HLM_TEST, HLM_ABORT和HLM_STWR）：
发送HostLink命令的从站节点号。范围:[0,31]。
mode:(HLM_STWR)
指定CPU单元的工作模式。
0程序模式
2监控模式
3运行模式
mc_area:(HLM_MREAD)
MC Unit的存储器选择，用于将接收到的数据写入。
MC_TABLE8Table变量数组
MC_VR9全局(VR)变量数组
mc_offset:(HLM_MREAD)
指定的MC Unit要读取的内存区域的地址。

当使用Hlm_command时，请确保使用SETCOM命令设置HostLink主协议。

HostLink主命令只需要从一个程序任务执行，以避免任何多任务时序问题。

例

例1

下面的命令将读取连接到RS-232C端口的节点地址为12的HostLink从站的CPU单元型号代码。结果被写入VR(233)。

HLM_COMMAND(HLM_MREAD, 1, 12, MC_VR, 233)

如果连接的从站是一台C200HX PC，那么VR(233)在成功执行后将包含值12(十六进制)。

例2

下面的命令将检查与连接到RS-422A端口的主机链路从机(节点23)的主机链路通信。

HLM_COMMAND(HLM_INIT, 2)

HLM_COMMAND(HLM_ABORT, 2, 4)

如果hlm_status参数的值为0，则表示通信正常。

示例3

下面两个命令将在RS-422A端口2上执行主机链路INITIALIZE和ABORT操作。从站的节点号为4。

HLM_COMMAND(HLM_INIT, 2)

HLM_COMMAND(HLM_ABORT, 2, 4)

例4

当需要通过HostLink向PC写入数据时，CPU Unit不能处于运行模式。可以使用Hlm_command命令将其设置为MONITOR模式。从站节点地址为0，连接RS-232C端口。

HLM_COMMAND(HLM_STWR, 2, 0, 2)
