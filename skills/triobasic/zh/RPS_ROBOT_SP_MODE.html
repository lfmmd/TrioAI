ROBOT_SP_MODE

类型

机器人参数

语法

ROBOT_SP_MODE = value

描述

该参数将插补放入适合具有附加轴(UVW)的XYZ笛卡尔机器人的模式，可以控制工具末端或“关节”。

当ROBOT_SP_MODE开启时，系统分别计算XYZ组(例如轴0,1,2)和UVW组(例如轴3,4,5)的路径长度。假设XYZ组从0轴取速度，UVW关节组从3轴取速度，它将根据哪个移动需要更长的时间来计时。它通过调整“路径长度”来实现这一点，因此U轴速度仅在移动计算时间内产生影响。

在这种模式下，所有轴上的INTERP_FACTOR都应该设置为1.0。

这个参数被设计用来处理非常特定的轴布局。如果在更普遍的情况下使用，可能达不到预期的效果。如有疑问，请联系Trio。

值

0插补使用组内所有轴来计算路径。
1插补在机器人模式1

例

例1

因此，运动可以在关节轴上执行，同时保持XYZ位置静止，设置ROBOT_SP_MODE = ON并按顺序运行6轴移动。

ROBOT_SP_MODE = 1

FOR ax=0 TO 5

    INTERP_FACTOR AXIS(ax)=ON

NEXT ax



MOVEABS(0,0,0,0,0,0)

MOVEABS(5,15,20,u1,v1,w1) ' 6个轴全部移动

MOVEABS(10,20,30,u1,v1,w1) ' 只有XYZ轴被移动

MOVEABS(10,20,30,u2,v2,w2) ' 只有关节轴被移动

将轴3,4和5上的INTERP_FACTOR设置为OFF将导致最后一次移动无法按预期运行。

例2

演示如何XYZ和UVW可以有自己的速度设置。花费最长时间完成移动的轴将控制两组轴的完成时间。也就是说，“更快”的轴将运行得更慢。

ROBOT_SP_MODE = 1



FOR ax=0 TO 5

   INTERP_FACTOR AXIS(ax)=ON

NEXT ax



MOVE_COUNT=0



UNITS AXIS(0)=1000 ' gives mm on XYZ

UNITS AXIS(1)=1000 ' gives mm on XYZ

UNITS AXIS(2)=1000 ' gives mm on XYZ



UNITS AXIS(3)=10000/360 ' gives deg on wrist

UNITS AXIS(4)=10000/360 ' gives deg on wrist

UNITS AXIS(5)=10000/360 ' gives deg on wrist



SPEED AXIS(0)=100 ' mm / sec

SPEED AXIS(3)=60 '  deg / sec

ACCEL AXIS(0)=5000 : DECEL AXIS(0)=5000



TRIGGER

WA(20)



DEFPOS(0,0,0,0,0,0)



' XYZ和UVW距离使用3D毕达哥拉斯计算

MOVEABS(5,15,20,30,45,20)

WAIT IDLE



' 只移动XYZ  XYZ:46.4mm UVW:0 度

MOVEABS(10,60,30,30,45,20)

WAIT IDLE



' 只移动关节轴   XYZ:0mm    UVW:57.7 度

MOVEABS(10,60,30,0,0,0)

WAIT IDLE
