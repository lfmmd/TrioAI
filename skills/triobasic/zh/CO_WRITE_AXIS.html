CO_WRITE_AXIS

类型

系统命令

语法

CO_WRITE_AXIS(axis_number, index, subindex, type, vr_number[, value])

描述

该函数在远程驱动器或I/O设备中设置CANopen-over-EtherCAT (CoE)对象。对象的索引和子索引用于向该对象写入值。该值可以来自VR，也可以在VR编号为-1时直接输入命令。

参考远程设备的手册获取可用对象列表。如果对象值设置成功，该命令返回TRUE(-1)。否则，在写入值时出现错误的情况下，命令返回FALSE(0)。

参数

Axis_number:EtherCAT驱动器的轴号。
index:CANopen对象索引
subindex:CANopen对象子索引
Type:1Boolean
2Integer 8
3Integer 16
4Integer 32
5Unsigned 8
6Unsigned 16
7Unsigned 32
8Real 32
17Real 64
21Integer 64
27Unsigned 64
vr_number:当VR值介于0和最大值之间时，结果将被存储。

(如果下一个参数包含要写入的值，则为-1)
value:直接设置对象的可选数据值

例

例1

将值1写入MC464轴3的伺服驱动器上的制造商特定对象。CoE对象0x2802子索引0x00，类型2(8位整数)。获取TRUE/FALSE成功指示并打印到终端。

>>?CO_WRITE_AXIS(3, $2802, 0, 2, -1, 1)

>>-1.0000

>>

例2

将位置控制器的速度前馈增益写入MC464轴12的伺服驱动器。CoE对象0x60FB子索引0x02，类型6(无符号16位整数)。

VR(2010) = 1000

'从VR(2010)写入值

error_flag = CO_WRITE_AXIS(12, $60fb, 2, 6, 2010)



IF error_flag = FALSE THEN

  PRINT "向驱动器写入CANopen对象错误"

ENDIF

警告:在通过EtherCAT写入CANopen对象之前，请务必参阅制造商的用户手册。
