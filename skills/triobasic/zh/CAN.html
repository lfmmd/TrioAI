CAN

类型

系统命令

语法

CAN(channel, function[, parameters])

描述

该功能允许从Trio BASIC控制CAN通信通道。所有_运动控制器_都有一个内置的CAN通道，通常用于使用Trio I/O模块的数字量和模拟量I/O。

除了使用CAN命令来控制CAN通道外，固件中还有特定的协议功能。这些功能是与特定设备接口的专用软件模块。如果系统参数CANIO_ADDRESS设置为默认值32，内置的CAN通道将自动扫描Trio I/O模块。
通道:通道数:最大波特率:
内置 CAN11 MHz

控制器中有16个消息缓冲区

参数

slot:内置CAN端口设置为-1
function:0读取寄存器，除非Trio或经销商指示，否则不要使用。
1写寄存器，除非Trio或经销商指示，否则不要使用。
2初始化波特率
3检查收到的信息
4发送OK
5初始化消息
6读消息
7写消息
8读取CANopen对象
9写CANopen对象
11初始化29位消息
20CAN模式
21启用CAN驱动
22重置CAN消息缓冲区
23指定CAN VR映射
24启用并配置同步报文
25设置Heartbeat消费者
26开启Heartbeat监视器

语法

描述

语法

variable:要写入的消息在VR存储器中的起始位置

语法

CAN(channel, 23, [message, map, offset, length, order, variable, direction[, data_type]])

描述

参数

函数 = 24

语法

CAN(channel, 24, enable, message, period)

描述

设置CANopen I/O模式的循环同步报文。CANIO_ENABLE设置为1后，固件将在指定的周期发送同步报文，与运动控制器的内部伺服周期同步。

参数

enable:1 =启用同步报文，0 =禁用
message:消息缓冲区(0..63)
period:同步周期(毫秒)

例

CAN(-1, 5, 14, 128, 0, 1) '设置 SYNC CobID=$80 (128)缓存14

CAN(-1, 24, 1, 14, 4) '每4毫秒同步一次报文



CAN(-1, 7, 15, 1, 0) '将CANopen从模块设置为运行状态

CANIO_ENABLE = 1

函数 = 25

语法

CAN(channel, 25, message, node, period)

描述

设置一个CAN消息缓冲区来监视产生的heartbeat报文，如果在指定的时间内没有接收到heartbeat，则触发故障。每个具有heartbeat功能的节点都需要在控制器中为heartbeat报文定义一个单独的CAN消息缓冲区。

不需要使用CAN函数5设置COB-ID。功能25包括在所选CAN缓冲区中自动设置heartbeatCOB-ID。

参数

message:消息缓冲区(0..63)
node:heartbeat生产者的CANopen节点地址
period:同步周期(毫秒)

例：

在运动控制器中设置3个从站设备作为heartbeat生产者和相应的heartbeat消费者通道。CAN缓冲区41 ~ 43在本例中可用。CAN缓冲器62和63用于SDO Tx和Rx报文。

'配置从站每30毫秒发送一次heartbeat

'为从节点设置SDO缓冲区

node = 1

CAN(-1, 5, 62, $580 + node, 8, 0) 'Receieve for SDO

CAN(-1, 5, 63, $600 + node, 8, 1) 'Transmit for SDO

'发送SDO写入

CAN(-1, 9, 63, 62, 16, $1017, 0, 30)



'在主站中设置heartbeat监测器，超时40毫秒

CAN(-1, 25, 40 + node, node, 40)



'重复节点2和节点3

node = 2

CAN(-1, 5, 62, $580 + node, 8, 0)

CAN(-1, 5, 63, $600 + node, 8, 1)

CAN(-1, 9, 63, 62, 16, $1017, 0, 30)

CAN(-1, 25, 40 + node, node, 40)



node = 3

CAN(-1, 5, 62, $580 + node, 8, 0)

CAN(-1, 5, 63, $600 + node, 8, 1)

CAN(-1, 9, 63, 62, 16, $1017, 0, 30)

CAN(-1, 25, 40 + node, node, 40)

函数 = 26

语法

CAN(channel, 26, enable)

描述

在运动控制器中启用和禁用CANopen Heartbeat监视器。启用后，指定的Heartbeat CAN缓冲区将按照CAN功能25中的设置进行定期监控。如果从设备(heartbeat生产者)没有在规定的时间内发送heartbeat报文，则将WDOG设置为OFF，并设置CANIO_STATUS位以指示错误的从站设备身份。

参数

enable:1 =启用heartbeat，0 =禁用

例

设置一个CIA401输入/输出模块使用CANopen heartbeat，并配置运控制器监控从站发出的heartbeat报文。从站必须支持CANopen对象0x1017 Heartbeat Time (ms)。运动控制器将周期性地发送输出值，或者同步报文作为从设备的heartbeat输入。

node = 2 '从站节点号

heartbeat_buf = 40 'heartbeatRx的CAN缓冲区数

txbuf = 62 ' SDO Tx/Rx的CAN缓冲区

Rxbuf = 63



CAN (- 1,26,0) '设置时禁用heartbeat系统



'配置从站每25毫秒发送一次heartbeat

CAN(-1, 9, txbuf, rxbuf, 16, $1017, 0, 25)



'在master上设置heartbeat监视器，超时50毫秒

CAN(-1, 25, heartbeat_buf, node, 50)



CANIO_ENABLE = 1 '启动CANI /O PDO传输

CAN(-1,26,1) '启用heartbeat系统
