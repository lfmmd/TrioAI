ATTACH_OBJECT_FRAME

类型

机器人命令

语法

value = ATTACH_OBJECT_FRAME(object frame)

附加在传送带上:

ATTACH_OBJECT_FRAME(control, object frame, syncVector, syncPos, syncAxis)

附加到多用途机器上:

ATTACH_OBJECT_FRAME(control, object frame, axisX, axisY, axisZ, axisU, axisV, axisW, xOffset, yOffset, zOffset, uOffset, vOffset, wOffset)

描述

附加对象框架是一种将对象框架的向量与最多六个轴连接并将其放置在特定位置的方法。然后，对象框架将沿着这些轴移动。

附加对象框架有两种不同的模式;一种是专门设计用于与传送带一起工作的，另一种是与多用途机器一起工作的。

在与轴连接或分离之前，必须先创建一个对象框架。

OBJECT_FRAME需要RPS run-time FEC。

参数

参数数据类型描述
ValueBooleanFalse / 0 =指定的对象未附加框架。
True /非零值=指定的对象框架已经附加。
controlIntegerATTACH_OBJECT_FRAME模式。
1 =用传送带模式附加对象框架
2 =以多用途机器模式附加对象框架
4 =分离对象框架
object frameInteger /
String		要附加的对象_框架_。
值=索引或名称或对象_框架_。
单独使用时，该命令将返回:
TRUE =指定的对象框架依附于至少1个轴
FALSE =指定的对象框架不附加到任何轴
syncVectorInteger将附加到同步轴的对象_框架_向量。
值:
1 = x
2 = y
3 = z
syncPosTarget在同步轴上捕获位置。
syncAxisInteger与之同步的轴。
axisX, axisY, axisZ,
axisU, axisV, axisW		Integer与将要附加的对象_框架_向量对应的机器轴编号。
非附加向量设为-1。
xOffset, yOffset, zOffset,
uOffset, vOffset, wOffset		Integer对象_框架_相对于机器人原点的位置和方向

例

例1

检查对象框架“of2”是否附加到任何轴上。

ATTACH_OBJECT_FRAME("of2")

例2

将对象框架“托盘”附加到轴11。

ATTACH_OBJECT_FRAME(1,"pallet",1,registed_position,11)

例3

将物体框架3与多用途机器的11轴Y向量相连。x和z位置偏移400mm, Y方向偏移90度。

ATTACH_OBJECT_FRAME(2,3,-1,11,-1,-1,-1,-1,400,0,400,0,90,0)

例4

由于传送带可以处于不同的方向，因此可以在多个方向上设置对象_框架_，以匹配传送带的方向。所选择的对象_框架_向量将遵循传送带矢量的方向。

例如，取放操作。一个常见的行业例子是从托盘中取出物品并将其放置到另一个工位或包装中。采用附加对象框架技术可以简化同步过程。对象框架将被放置在托盘的特定位置。所有的对象将被教导相对于对象框架“托盘”。对象框架“托盘”将使用三种可能的选项之一连接到输送机上:向量X，向量Y或向量z。请记住，传送带无法与机器人完美对齐，因此可以使用对象框架的方向数据来调整它。

ATTACH_OBJECT_FRAME(1,"pallet",1,registed_position,11)

命令执行后，对象框架托盘”将精确地跟随输送机。在这种状态下，机器人应该与对象框架“托盘”同步。请转到下一节继续示例。

例5

ATTACH_OBJECT_FRAME也可以与多用途机器一起使用。

比如多轴的机器。可以附加最多6个轴的位置和方向矢量来改变控制参数。例如，如果将对象_框架_附加到6自由度的机器人手臂上，则对象_框架_的位置和方向将遵循机器人TCP。

ATTACH_OBJECT_FRAME(2,3，-1,11，-1，-1，-1，- 1400,0,400,0,90,0)

例6

使用控制模式4从坐标轴上分离对象_框架_。

ATTACH_OBJECT_FRAME(4，"of2")
