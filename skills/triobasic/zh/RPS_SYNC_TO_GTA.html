SYNC_TO_GTA

类型

机器人命令

语法

SYNC_TO_GTA(control, time[, syncPos, syncAxis, object frame, GTA])

描述

机器人TCP可以与移动的物体或另一台机器或机器人同步。这意味着机器人TCP将以与同步对象相同的方向移动或旋转，并在其上进行绝对运动。

同步有两种类型:

  • 对象框架同步
  • GTA同步

GTA同步是为更简单的同步而设计的。它使机器人TCP在X方向上与一个轴同步。如果机器人没有与X方向完美对齐，则始终可以预先设置和选择对象Frame。

参数

参数数据类型描述
ControlInteger1 =启动同步
4 =停止同步
10 =重新同步到另一个GTA
timeInteger以毫秒为单位完成同步的时间。
(optional) syncPosTarget在同步轴上捕获位置。
不需要控制值4(停止同步)。
(optional) syncAxisInteger与之同步的轴。
不需要控制值4(停止同步)。
(optional) object frameInteger /
String		与选定GTA相关的对象Frame。
值=对象Frame的索引或名称，如果没有对象Frame则为0(默认)。
不需要控制值4(停止同步)。
(optional) GTAInteger与GTA同步。
值= GTA的索引或名称，或-1表示没有GTA。
不需要控制值4(停止同步)。

产品随机放置在传送带中。然后将它们挑选出来并放置到特定的位置。相机视觉系统将处理产品的位置和方向，并将这些信息发送给控制器。根据所使用的相机技术，可以使用以太网套接字连接或Trio ActiveX发送信息。相机测量的偏移量是相对于相机原点的，因此我们需要通过使用OBJECT_FRAME来告诉系统原点在哪里。

请记住，机器人将与X方向同步，因此OBJECT_FRAME的X向量应与输送机的方向对齐。在相机原点设置OBJECT_FRAME后，GTA应该与检测到的偏移量一起设置。在这个例子中，相机会向一个表data发送5个参数:

表0:X偏移量，表1:Y偏移量，表2:方向，表3:输送机位置，表4:检测到

代码:

fixed_z = 50 ' 由于输送机与地面平行，Z将始终相同



' 打开夹具

OP(gripper,OFF)



SYNC_TO_GTA(4,500) ' 在500ms内进行同步

WAIT UNTIL SYNC_CONTROL = 0 ' 等待完全同步



WAIT UNTIL TABLE(4) = 1 ' 等待检测到产品

TABLE(4,0) ' 重置产品检测



'使用相机测量的偏移量设置GTA值



GTA(0) = TABLE(0), TABLE(1),fixed_z, 180,0, TABLE(2)



'当相机检测到产品时，注册输送机位置

registed_position = TABLE(3)



' S同步到10轴，GTA(0)在目标Frame"相机"(1)中2000ms

SYNC_TO_GTA(1,2000,registed_position,10,1,0)

WAIT UNTIL SYNC_CONTROL = 3 ' 等待直到完全同步



' 关闭夹具

OP(gripper,ON)

WA(200) ' 等待抓手关闭



SYNC_TO_GTA(4,500) ' 等待抓手关闭

WAIT UNTIL SYNC_CONTROL = 0 ' 直到完全同步



STOP
