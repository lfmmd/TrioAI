CANIO_ADDRESS

类型

I/O参数(MC_CONFIG / FLASH)

描述

CANIO_ADDRESS用于设置CANbus的工作模式。当您想使用CAN命令实现自己的CANbus协议时，您可以在Trio CAN, DeviceNet, CANopen或用户配置之间进行选择。

对于大多数系统，CANIO_ADDRESS不需要从默认值32更改。

CANIO_ADDRESS的上字节用于设置额外的函数。例如，CANbus的波特率(以比特/秒为单位)和一些特殊的操作模式。某些特殊功能可能需要更新Trio CAN I/O模块的固件。

必须设置为32，才能连接Trio CAN I/O

以下运动控制器在Flash中不能设置CANIO_ADDRESS: MC4N-ECAT、MC4N-RTEX和MC664。必须使用MC_CONFIG。

固件2.0255及以下的运动控制器MC403(z)、MC405、Euro404和Euro408不支持第8位和第9位。

例

例1

在MC_CONFIG中，设置CANbus端口使用预定义的CANopen模式，带有2个从站节点，运行速度为250 kbps。

Canio_address = $100 + 41

例2

在MC_CONFIG中，将CANbus端口设置为125 kbps的DeviceNet从站。这使端口能够以正确的波特率联机(某些PLC DeviceNet主机需要)。

Canio_address = $200 + 33

在BASIC程序中，将DeviceNet从站设备初始化为以下状态:

  • 125 kbps
  • Mac ID = 3
  • 轮询数据从TABLE(20)开始
  • 轮询I/O输入数据= 2个字
  • 轮询I/O输出数据= 4个字
Devicenet (- 1,0,125, 3,20,2,4)
