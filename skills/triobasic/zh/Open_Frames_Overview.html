Open Frames概述

Trio控制器支持用户描述的Frame。本文档解释了在使用此功能进行开发时如何设置、使用和调试。您将需要Kinematics FEC来使用此功能。

设置一个Open Frame

创建一个BASIC函数库。为每个Frame定义2个函数。一个函数执行正向运动学，另一个执行逆向运动学。用户函数需要按以下方式命名:

     \{prefix\}_\{frame number\}
前缀描述
forward_transform正向运动学函数
Inverse_transform逆向运动学函数
Frame number限制:{1 - 999}除了任何已存在的Frame

例:用户编写的Frame 999，将定义以下函数

FUNCTION forward_transform_999(inpos AS TARGETJ) AS TARGET

RETURN retval
ENDFUNC

FUNCTION inverse_transform_999(inpos AS TARGET) AS TARGETJ

RETURN retval
ENDFUNC

函数定义

正向转换函数将接受数据类型为TARGET的参数并返回TARGETJ。

逆向转换函数将接受数据类型为TARGETJ的参数并返回TARGET。

TARGET是一种存储笛卡尔位置和方向的数据类型。TARGET变量仅供本地使用，可以像程序中的其他变量一样定义。
名字描述
X, y, z, u, v, w笛卡尔位置x、y、z (mm)，方向u、v、w(度)

例:

X = inpos.x
    y = inpos.y
    z = inpos.z
    w = inpos.w

TARGETJ是存储关节位置的数据类型。TARGETJ变量仅供本地使用，可以像程序中的其他变量一样定义。
名字描述
X, y, z, u, v, w关节位置x, y, z, u, v, w (mm或度，根据机器人类型要求)

     例:



        j1 = inpos.x<br>

j2 = inpos.y
     j3 = inpos.z
     j4 = inpos.u

错误处理

运行Frame时的任何RUN_ERROR都会导致frame被关闭。控制器将生成一个错误来表示这一点。

请注意，最常见的错误可能是“除零”或将无效值传递给“Arc”函数(ACOS, ATAN, ASIN等)。

建议在激活Frame之前，先使用FRAME_TRANS函数对Frame进行调试。

限制

  • 注意，当Frame处于活动状态时，断点将不起作用。因为打开的Frame是实时运行的，它的执行不能被“暂停”。
  • 这取决于Frame的开发人员，以确保输出不会计算不连续的输出。Frame返回的任何位置，都将被传递到驱动器。虽然会出现错误，但可能会防止任何“跳转”。一个足够大的值可以绕过这一点，并使您的机器以非常高的速度移动。
  • 不要在你的函数中包含无限循环。这将导致控制器挂起并重新启动。建议开发者监控SYSTEM_LOAD和SYSTEM_LOAD_MAX以跟踪它们的Frame消耗。

例

函数库

FUNCTION forward_transform_999(inpos AS TARGETJ) AS TARGET
    table_data = 100
    DIM retval AS TARGET
    link1 = TABLE(table_data + 0)
    link2 = TABLE(table_data + 1)
    link3 = TABLE(table_data + 2)
    rad_enc1 = TABLE(table_data + 3)
    rad_enc2 = TABLE(table_data + 4)
    mm_enc3 = TABLE(table_data + 5)
    rad_enc4 = TABLE(table_data + 6)
    VR(20) = input_pos.x

    j1 = inpos.x
    j2 = inpos.y
    j3 = inpos.z
    j4 = inpos.u

    y1 = link2 * SIN(j1)
    x1 = link2 * COS(j1)
    y2 = y1 + link3 * SIN(j2)
    x2 = x1 + link3 * COS(j2)
    z = 0'-j3 - ((j4 - j2) / (2 * PI)) + link1
    w = 0'j4 - j2 - j1
    VR(0) = x2
    VR(1) = y2
    VR(2) = z
    VR(3) = w
    VR(4) = RND(100)
    retval.x = 325 * FRAME_SCALE
    retval.y = 275 * FRAME_SCALE
    retval.z = z * FRAME_SCALE
    retval.w = w * FRAME_SCALE

    RETURN retval
ENDFUNC


FUNCTION inverse_transform_999(inpos AS TARGET) AS TARGETJ
    table_data = 100
    DIM retval AS TARGETJ
    link1 = TABLE(table_data + 0)
    link2 = TABLE(table_data + 1)
    link3 = TABLE(table_data + 2)
    rad_enc1 = TABLE(table_data + 3)
    rad_enc2 = TABLE(table_data + 4)
    mm_enc3 = TABLE(table_data + 5)
    rad_enc4 = TABLE(table_data + 6)
    ab = (link2*link2) - (link3*link3)
    lk1b = 2 * link2

    x = inpos.x
    y = inpos.y
    z = inpos.z
    w = inpos.w

    r2 = x*x + y*y
    r = SQR(r2)
    fi = ATAN2(y, x)
    beta = ACOS((ab + r2) / (lk1b*r))
    t1 = fi - beta ' 左手系
    t2 = ACOS((link2*link2 + link3*link3 - r2) / (2*link2*link3))
    t4 = 0'w
    t3 = 0'(-z - (t4/(2*PI)) + link1)

    VR(10) = t1
    VR(11) = t2
    VR(12) = t3
    VR(13) = t4
    VR(14) = RND(100)

    retval.x = t1 * rad_enc1
    retval.y = t2 * rad_enc2
    retval.z = t3 * mm_enc3
    retval.u = t4 * rad_enc4

    RETURN retval
ENDFUNC
