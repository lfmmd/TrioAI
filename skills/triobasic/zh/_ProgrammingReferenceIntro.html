TrioBASIC简介

介绍

TrioBASIC编程参考指南按字母顺序列出了运动控制器的MC4xx范围中使用的所有BASIC关键字。 TrioBASIC关键字可以是简单的参数，也可以是具有明确定义的功能的命令（例如FORWARD或HALT），而其他参数可能采用一个或多个影响命令操作的参数。

这篇简短的介绍旨在为使用主要编程参考提供指南。 在本手册的上下文中，它标识了概念以及一些具有特定含义的单词和短语。

关键字类型

根据关键字的功能，可以在何处使用关键字以及将它们存储在运动控制器中的位置将关键字分为几组， 关键字可以具有多个类型。 例如，关键字可以是系统变量，并且可以在MC_CONFIG初始化程序中使用。

下表描述了所有关键字类型。
轴命令发送到特定轴的命令。轴命令通常会在括号中包含一个或多个参数。它将在设置的BASE轴上操作，但它也可以使用AXIS修饰符关键字。例如:
MOVE(100)
REGIST(21, 4, 0, 1, 0) AXIS(15)
轴参数与特定轴相关联的参数。轴参数将对所设置的BASE轴进行操作，但它也可以使用AXIS修饰符关键字。例如:
P_gain = 1.2
x = MPOS AXIS(2)
仅限命令行命令或参数可以在Motion Perfect终端0的命令行中输入。它不能在可执行的TrioBASIC程序中使用。
常数关键字返回一个常量值。用于使常见的程序常量更具可读性。例如:
OP(10, ON)
WAIT UNTIL MARK = TRUE
FLASH该参数将自动存储在闪存中，因此在下一次和所有后续上电时可用。
注意，从命令行存储到Flash的参数不会在Motion Perfect项目中引用，必须单独记录。出于这个原因，建议使用MC_CONFIG，即使该参数也存储在Flash中。
数学函数该关键字是一个典型的TrioBASIC数学函数，它可以接受一个或多个操作数，并返回一个结果。例如:
x = COS(y)
value = ATAN2(VR(10)， VR(11))
MC_CONFIG该参数可在MC_CONFIG脚本中使用，该脚本在配置系统时上电时会自动运行。
修饰符modifier关键字用于修改命令发送到的目标轴、进程、端口或槽位，或者参数发送到或从中读取。例如:
CONNECT(1,3) AXIS(10)
x = PROC_STATUS PROC(21)
PRINT FPGA_VERSION SLOT(2)
进程参数它在多任务中给出进程的状态，或者如果写入该参数，它在多任务中具有某种控制功能。进程参数在进程0上操作，除非使用了PROC修饰符。
程序结构
槽参数插槽参数提供有关该插槽上硬件状态的一些信息。一些槽参数在写入时也具有控制功能。除非使用了SLOT修饰符，否则slot参数对slot 0进行操作。例如:
VR(10) = SERCOS_PHASE SLOT(2)
PRINT FPGA_VERSION SLOT(-1)
系统命令在系统固件或运动控制器硬件的一部分上操作的命令。一个系统命令可以有一个或多个参数包含在括号内。例如:
AUTORUN
SETCOM(19200,8,1,2,2,4)
系统参数与整个系统相关联的参数。系统参数可以控制或给出操作固件中某些东西的状态，也可以是特定于硬件的。例如:
NIO
TIME$

所有的函数和命令都接受一个表达式和一个变量。例如，一个有效的表达式可能是:

MOVE(COS(x) * VR(1) / 100)

关键字语法

TrioBASIC参考手册中的每个条目都以标准形式显示了关键字的语法。语法，即使用关键字的方式，在TrioBASIC中以四种方式出现。

命令

命令有两种类型;带参数的和不带参数的。下面是一个带有参数的命令的例子。

MHELICAL(end1, end2, centre1, centre2, direction, distance3 [，mode])

参数包含在圆括号(圆括号)内。如果有多个参数，则用逗号分隔。可选参数在方括号内的语法说明中显示。在程序中编写命令时不使用方括号，所以如果使用了可选参数，只需插入逗号和不带方括号的值或表达式即可。

不带参数的命令直接作为关键字输入，不带括号或括号，如:

FORWARD

函数

函数既可以接受一个或多个值，也可以返回一个值。赋给函数的值放在括号中，与命令的方式相同。可以将一个或多个值传递给函数。数学函数是这种语法类型的典型表达。

value = COS(expression)

value = ABS(expression)

参数

参数携带一个值，因此作用与变量相同。可以将值赋给参数，也可以从参数中读取值。有些参数是只读的。这将在关键字类型信息中显示。

参数语法的一些例子如下:

P_GAIN = 1.0

VR(10) = PROC_STATUS PROC(3)

IF MPOS AXIS(10) > (ENDMOVE AXIS(10) – 200) THEN

CANIO_ADDRESS = 40

常数

一些关键字被提供给程序员来使用公共常量。当然，这些都是只读的。出于语法的目的，常量可以被认为是参数类型的子集。下面的例子展示了常量PI, FALSE, TRUE, OFF和ON的使用:

circumference = PI * diameter

IF result = FALSE THEN

WHILE TRUE

OP(30, OFF)

bit3 = ON

变量

可以在表达式中使用或作为命令或函数中的参数的变量可以存储在易失性RAM、非易失性电池支持的RAM或非易失性闪存中。变量也可以是局部的或全局的。
局部变量局部变量被赋予一个用户定义的名称。名称可以包含字母、数字和下划线“_”字符。它可以是任意长度，但只使用前32个字符来标识唯一的变量名称。局部变量的值仅为定义它的进程所知。
局部变量是易失性的，会在关闭电源时丢失。例如:
elapsed_time = -TICKS / 1000
全局变量全局变量，也称为VR变量，保存在非易失性内存中。在MC464中，这是由锂电池维护的。在MC403/MC405中，全局变量存储在闪存中。全局变量可以从包括终端0中的命令行在内的所有进程中访问。
全局变量的数量是固定的。每个变量通过索引号访问，例如value = VR(123)。最高索引号见相关硬件手册。例如:
batch_size = VR(101)
TABLE值另一个全局可访问值的范围是TABLE内存。这是一个大的、有索引的变量数组，在某些命令中有特殊用途。它也可以被用作应用程序的通用内存。
表内存可以是易失性的也可以是非易失性的。详情请参阅相应的硬件手册。例如:
TABLE(100,1.2,2.3,4.5,6.8,9.0,15.4,23.7)

变量的语法

所有变量的默认数据类型都是双精度浮点型。不过，浮点数据类型也可以存储最大52位有符号整形。因此，所有变量和大多数参数都可以被当作整数来引用，而不需要创建单独的整形数据类型定义。

my_variable = 450.023 '十进制浮点数

my_variable = 450 '十进制整数

my_variable = $FF6A '十六进制整数

my_variable。5 = 1 '将位5设置为1

2012年中期以后发布的固件版本有更高级的数据类型可用。例如，STRING类型可以使用DIM语句来定义。更多信息请参见TrioBASIC参考手册中的DIM章节。

标签

标签是程序中的位置标记。标签是用户自定义的名称。名称可以包含字母、数字和下划线“_”字符。它可以是任意长度，但只使用前32个字符来标识唯一的变量名称。标签位置是通过在标签名称后面加上冒号“:”字符来定义的。然后可以在GOTO或GOSUB命令中引用包含标签的行。

start_of_program:

  raduis1 = 123

  GOSUB calc_circle_radius

  PRINT #5, area1

  WA(500)

GOTO start_of_program



calc_circle_area:

  area1 = PI * radius1 ^ 2

RETURN
