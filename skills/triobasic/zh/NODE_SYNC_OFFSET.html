NODE_SYNC_OFFSET

类型

系统数组(MC_CONFIG)

语法

NODE_SYNC_OFFSET(slot, node_address) = offset_value

描述

这个2D数组可用于设置网络上任何节点的EtherCAT 同步0偏移时间。这可用于定义应用于已定义节点的特定偏移量，使所有其他节点保持默认同步0偏移值。默认的同步0偏移量也可以使用ETHERCAT函数$91来定义。

数组是二维的，第一维是主站槽标识符，第二维是节点在主站网络内的位置。

值

以纳秒为单位的偏移值。这是一个带符号的值，是ETHERCAT函数$91设置的系统值或默认系统值-70000 ns(纳秒)的偏移量。

ETHERCAT函数$91以微秒(μs)为单位进行缩放。

在V2.0287及以下版本中，默认同步0偏移量为-70000 ns。在V2.0288及更高版本为0 ns。

例

例1

Linmot C1250驱动器要求同步0偏移量为200 μs。在同一个EtherCAT网络上还有其他不需要这个偏移量的驱动器。Linmot驱动器位于节点2、3、4和5。sync0的起始值为缺省值1000000 μs，因此系统同步0偏移量为0 ns。

默认的同步0偏移量可以在MPv4终端#0中使用ETHERCAT($92)进行检查

在MC_CONFIG

NODE_SYNC_OFFSET(0, 2) = 200000

NODE_SYNC_OFFSET(0, 3) = 200000

NODE_SYNC_OFFSET(0, 4) = 200000

NODE_SYNC_OFFSET(0, 5) = 200000

例2

一个网络在节点3包含3个Kollmorgen AKD驱动器和一个ABB Microflex e150。ABB驱动要求sync0偏移量为-70 μs。Kollmorgen驱动器不需要这种偏移。使用缺省的sync0起始值，因此系统sync0偏移量为0 ns。

默认的sync0偏移量可以在MPv4终端#0中使用ETHERCAT($92)进行检查

在MC_CONFIG

NODE_SYNC_OFFSET(0, 0) = 0

NODE_SYNC_OFFSET(0, 1) = 0

NODE_SYNC_OFFSET(0, 2) = 0

NODE_SYNC_OFFSET(0, 3) = -70000
