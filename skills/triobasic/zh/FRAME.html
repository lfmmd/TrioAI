FRAME

类型

轴参数

描述

FRAME是一种转换，当机器或机器人与该坐标系没有直接或一对一的机械连接时，它使用户能够在一个坐标系中进行编程。

FRAME命令选择在FRAME_GROUP中的轴上使用哪个转换。将FRAME应用到FRAME_GROUP中的一个轴上将会将该FRAME应用到该组中的所有轴上。为了使其与旧固件兼容，如果没有配置FRAME_GROUPs，则使用最低轴生成一个默认组，而不管FRAME参数是在哪个轴上发出的。

大多数转换需要配置数据来指定机械链接或操作模式的长度。这些数据存储在Table中，并在下面的参数列表中详细说明偏移量。这些Table位置通过FRAME_GROUP中的' table_offset '参数进行偏移。对于默认的FRAME_GROUP, table_offset为0。

在FRAME使能的情况下，不能修改FRAME TABLE参数。这可能会导致不可预测的移动，从而造成损坏或伤害。

Kinematic 功能启用代码是运行FRAME 14及更高版本所必需的

轴缩放

当FRAME启用时，UNITS将缩放应用到世界坐标系，AXIS_UNITS将缩放应用到轴坐标系。

当FRAME被启用时，MPOS按AXIS_UNITS缩放，当FRAME被禁用时，MPOS按UNITS缩放。

位置和跟随误差

当FRAME处于活动状态时，MPOS为电机位置，DPOS为世界坐标。可以读取AXIS_DPOS来查找电机坐标系中的给定位置。

MPOS与AXIS_DPOS之间计算跟随误差，电机的跟随误差也是如此。

当使用多FRAME时，或者如果您希望对轴进行分组，您可以使用DISABLE_GROUP，以便一个轴上的MOTION_ERROR不会影响所有轴。

硬件和软件限位

由于FS_LIMIT和RS_LIMIT使用DPOS，因此它们在世界坐标系中均处于活动状态。VOLUME_LIMIT也使用DPOS，所以也在世界坐标系统中。FWD_IN和REV_IN,AXIS_FS_LIMIT和AXIS_RS_LIMIT使用AXIS_DPOS作为电机的正反向限位。

当移出FWD_IN和AXIS_FS_LIMIT时，电机必须反向移动。由于FRAME变换，这可能不是世界坐标系中的反向运动。当离开REV_IN和AXIS_RS_LIMIT时，电机必须正向移动。由于FRAME变换，这可能不是世界坐标系中的正向运动。

补偿位置

当一个FRAME被启用时，OFFPOS和DEFPOS不能被使用，因为它们会导致DPOS和MPOS的跳变。由于转换使用这些命令将DPOS和MPOS分开，将导致电机位置发生不希望的跳跃。。

REP_DIST也会导致DPOS和MPOS的跳变，所以当使用FRAME时，位置必须永远不会到达REP_DIST。REP_OPTION必须设置为0,REP_DIST必须至少是系统上最大移动的两倍。

当DATUM完成时，它也会导致DPOS和MPOS的跳变，所以当FRAME <> 0时绝不能使用DATUM

您可以使用USER_FRAME定义一个不同的原点来进行编程。

上电序列和回零

一些FRAME转换需要在启用FRAME之前将机器回零和/或移动到某个位置。这可以使用DATUM函数来完成。如果原始位置不是FRAME的零位置，则可以在启用FRAME之前使用DEFPOS/ OFFPOS来设置正确的偏移量。

当一个FRAME被启用时，DPOS被调整为从当前AXIS_DPOS计算出来的世界坐标。

当FRAME启用时，不应该执行DATUM回零，因为这会改变DPOS，从而可能导致不良运动。如果需要在启用FRAME时执行回零，可以移动到Registration位置，然后使用USER_FRAME应用偏移量。

FRAME= 1,2轴SCARA

描述

Frame=1允许用户在X, Y，笛卡尔坐标系中为2轴SCARA手臂编程，如下例所示。FRAME允许SCARA的2种配置，这取决于第二轴电机在关节或在基座。不同之处在于，角度t2是从链接1引用的，或者t2是从基座引用的。通常使用连杆或皮带来保持t2与基座的引用。

第二个电机装在连杆1的末端，t2相对于连杆1。

第二电机在基座上用连杆臂移动上部，t2相对于基座。

一旦FRAME启用DPOS以微米为单位测量，UNITS可以设置为方便的范围。

FRAME=2, XY单皮带

描述

切换到FRAME=2将允许使用单皮带配置X-Y运动。在这种模式下，move(0,100)的插补移动会在电机1和电机2上产生运动，以根据转换后的位置垂直提升负载。注意，这两个电机位于x轴上。在这种配置中，y轴的质量可以最小化。X轴和Y轴的变换位置方程如下:

Xtransformed = ([MPOS](MPOS.md) [AXIS](AXIS.md)(0)+ [MPOS](MPOS.md) [AXIS](AXIS.md)(1))\*0.5



Ytransformed = ([MPOS](MPOS.md) [AXIS](AXIS.md)(0)- [MPOS](MPOS.md) [AXIS](AXIS.md)(1))\*0.5

变换后的X-Y坐标由AXIS (0)和AXIS(1)测量的编码器位置(MPOS)导出。当FRAME=2时，这种转换由运动控制器自动完成。

一旦FRAME启用，DPOS以编码器脉冲测量，UNITS可以设置为方便的范围。

FRAME=5, 2轴旋转

描述

这个frame被设计成允许两个正交轴通过一个角度被“转动”，这样对x、y(沿着所需平面)的指令输入就被转换为固定轴x’和y’。

这个变换是通过一个2 × 2的矩阵来完成的，它的系数可以很容易地从操作平面的所需旋转角度推导出来。

计算矩阵系数

为了使FRAME工作，必须输入两组矩阵系数，一组用于正变换，另一组用于逆变换。变换按照以下方式计算x和y:

(x', y') = (x, y) * TABLE(0), TABLE(1)

                    TABLE(2), TABLE(3)

逆变换计算为:

(x, y) = (x', y') * TABLE(4), TABLE(5)

                    TABLE(6), TABLE(7)

回零

坐标轴应在FRAME=0下回零。一旦这样做了，那么FRAME可以被设置为5，运动命令以常规方式指向任何一个轴或两个轴。然而，x '和y '(真正的轴)的实际移动将依据变换值。

如果轴需要根据实际轴重新定位，只需设置FRAME =0即可关闭FRAME。当这样做时，DPOS值将改变为与MPOS位置相同，即它们成为x ' / y '平面中的位置。然后可以将轴移动到一个新的起始位置，并将FRAME设置回5，可能还会设置一个新的角度。

FRAME=10，笛卡尔到极坐标变换

描述

这个FRAME转换允许用户在一个以极(半径，角度)坐标系移动的系统上以笛卡尔(X,Y)坐标进行编程。这通常用于圆柱形机器人，你需要对手臂的延伸(半径)和角度进行编程。可以简单地加上垂直的Z轴，做成一个3个自由度的系统。

一旦FRAME被启用，原始位置数据(UNITS=1)在X轴和Y轴上缩放相同，分辨率从半径轴设置。然后可以将UNITS设置为合适的范围。原点是极坐标系的中心。

组中的第一个控制半径方向，第二个轴控制旋转。

回零

在启用FRAME=10之前，坐标轴必须回零，使它们处于已知位置。当FRAME被启用时，X和Y位置从当前的极坐标位置计算。

在执行靠近原点的移动时要小心。通过原点的移动将需要无穷大的速度和加速度。这通常是不可能实现的，并且轴将由于过大的跟随误差断开。

参数

1旋转脉冲，由固件设置
2前一个伺服周期的角度，由固件设定

例

FRAME = 116,3轴机器人带有2轴腕部

描述

FRAME 116变换允许具有2轴手腕的XYZ机器人轻松编程。当两个腕轴旋转时，转换函数在XYZ中提供补偿。

FRAME=116要求运动学runtime FEC

启用FRAME后，将UNITS设置为FRAME_ANGLE_SCALE，以便笛卡尔移动使用与table数据中使用的相同的比例。因此，如果TABLE数据以毫米为单位编程，那么当UNITS设置为FRAME_ANGLE_SCALE时，机器人可以以毫米为单位编程。

将旋转(手腕)轴上的UNITS设置为FRAME_ANGLE_SCALE，以便它们以弧度编程。当然，您可以将所有轴的UNITS设置为任何合适的比例。

回零

两个腕轴必须记录到正确的零位置，以便FRAME 116转换操作。变换不使用XYZ轴的零位置。

C轴上的零位置(绕Z轴旋转)是当偏移臂与X轴在一条线上时。下图是从上方向下看X-Y平面绘制的。

B轴上的零位置(绕Y旋转)是当偏移臂是图中所示的“直线向下”位置时。

所有5个轴上的运动方向必须与框图相匹配，FRAME 116变换才能运行。

如果一个轴的运动方向被取反，它可以被取反:

利用伺服/步进驱动器的功能来取反运动方向

在脉冲方向轴上使用运动控制器内的STEP_RATIO函数

在闭环伺服轴上使用ENCODER_RATIO / DAC_SCALE函数在运动控制器内

回零：

轴(0)-基础旋转

正方向是从上方顺时针看

轴(1)-手臂伸展

将手臂伸到最短位置。使用DEFPOS来定义从旋转中心到手腕的偏移量

正方向是远离中心移动

范围:大于零

手臂伸展绝不允许变为零或负值，因为这会导致运动中的跳变。你可以设置你的RS_LIMIT来防止这种情况。

正方向是往下运动

范围:0到无穷大

回零使手腕水平

范围:-无穷大至无穷大
