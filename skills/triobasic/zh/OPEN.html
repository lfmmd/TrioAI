OPEN

类型

文件函数

语法

value = OPEN #channel AS "[location:]name" FOR access

描述

OPEN提供对控制器上文件的访问。文本文件可以初始化为_Motion_ Perfect可以同步的文件，临时文件，SD卡上的文件或FIFO缓冲区。所有文件都在控制器文件目录中，但是只有一个文本文件可以在_Motion_ Perfect中查看或编辑。

一旦文件被打开，它就可以被标准的TrioBASIC通道命令操纵。如果文件是用读访问打开的，那么任何TrioBASIC GET类型的命令，如GET、INPUT、LINPUT和KEY都可以在通道上使用。如果文件是用写访问打开的，那么可以在通道上使用PRINT类型命令。

使用完通道后，应该使用TrioBASIC CLOSE命令关闭通道。

参数

value:在文件打开操作期间发生的TrioBASIC错误。如果没有错误，则值为0。
channel:要与文件关联的TrioBASIC #通道。通道范围取决于文件的类型:
对于文件和字符串变量为40到48。
ARM基础控制器上的TCP和UDP多客户端连接为20 ~ 29。(固件版本2.0256起)。
location:指定以下“name”所代表的内容:
文件和字符串变量根据指定的OUTPUT(mode)(见下文)存储在控制器文件系统中的文件
RAM存储在控制器RAM中的文件(易失性)
SD存储在控制器sd卡上的文件(非易失性)
USB1

或

USB2		存储在USB闪存上的文件，插入控制器的一个USB插座(仅限Flex-7，非易失性)。
字符串字符串变量
协议TCPTCP连接的IP地址和端口号
DGRAMUDP连接的IP地址和端口号
TCPSERVERTCP服务器端口号
name:对于文件和字符串变量(通道40到48)，这是要打开的文件的名称，例如“data_points”、“SD:process_log”、“RAM:temp_data”或“string:my_var”。
对于TCP和DGRAM连接(通道20 ~ 29)，IP地址用点表示，端口号用括号括起来，例如:“TCP:192.168.8.152(3000)”或“DGRAM:192.168.0.250(1000)”。
对于TCP服务器(通道20 ~ 29)，只需要输入端口号，用括号括起来，例如“TCPSERVER:(790)”。
access:允许对文件进行的操作。
INPUT文件将被打开以供阅读。当到达文件末尾时，KEY将返回FALSE，并且GET和INPUT函数将失败。
OUTPUT(mode)文件将被打开用于写入。如果该文件不存在，则创建该文件。如果文件已经存在，那么它将被清除。
模式函数
0打开非易失性文本文件，Motion Perfect可以读取，编辑并保存到项目中。 该文件与其余的Motion Perfect文件一起与PC目录同步。
1打开一个只能由控制器访问的非易失性文本文件。 该文件未同步，但是可以通过Motion Perfect将其添加到项目中。
2打开一个只能由控制器访问的临时文件(RAM)。
3打开一个只能由控制器访问的非易失性文本文件。
READ_WRITE通道将为读/写操作打开。
APPEND文件将被打开以供写入。该文件必须存在。所有的数据都被写到文件的末尾。
FIFO_READ[(mode[, terminator])]该文件将被打开以供读取，并作为循环缓冲区进行管理。这只对存储在内部RAM中的文件有效。
Mode默认为零。如果mode为0，则KEY命令将返回FIFO中的字符数。如果mode为1，则KEY命令将返回FIFO中的行数。
终端默认为13行(回车)。如果mode为1，并且该字符被添加到FIFO中，则行计数将增加1。如果mode为1，并且该字符从FIFO中删除，则行计数减少1。
FIFO_WRITE(size[， mode[， terminator]])该文件将打开以供写入，并将作为循环缓冲区进行管理。这只对内部RAM中的文件有效。如果文件不存在，它将被创建(大小)字节长(空闲内存允许)。注意，创建的最小大小将是256字节。
如果文件确实存在，那么它必须是FIFO类型，大小参数被忽略，内容被清除。
如果模式位0被设置，那么FIFO将在行模式下打开。如果设置了模式位1，则FIFO将以非阻塞模式打开，这将允许FIFO覆盖。如果FIFO处于非阻塞和行模式，那么当一行溢出时，它将清除下一行，直到结束符字符。
如果FIFO处于行模式，则终止符包含行结束字符。
OUTPUT_PROGRAM_TYPE(type)在RAM中打开一个文件，用显式的程序类型输出。

如果要在SD卡上创建文件，则需要附加文件扩展名。存储在控制器内存中的文本文件将被Motion Perfect保存为项目中的.txt文件。这使您能够以任何基于文本的格式生成和读取SD卡上的文件。

如果你正在写一个Motion Perfect可以读取的文本文件，那么要注意，在你执行项目检查之前，Motion Perfect不会看到更改。在连接到Motion perfect时写入文本文件时要非常小心。如果需要在连接到Motion perfect时写入文件，建议使用临时文件或SD卡上的文件。

与OUTPUT_PROGRAM_TYPE一起使用的类型列表
type:0 => Trio BASIC
1 => Trio BASIC 编码
2 =>G代码
3 =>文本
4 => IEC
5 => fifo
6 => temp
7 => project
8 => binary
9 => Mc_config
10 => IEC 库
11 => HMI
12 => Trio BASIC Library
13 => Trio BASIC Library 编码
14 => HMI库
15 => 易失文本(控制器重启不保留)

例

例1

打开一个可用于将信息记录到SD卡上的.txt文件的文件，然后将移位结束信息打印到该文件中。

IF (OPEN #40 AS "SD:product_log.txt" FOR OUTPUT(0)) = 0 THEN

  PRINT #40, DATE$ 'Print the date

  PRINT #40, products_complete[0]; " products completed"

  PRINT #40, product_failures[0]; " products failed"

  CLOSE #40

ELSE

    PRINT "Error opening product log"

ENDIF

例2

G-Code文件从串行端口加载到控制器中，它被保存到控制器上的临时文件中供以后使用。

OPEN #41 AS "gcodeprogram" FOR OUTPUT(1)

WHILE file_downloading

  IF KEY #1

    GET #1, char

    PRINT #41, char;

  ENDIF

  Length = length + 1

WEND

CLOSE #41

例3

G-Code程序已下载到临时文件中;然后它应该被转移到FIFO，这样它就可以被解释为运动。

OPEN #41 AS "gcodeprogram" FOR INPUT

OPEN #42 AS "gcodefifo" FOR FIFO_WRITE(length)



WHILE KEY #41

  GET #41, char

  PRINT #42, char;

WEND



CLOSE #42

CLOSE #41

例4

读取从远端对等体接收到的UDP报文。

OPEN #20 AS "DGRAM:192.168.0.250(1000)" FOR READ_WRITE

CHANNEL_READ(20, strInput)

CLOSE #20

例6

作为TCP服务器打开一个连接，等待客户端通过TCP/IP端口790进行连接。一旦连接，服务器发送一个消息回客户端，其中包括连接时间，VR(0)值和编码器轴0值。

当客户端发送“exit”消息时，连接随即关闭。

DIM time_to_leave AS INTEGER

DIM server_message AS STRING(512)

DIM tcpserver_channel AS INTEGER

DIM tcp_port AS INTEGER

DIM tcpserver_id AS STRING(50)

DIM local AS STRING(20)



tcpserver_channel = 25

tcp_port = 790

time_to_leave = 0

tcpserver_id = "TCPSERVER:(" + STR(tcp_port, 0) + ")"

OPEN #tcpserver_channel AS tcpserver_id FOR READ_WRITE

PRINT #5, "Awaiting connection for " + tcpserver_id



REPEAT

  IF KEY #tcpserver_channel THEN

    CHANNEL_READ(tcpserver_channel , server_message, CHR(10))



    'We can also write back to the client. Here we send back what we received

    local = STR(VR(0), 0)

    CHANNEL_WRITE(tcpserver_channel, " " + TIME$ + " " + STR(VR(0), 0) + " " + STR(ENCODER AXIS(0)))

    PRINT #5, "Received: " + server_message



    IF LEFT(server_message, 4) = "exit" THEN

      PRINT #0, "EXITING " + tcpserver_id

      CLOSE #tcpserver_channel

      time_to_leave = 1

    ENDIF

  ENDIF

UNTIL time_to_leave > 0

例7

作为客户端，在790端口连接到服务器，并向服务器发送消息

DIM time_to_leave AS INTEGER

DIM server_message AS STRING(512)

DIM tcpserver_channel AS INTEGER

DIM tcp_port AS INTEGER

DIM tcpserver_id AS STRING(50)

DIM tcpserver_ip_address AS STRING(15)



tcpserver_channel = 25

tcp_port = 790

tcpserver_ip_address = "192.168.0.250"



tcpserver_id = "TCP:" + tcpserver_ip_address + "(" + STR(tcp_port, 0) + ")"



OPEN #tcpserver_channel AS tcpserver_id FOR READ_WRITE

PRINT #5, "Connecting to server:  " + tcpserver_id



WA(20)



PRINT #tcpserver_channel, "Hello, server!"; CHR($0D); '$0d = Carriage return



REPEAT

    time_to_leave = VR(0) '评估一下是否该退出



    WHILE KEY #tcpserver_channel

        GET #tcpserver_channel, k

        PRINT #5, CHR(k); '从服务器打印答案到终端#5

    WEND    



    IF VR(0) = 1 THEN

        PRINT #tcpserver_channel, "exit"; CHR($0D); '$0d = Carriage return

    ENDIF

UNTIL time_to_leave > 0



CLOSE #tcpserver_channel

PRINT #5, "Communication is closed."

例8

将输出发送到字符串变量，这在输出需要扫描时很有用。

DIM my_var AS STRING(1000)

OPEN #41 AS "STRING:my_var" FOR OUTPUT(0)

PRINT #41, "Pi = ", PI

CLOSE #41



PRINT #5, my_var

将在终端5上显示“Pi = 3.1416”。
