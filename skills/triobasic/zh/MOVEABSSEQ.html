MOVEABSSEQ

类型

轴命令

语法

MOVEABSSEQ(table pointer, axes, npoints, options, radius [, output [, transition_angle]])

描述

MOVEABSSEQ命令允许通过TABLE值加载一系列移动。这些动作可以使用圆形或球形弧线自动合并在一起。2到6个轴上的移动可以用这种方式控制。

MOVEABSSEQ作为MOVEABS->MOVECIRC->移动(如果指定2个轴)和MOVEABS->MSPHERICAL->(如果指定3个或更多轴)的序列加载到控制器移动缓冲区中。如果弧线混合在一起，则可以省略线性移动。如果“选项”位0设置为1，则加载的移动序列将是指定2轴时MOVEABSSP->MOVECIRCSP->移动序列，如果指定3+轴时MOVEABSSP->MSPHERICALSP->移动序列。

如果设置了选项位0，则必须为每个表点提供4个额外的速度控制值。

MOVE_COUNT在每次加载时递增。这个参数可以由用户程序设置。

参数

Table pointer:绝对点在TABLE内存中的位置。
Axes:轴数2 ~ 6。
npoints:点数，每个点数需要2或3个表值。
Options:位00序列加载MOVEABS、MOVECIRC等。
1序列将加载嵌入的速度移动MOVEABSSP, MOVECIRCSP等。
位1保留
位20加载一系列标准的移动类型..
1加载MOVEABSSEQ/MOVEABSSEQSP移动类型序列。
Bits 8…13设置为指定除“Axes”以外的数据点之间的偏移量。
Radius:要应用的合并/圆角半径。0表示不进行倒圆角。
Output:可选的TABLE数据输出。0表示不向TABLE输出数据。否则值为该数据的TABLE起始索引。
Transition angle:生成的任何圆角弧线的可选过渡角(以弧度为单位)规格。如果不应用过渡曲线，设置为0。

如果指定的点相距不够远，无法应用圆角，圆角半径将自动减小到可能的最大值。如果设置了选项的第2位，则不会应用圆角。

MOVEABSSEQ开始处的当前轴位置用于计算第一个圆角。

例

例1

使用单独的MOVE和MOVECIRC绘制O(参见Trio Manual MOVECIRC)，并使用MOVEABSSEQ绘制类似的O。

'MOVE和MOVECIRC:

MOVE(0, 60) ' move A -> B 

MOVECIRC(30, 30, 30, 0, 1) ' 运动从 B -> C 

MOVE(20, 0) ' move C -> D 

MOVECIRC(30, -30, 0, -30, 1)' 运动从 D -> E 

MOVE(0, -60) ' move E -> F 

MOVECIRC(-30, -30, -30, 0, 1)' 运动从 F -> G 

MOVE(-20, 0) ' move G -> H 

MOVECIRC(-30, 30, 0, 30, 1) ' 运动从 H -> A 

WAIT IDLE 

DEFPOS(100, 30) 

WAIT UNTIL OFFPOS = 0 



'MOVEABSSEQ: 

TABLE(1000, 100, 120) 

TABLE(1002, 180, 120) 

TABLE(1004, 180, 0) 

TABLE(1006, 100, 0) 

TABLE(1008, 100, 30)



MOVEABSSEQ (1000,2,5,0,30)

例2

如果“options”位0被设置，MOVEABSSEQ命令将加载嵌入的速度移动类型。每个位置加载四个附加参数，以控制加载移动的速度和潜在的后续圆角弧:

0 - FORCE_SPEED线性移动

1 - ENDMOVE_SPEED线性移动

2 - FORCE_SPEED圆角弧

3 - ENDMOVE_SPEED圆角弧

即使不使用圆角，也必须提供潜在圆角弧的速度参数。因此，对于2轴程序，需要6个参数。对于3轴程序，必须输入7个参数。

UNITS AXIS(0) = 1000

UNITS AXIS(1) = 1000



BASE(0, 1)

MERGE = ON

trans = 0

SPEED = 2500

ACCEL = 100000

DECEL = 100000

lin_sp = SPEED

lin_end = SPEED * 0.5

arc_sp = SPEED * 0.5

arc_end = SPEED * 0.5



TABLE(1020, 0, 0, lin_sp, lin_end, arc_sp, arc_end)

TABLE(1026, 180, 0, lin_sp, lin_end, arc_sp, arc_end)

TABLE(1032, 360, 0, lin_sp, lin_end, arc_sp, arc_end)

TABLE(1038, 360, 360, lin_sp, lin_end, arc_sp, arc_end)

TABLE(1044, 0, 360, lin_sp, lin_end, arc_sp, arc_end)

TABLE(1050, 0, 0, lin_sp, lin_end, arc_sp, arc_end)

TABLE(1056, -180, 0, lin_sp, lin_end, arc_sp, arc_end)



MOVEABSSEQ(1020, 2, 7, 1, 80, 0, trans * PI / 180)

例3

当在短时间内处理了非常大量的移动时，从大量的数据点创建一个单一的移动可能会产生速度轮廓的问题:

- Trio 运动控制器在每个伺服周期中每个轴只能加载一个动作。

-在非常短的时间内，速度曲线可以开始降低速度，以便以编程的减速度达到当前目标终点位置。

为了避免这些问题，可以单步加载整个点序列。 要执行这种类型的移动，必须设置move命令的参数：

-“options”参数必须设置位2。

-“output”数据区必须定义为10个值* n点的空间。

'这个例子展示了坐标轴以恒定的路径速度沿着椭圆运动。

'假设一个椭圆被定义为间隔1度的点的序列:



UNITS AXIS(0) = 1000

UNITS AXIS(1) = 1000 



SPEED = 500

ACCEL = 50000

DECEL = 50000

SRAMP=30 

DEFPOS(100,0) 

a = 100 

b = 50 

FOR d = 0 TO 360 

    x = COS(d * PI / 180) * a 

    y = SIN(d * PI / 180) * b 

    TABLE(1000 + d * 2, x, y) 

NEXT d 



MOVEABSSEQ(1000, 2, 361, 4, 0, 2000)

注意，本例中MOVEABSSEQ移动类型在TABLE(2000) .. TABLE(5610)中建立了一个数据值表，使移动能够被处理。
