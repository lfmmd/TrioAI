CANIO_BASE

类型

I/O参数(MC_CONFIG)

描述

该参数设置任意CAN模块I/O通道的起始地址。与MODULEIO_BASE, DRIVEIO_BASE和NODE_IO一起，I/O分配方案可以替换和扩展MODULE_IO_MODE的操作，但是如果MODULE_IO_MODE的值被更改为2 (CANIO后跟MODULEIO)，则优先考虑。

值

-1没有效果(应该使用CANIO_ADDRESS禁用CANIO)
0自动分配CAN I/O(默认)
> = 8CAN I/O位于这个I/O点地址，截断到最近的8的倍数

例

具有MC464、松下模块(槽0)和CANIO模块的系统将具有以下I/O分配:

CANIO_BASE =0 + DRIVEIO_BASE =0 + MODULEIO_BASE=0
0-7内置输入
8-15内置双向I/O
16-23松下模块输入
24-39CANIO双向I/O
40-47松下驱动输入
48-1023虚拟I / O

CANIO_BASE = 100 +DRIVEIO_BASE= 0 + MODULEIO_BASE= 0
0-7内置输入
8-15内置双向I/O
16-23松下模块输入
24-31松下驱动输入
32 - 95虚拟I / O
96 - 103CANIO双向I/O
104 - 1023虚拟I / O
