PROFINET

类型

系统命令

语法

PROFINET(function, slot[， parameters…])

描述

PROFINET命令用于配置以Trio控制器为从站的PROFINET通信。

TRIO控制器固件中的PROFINET堆栈在上电时不会自动启动。要打开通信通道，请使用STARTUP程序中的功能1或任何配置为上电后运行的程序。

参数

function:要执行的功能
0显示或定义模块映射
1启动Profinet协议
2停止Profinet协议
8设置Profinet设备名称
slot:-1
Parameters:附加参数设置
slot模块槽位号(-1)
pn_api应用进程标识符，目前只支持0。
pn_slot用户槽位(1..63)(槽位0为接口)
pn_subslot1 . . 7
datasource-1:删除
0:VR
1: TABLE
index数据源中的起始位置(即基准地址)
type通过总线发送的数据的大小和类型

Function = 0:显示或定义模块映射

该命令有几个与模块映射的显示和配置有关的函数。

当只使用2个参数时，它将显示给定槽位的模块映射。例如: PROFINET(0, -1)

当使用3个参数时，具有有效槽位号且数据源为-1，则会重置整个模块映射。例如: PROFINET(0, -1, -1)

当使用11个参数时，他们会配置模块图中给定槽位的所有参数。

Function = 1:启动Profinet协议

该命令启动profinet协议。

Function = 2:停止Profinet协议运行

该命令用于停止profinet协议。

Function = 8:设置Profinet设备名称

此命令设置Trio控制器的profinet名称。

例

例1

设置2个输入模块和2个输出模块，对vr进行读写。

>>profinet(0, -1, 0, 1, 1, 0, 10, 2, 1, 0, 1) '输入模块插槽1

>>profinet(0, -1, 0, 2, 1, 0, 11, 2, 1, 0, 1) '输入模块插槽2

>>profinet(0, -1, 0, 3, 1, 0, 100, 2, 1, 1, 1) '输出模块插槽3

>>profinet(0, -1, 0, 4, 1, 0, 101, 2, 1, 1, 1) '输出模块插槽4

>>profinet(0, -1) '显示模块映射

0 1 (1) : 0 0 10 2 1 0 1

0 2 (1) : 0 0 11 2 1 0 1

0 3 (1) : 0 0 100 2 1 1 1

0 4 (1) : 0 0 101 2 1 1 1

例2

停止profinet协议。

>> profinet(2, -1)'停止profinet
