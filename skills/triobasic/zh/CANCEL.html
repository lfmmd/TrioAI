CANCEL

类型

轴命令

语法

CANCEL([mode])

替代格式

CA([mode])

描述

用于取消轴或插补轴组上的当前或缓冲轴命令。

例如，运动的速度轮廓会逐渐下降，然后逐渐终止，例如：FORWARD、REVERSE、MOVE、MOVEABS、MOVECIRC、MHELICAL、MOVEMODIFY，，而其他移动类型则会立即终止。

如果指定取消当前移动(取消模式0和2)将以FASTDEC减速，否则以DECEL减速。

取消缓冲的移动(那些将在当前移动完成后处理的移动)将导致当前移动在其目标位置正常终止(即以DECEL的减速，就好像缓冲的移动从未加载过一样)，除非在当前移动完成之前没有足够的空间来逐渐停止，在这种情况下，运动将在取消的移动中逐渐停止，之后它们将从移动缓冲区中卸载。

取消最后一个缓冲的移动将导致它立即从移动缓冲区中移除，除非它的距离需要在运动缓冲区中静止，之后它将从运动缓冲区中卸载。如果没有被缓冲的运动，当前的运动将被取消。

CANCEL可以手动调用，也可以通过软件限制、硬件限制和MOTION_ERRORs自动调用。

参数

mode:0 (default)取消当前正在执行的移动，由MTYPE表示
1取消在基准轴上NTYPE区所有缓冲移动(不包括PMOVE)
2取消所有进行的和缓冲区的运动，包括PMOVE(如果在BASE轴上加载)
3终止当前的移动而不先停止以允许后续移动合并
4终止最后一次在基轴上的缓冲移动
5取消MOVEADD附加移动

CANCEL(0)只会取消当前正在执行的移动。如果新的运动进入缓冲区，那么它们将被加载，因此移动不会停止。

例

例1

按编程的SPEED向前移动基轴，等待10秒，然后减速，按编程的DECEL速率停止轴。

FORWARD

WA(10000)

CANCEL '10秒后停止移动(与CANCEL(0)相同)

例2

飞行剪切机使用一系列movelink使基轴跟随轴4上的参考编码器。当剪切返回到顶部位置时，一个输入被触发，这将移除缓冲的MOVELINK，并替换为减速的MOVELINK使得从轴逐渐减速(基准轴)。

ref_axis = 4

REPEAT

  MOVELINK(100, 100, 0, 0, ref_axis)

  WAIT LOADED '确保NTYPE缓冲区每次都是空的

UNTIL IN(5) = ON

CANCEL(1) '取消NTYPE缓冲区中的MOVELINK

MOVELINK(100, 200, 0, 200, ref_axis) '逐渐减速

CANCEL '取消MOVELINK

例3

两个轴以1:2的比例连接。0轴在1秒后取消，然后1轴在速度下降到指定水平时取消(本例中为7500)。在第一个CANCEL之后，轴1将以DECEL速率减速，直到轴1的CONNECT被取消，使其立即停止。

BASE(0)

SPEED = 10000

FORWARD

CONNECT(0.5, 0) AXIS(1)

WA(1000)

CANCEL

WAIT UNTIL VP_SPEED <= 7500

CANCEL AXIS(1)
