GTAE

类型

机器人参数

语法

GTAE(index) = x, y, z, u, v, w, name, ptype, rnum, rconf, revs, of, tool, rf, userdata

描述

GTAE是GTA(全局目标阵列)和GTAJ(全局目标阵列-关节)的扩展版本，其中包含用于机器人应用的全局可用位置的扩展阵列。

与GTA和GTAJ一样，GTAE对所有程序都是全局的，它们的值被保留。对于局部目标，请参见TARGETE。

可以使用“点”表示法(“.”)对目标的元素进行读写，如下例所示。

例:

Axis4_rev = GTAE(1).revs

GTAE需要RPS run-time FEC。

GTA、GTAJ和GTAE共享相同的内存分配位置。这意味着它们不能共享索引。目标的最大数量取决于控制器。

例:
GTA(1) =  X1, Y1,  Z1, U1, V1, W1, “pt1”

GTAE(2) = 154.694, -440.071, -27.495, 180, 0, 200, "test", 0, 0, 1, 0, 0, 0, 0, 69.420

在本例中，GTAE不能有索引1,GTA不能有索引2。

参数

名称描述
x, y, z, u, v , w: Float位置(毫米)和方向(度)在笛卡儿坐标。
(optional) “name”: String目标的唯一名称。重复的名称会产生编译错误。
ptype: Integer点类型。
0 =笛卡儿坐标，1 =关节，2+ =用户定义。
rnum: Integer用于目标的机器人编号。
rconf: Integer目标的机器人配置设置。
revs: Integer取目标时从校准点算起的最后一个轴旋转圈数值。
of: Integer用于目标的对象框架。
tool: Integer用于目标的工具偏移量。
rf: Integer用于目标的机器人框架。
userdata: Float任何有用的数值都可以存储在这里。

名称是可选的，但如果指定必须是唯一的(不被任何其他GTA元素使用)。

不能使用名称声明GTAE;相反，它必须使用id号。如果是这样，将抛出运行错误“ID未定义”。

例

例1

定义一个名为“p1”的扩展全局目标。

GTAE(1011) = 100, 0, 50, 180, 0, 180， "p1"， 0, 0, 1, 10, 0, 0, 0, 69.420
