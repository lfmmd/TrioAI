READPACKET

类型

命令

语法

READPACKET(port, variable, count [,format])

描述

READPACKET用于通过串行通信端口将数据读入VR变量。数据以二进制格式从PC传输，带有16位校验和。CRC有四种不同的数据格式，都使用相同的数据包结构:
数据CRC
字节0字节1字节2…字节n字节0字节1

16位校验和使用生成器多项式:
x16+x15+x2+x0或$8005

参数

port:这个值应该是0到2
variable:这个值告诉_运动控制器_从哪里开始设置VR全局内存数组中的变量。
VR count:要下载的变量数，最多250个
format:正在下载的号码格式
0标准字符
1标准整形
2标准长整形
47位长

根据使用的格式，数据可能被分割成多个字节。这取决于用户重新组合这些以获得最终值。

Format = 0(标准字符)

每个值在每个字节中:

Value0 = Byte 0
Value1 = Byte 1
…

Format = 1(标准整数)

每个值拆分为2Bytes:

Value0 = Byte1 * 256 + Byte0
Value1 = Byte3 * 256 + Byte2
…

Format = 2(标准长整形)

每个值被分成4Bytes

Value0 = ((Byte3 * 256 + Byte2) * 256 + Byte1) * 256 +Byte0
Value1 = ((Byte7 * 256 + Byte6) * 256 + Byte5) * 256 +Byte4
…

Format = 4(7位长)

每个值被分成4Bytes，但每个字节只使用7位。CRC只有字节0(包括)设置了第7位。因此发送的值长度为24位。

位15和位7 CRC没有发送，因此被检查忽略。

Value0 = ((Byte3 * 128 + Byte2) * 128 + Byte1) * 128 + Byte0
Value1 = ((Byte7 * 128 + Byte6) * 128 + Byte5) * 128 + Byte4
…

例

使用标准长整形(格式= 2)从端口1开始将值读入VR序列0。来自READPACKET命令的字节存储在VR(100)中。

READPACKET(1, 100, 10, 2)
