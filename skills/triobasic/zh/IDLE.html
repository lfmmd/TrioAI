IDLE

类型

轴参数

描述

IDLE是一个轴参数，用于检查轴是否正在进行移动，或者是否所有移动都已完成。如果在运动缓冲区MTYPE中没有移动，并且在下一个NTYPE移动缓冲区中没有移动，则轴处于空闲状态。IDLE可以单独用作WAIT IDLE，也可以用作使用WAIT UNTIL <expression>或IF<expression> THEN的表达式的一部分。

只有当与程序的进程PMOVE和轴的两个运动缓冲区NTYPE和MTYPE相关联的运动缓冲区没有移动时，IDLE才会返回0。如果在移动开始后直接使用WAIT IDLE，它将自动检查移动是否已加载，然后等待它结束。

IDLE状态仅指轴的要求位置。当IDLE时，轴已完成VPU中的移动，但伺服系统可能仍在定位。

值

FALSE一个或多个运动在轴上运行
TRUE轴上没有运行或缓冲的运动

例

例1

开始移动，然后暂停程序执行，直到移动完成。注意:这并不一定意味着电机是静止的，因为它可能仍在运动到位置。

MOVE(100)

WAIT IDLE

PRINT "Move Done"

OP(move_finished, ON)

例2

如果轴没有加载任何移动，那么加载一个新的序列。

IF IDLE AXIS(1) THEN

  BASE(1)

  MOVE(100)

  MOVE(50)

  MOVE(-150)

ENDIF

例3

开始移动，然后等待，直到Registration事件发生，或者移动已经完成。

REGIST(20)

MOVELINK(100, 120, 20, 20, 3)

WAIT UNTIL MARK OR IDLE

IF MARK THEN

  captured_position = REG_POS

  missed_mark = 0

ELSE

  PRINT #5, "Failed to find MARK"

  missed_mark = missed_mark + 1

  IF missed_mark > 10 THEN

    GOSUB flash_amber_lamp

  ENDIF

ENDIF

例4

将5轴插补运动指令输入缓存区缓冲一系列合并的5轴移动，然后等待指令完成。只有BASE轴需要检查IDLE，因为所有5个轴在同一BASE中工作。

BASE(4,5,6,7,8)

Merge = on

INTERP_FACTOR AXIS(7) = OFF '仅计算XYZ的矢量速度

INTERP_FACTOR AXIS(8) = OFF 'A轴和B轴只是同步跟随。

MOVE(x1, y1, z1, a1, b1)

MOVE(x2, y2, z2, a2, b2)

MOVE(x3, y3, z3, a3, b3)

MOVE(x4, y4, z4, a4, b4)

MOVE(x5, y5, z5, a5, b5)

MOVE(x6, y6, z6, a6, b6)

WAIT IDLE

在这个序列之后，INTERP_FACTOR必须被设置回ON，以允许单个轴单独移动。
