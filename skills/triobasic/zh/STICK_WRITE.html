STICK_WRITE

类型

系统功能

语法

value = STICK_WRITE(flash_file, table_start [,length [,format]])

描述

用于以两种格式之一将table数据存储到SD卡上。

如果这个文件已经存在，它将被覆盖。

如果您想在不丢失任何精度的情况下存储数据，请使用二进制格式。

参数

value:TRUE =函数成功
FALSE =函数不成功
flash_file:一个数字，当附加到字符“SD”将形成数据文件名。
硬编码的字符串或string变量将形成数据文件名。
table_start:TABLE中的起始点，数据值将从这里传输。
length:要传输的table值的个数(默认128个值)
format:0 =二进制64位浮点格式，BIN文件(默认)
1 = ASCII码逗号分隔值，文件，无索引CSV
2 =包含索引的ASCII码CSV

当以format=0存储时，数据以IEEE浮点二进制格式little-endian存储，即优先存储最低有效位字节。

例

例1

将从TABLE(1000)开始的2000个值传输到称为SD1501.BIN的SD卡文件

success = STICK_WRITE (1501, 1000, 2000, 0)

例2

将从TABLE(1000)开始的2000个值传输到名为MYDATALOG2.CSV的SD卡文件

DIM flash_file AS STRING(20)

flash_file = "MYDATALOG1"

success = STICK_WRITE (flash_file, 1000, 2000, 1)

例3

保存从TABLE(5000)开始的50个值到名为SD000002.CSV的SD卡文件中。以格式索引保存的数据，data <CR>。
