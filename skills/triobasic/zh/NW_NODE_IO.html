NW_NODE_IO

类型

系统数组(MC_CONFIG)

语法

NW_NODE_IO(slot, alias_address) = offset_value

描述

这个二维数组可用于设置任何EtherCAT节点I/O通道的起始地址。这可以用来定义一个用户特定的I/O映射，将来自不同来源的I/O点固定到位。

默认的I/O分配顺序为;控制器I/O，模块I/O, CAN I/O, EtherCAT I/O(驱动器I/O和独立的EtherCAT I/O)

当使用多个I/O类型时，最好使用I/O基本修改参数来配置I/O顺序。看到NODE_IO。

参数

slot:EtherCAT端口槽位号。(MC4N、MC6N和Flex-6 Nano上为0)
alias_address:已配置的从站设备别名地址。
offset_value:0自动分配EtherCAT I/O(默认)
> = 8定义节点的EtherCAT I/O从这个I/O点地址开始

例

一个带有MC664的系统，使用内置的EtherCAT端口(插槽0)和5 个EtherCAT伺服驱动器与数字量I/O。配置的站别名地址为1 ~ 5。要求如果移除一个驱动器，数字I/O不改变。每个伺服驱动器有16个输入和16个输出。

NW_NODE_IO(0, 1) = 32

NW_NODE_IO(0, 2) = 48

NW_NODE_IO(0, 3) = 64

NW_NODE_IO(0, 4) = 80

NW_NODE_IO(0, 5) = 96
I / O点I/O的位置
0-7内置输入
8-15内置双向I/O
16-31无
32-47EtherCAT输入-节点1
48-63EtherCAT输入-节点2
64-79EtherCAT输入-节点3
80-95EtherCAT输入-节点4
96-112EtherCAT输入-节点5
112-1023虚拟I / O

即使中间节点从网络中移除，每个节点的I/O地址也保持不变。
