CRC16

类型

数学函数

语法

result = CRC16(mode,{parameters})

描述

计算存储在连续表内存或VR内存位置的数据的16位循环冗余校验(CRC)。

参数

模式:0初始化多项式
1计算CRC(legacy模式)
2计算CRC

在运行模式1计算之前，必须至少由模式0初始化一次CRC16。

Mode = 0

语法

result = CRC16(0, poly[, reverse_input [,reverse_output [, bits]]])

描述

用多项式初始化命令

参数

result:总是返回-1
poly:用作CRC校验范围0-65535(或0-$FFFF) 种子的多项式
reverse_input:如果reverse_input为0，则输入中每个字符的最高有效位在左边，否则在右边。此参数默认为0。
reverse_output:如果reverse_output为0，则CRC的最高有效位在左侧，否则在右侧。该参数默认为0。
bits:CRC计算的位数必须在8到32的范围内。该参数默认为16。

Mode = 1

语法

result = CRC16(1, source, start, end, initial)

描述

计算legacy模式下的CRC。该模式有以下限制:

  • 在CRC16(0，…)命令中，可选参数将被忽略，不应该被指定。
  • 计算出的CRC值必须通过在待处理数据的末尾附加两个ASCII NUL(数字0)字符来“刷新”，或者在两个ASCII NUL字符上执行额外的CRC16(1，…)
  • 一旦CRC被刷新，它就不能用于处理更多的数据。

参数

result:返回计算结果。CRC如果计算失败或未初始化CRC16，则返回0。
source:定义数据加载的位置
0Table内存
1VR内存
start:第一个字节的起始位置
end:最后一个字节的结束位置
initial:CRC通常为$0 - $FFFF

Mode = 2

语法

result = CRC16(2, source, start, end, initial)

描述

计算CRC。reverse_input和reverse_output标志被重视且不需要额外的处理。由于不需要刷新，我们可以随时向CRC计算中添加更多数据。

参数

result:返回CRC的计算结果。如果计算失败或CRC16未初始化则返回0。
source:定义在何处加载数据
0Table内存
1VR内存
start:第一个字节的起始位置
end:最后一个字节的结束位置
initial:初始CRC值。通常为$0 - $FFFF

例

例1

使用 表记忆 (legacy模式) 计算CRC：

poly = $8005

CRC16(0, poly)'初始化内部CRC表内存



'将数据加载到TABLE内存位置0-7

'将刷新数据加载到位置8-9的TABLE内存中

TABLE(0,1,2,3,4,5,6,7,8,0,0)

reginit = 0

calc_crc = CRC16(1,0,0,9,reginit) '源数据=TABLE(0..7)

例2

使用TABLE和VR(legacy模式)刷新计算CRC:

CRC16(0, poly)

= $8005'初始化内部CRC表内存



'将数据加载到TABLE内存位置0-7

TABLE(0,1,2,3,4,5,6,7,8,0,0)

'将刷新数据加载到位置0的VR内存中

VR (0)=0

reginit = 0

calc_crc = CRC16(1,0,0,7,reginit) '源数据=TABLE(0..7)

calc_crc = CRC16(1,1,0,0,calc_crc)'刷新第一个字节

calc_crc = CRC16(1,1,0,0,calc_crc)'刷新第二个字节

例3

使用VR值(legacy模式) 计算CRC：

'生成CRC查找表

poly=$8005

CRC16(0,poly)



'创建测试数据为"hello"

VR(100)=104

VR(101)=101

VR(102)=108

VR(103)=108

VR(104)=111

VR(105)=0

VR(106)=0

PRINT VRSTRING(100)



'计算

crc=0

crc=CRC16(1,1,100,106,crc)



'打印结果

PRINT HEX(crc)
