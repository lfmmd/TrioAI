NODE_INDEX

类型

系统数组(MC_CONFIG)

语法

NODE_INDEX(slot, node) = value

描述

这个2D数组可用于将指针设置为EtherCAT节点使用的VRs块。它可用于定义来自不同数据源(包括EtherCAT节点中的布尔和整形数据)的特定于用户的输入输出映射。

每个PDO对象都映射一个VR，从从站到主站的值开始(例如，从站的实际值，DIN，状态字，实际位置等)，然后从主站到从站的值(例如，从站的目标值，DOUT，控制字，目标位置等)。

该数组是二维的，第一维是主站槽标识符，第二维是主站网络内节点的位置。

对于使用包含 VR 映射循环 PDO 对象的同一 EtherCAT 配置配置文件的每个节点，控制器应自动递增 VR 基地址，从该配置文件中为第一个节点定义的地址开始递增，除非通过使用 NODE_INDEX 或 NW_NODE_INDEX 参数为特定节点显式设置了此地址

参数

slot:EtherCAT端口槽位号。(MC4N、MC6N和Flex-6 Nano上为0)
node:由节点在EtherCAT网络中的位置决定的地址。第一个节点是地址0。
value:0 ~ 65535EtherCAT循环数据被映射到从该VR索引开始的VR块。(MC664、MC6N、Flex-6 Nano)
0 ~ 4095EtherCAT循环数据被映射到从该VR索引开始的VR块。(MC4N)

根据EtherCAT配置的地址分配偏移量，使用NW_NODE_INDEX

例

在EC_EXTEND中，VR被映射为从地址100和200开始。有2个从设备使用相同的EC_EXTEND。将第一个设备映射为使用100和200，第二个设备使用从120和220开始的VR。

NODE_INDEX(0, 0) = 0

NODE_INDEX(0, 1) = 20
