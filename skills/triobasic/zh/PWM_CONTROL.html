PWM_CONTROL

类型

轴命令

语法

PWM_CONTROL(value)

描述

PWM控制需要在运动控制器的FPGA中使用特殊的硬件。使用前请检查是否支持此功能。任何轴的PWM输出都可以与在其他轴上运行的运动速度相关联。例如:轴4的PWM输出可以链接到在轴0,1和2上运行的插补移动的速度轮廓。

- PWM输出标记/空间可以通过应用软件程序简单地写入。

- PWM标记大小可以通过插补路径速度自动设置。

- PWM标记大小可以通过插补路径速度和沿XY路径定义的轮廓自动设置。

-一些实现还具有硬件门控功能。

20mhz时钟作为PWM产生的基准时钟。

参数

值描述
0:PWM输出OFF
1:PWM输出接通。PWM_MARK值来自PWM_MARK轴参数。
2:PWM_CONTROL(2, multiplier, offset, limit value, link axis)
PWM输出通过轴和乘法器的VP_SPEED(路径速度)计算。它在每个伺服周期上进行调整。

加载到PWM_MARK中的值由VP_SPEED * multiplier + offset给出。

如果计算值超过“限制值”，则应用限制值。VP_SPEED值是以用户为单位的链接轴的速度剖面速度。
3:PWM_CONTROL(3, GT/LT, ON/OFF, table start, table end)
这个轴函数将一系列位置加载到门控FIFO中。GT/LT值对于GT比较应该设置为1，对于LT比较应该设置为0。

当位置由功能自动加载时，ON/OFF位被切换。FIFO位置缓冲区可以加载一个或多个PWM_CONTROL(3，…)函数。当位置缓冲区可用时，可以加载更长的位置序列。
4:清空FIFO缓冲区。
5:如果硬件允许，打开PWM门控功能。
6:将PWM_ENCODER寄存器清除为零。
7:保留。
8:PWM_CONTROL(8, multiplier, offset, limit_value, velocity_link_axis, amplitude_link_axis)
PWM输出通过轴和乘法器的VP_SPEED(路径速度)计算，并将在每个伺服周期上进行调整。加载到PWM_MARK中的值计算为:
VP_SPEED velocity_link_axis * DPOS amplitude_link_axis * multiplier + offset.
如果这个计算值超过了limit_value，则应用limit_value。VP_SPEED值是速度链接轴的速度剖面速度。
可以使用CAMBOX剖面设置amplitude_link_axis的DPOS，使振幅变化。
这是通过3个阶段来实现的:
1. CONNPATH(基本XY)移动类型可用于将虚拟轴“连接”到XY路径。
2. 然后将CAMBOX剖面链接到虚拟轴上。
3.然后，PWM_CONTROL(8，…)函数链接到CAMBOX轮廓
9:PWM_CONTROL(9, multiplier, duty_cycle, link_axis)
PWM输出的频率通过链接轴和乘法器的VP_SPEED(轴的路径速度)计算，并将在每个伺服周期上进行调整。
加载到PWM_CYCLE中的值由20000000 / (VP_SPEED * multiplier)给出。VP_SPEED值是以用户为单位的链接轴的速度剖面速度。
加载到PWM_MARK中的值是在duty_cycle中指定的PWM_CYCLE的百分比。

注意，为了产生PWM，WDOG必须是ON。

PWM_CONTROL函数是一个轴函数，可以使用axis修饰符或BASE函数定向到一个轴。
