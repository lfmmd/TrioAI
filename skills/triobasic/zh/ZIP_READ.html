ZIP_READ

类型

命令

语法

ZIP_READ(function ,...)

描述

这个函数将把压缩文件读入运动控制器上的RAM，然后按块解压缩。

文件必须使用TextFileLoader(可执行或ActiveX)传输到运动控制器上的SD卡上，启用压缩，禁用解压缩，这样文件将以压缩格式存储。

在内部我们处理两个缓冲区:压缩缓冲区和解压缩缓冲区。压缩缓冲区从文件中填充，解压缩缓冲区从压缩缓冲区中填充。数据在需要的时候在缓冲区之间传输。

参数

功能:描述:
0初始化ZIP_READ资源。
1释放所有ZIP_READ资源。
2将一块数据从解压缩缓冲区传输到VR或TABLE内存。
3在解压缩缓冲区中跳过一些字节。
4读取缓冲区索引。
5移动到解压缓冲区中的一个位置。
6解压缩下一个缓冲区。

Function = 0

语法

value = ZIP_READ(0,”filename”[,decompress_block_size[,decompress_block_count]])

描述

这个函数初始化ZIP_READ资源。

由于内部解压缩数据结构的大小，TEXT_FILE_LOADER和ZIP_READ命令共享相同的数据结构。这意味着，如果TEXT_FILE_LOADER正在解压缩数据，那么ZIP_READ函数将失败，反之亦然，如果ZIP_READ函数正在运行，TEXT_FILE_LOADER解压缩将失败。这应该不是问题，因为TEXT_FILE_LOADER不能解压缩将由ZIP_READ命令处理的文件。

文件以块为单位解压缩。缺省情况下，有一个32 KB的块。这个decompress_block_size参数允许减小块大小。块大小将被四舍五入到最接近的2的幂。

如果decompress_block_count大于1，那么ZIP_READ将执行双缓冲。这意味着一个进程可能正在解压缩文件，而另一个进程正在使用解压缩的数据。解压缩数据的总量被限制为32 KB，因此可用的解压缩块的数量受到decompress_block_size的限制

参数

value:0初始化失败
1初始化成功，但完整的压缩文件无法加载到内存中。这意味着在某些时候需要读取另一个缓冲区。这个缓冲区读取可能需要相当长的时间，因此可能需要双重缓冲方案。
2初始化成功，完整的压缩文件被加载到内存中。
Filename:要打开的SD卡上的文件名称。
decompress_block_size:2 - 32768(默认=32768)
decompress_block_count:1 - (32768 / decompress_block_size)

例

IF ZIP_READ(0,"myfile.tfl",2048)=0 THEN

    PRINT "Error initialising reader"

    STOP

ENDIF

ZIP_READ(1)

Function = 2

语法

value=ZIP_READ(2,format,destination,start,length)

描述

这个函数从解压缩的缓冲区中读取一块数据到VR或TABLE内存中。如果没有足够的解压缩数据可用，那么更多的数据将被解压缩。

参数

value:存储的值的数量
format:08位整数(ASCII字符数据)
116位整数(小端位数)
216位整数(大端位数)
332位整数(小端位数)
432位整数(大端位数)
564位整数(小端位数)
664位整数(大端位数)
732位浮点数(小端位数)
832位浮点数(大端位数)
964位浮点数(小端位数)
1064位浮点数(大端位数)
destination:0在TABLE中存储数据
1在VR中存储数据
start:0 ≤start目标内存区域中开始存储数据的位置。
length:要存储的值的数量。处理的字节数将取决于格式，例如，如果format为7，则长度为100将处理400字节

如果返回值小于length参数，那么我们已经到达了文件的末尾，任何进一步的读取都会导致TrioBASIC错误。

例

IF ZIP_READ(0,"myfile.tfl",2048)=0 THEN

    PRINT "Error initialising reader"

    STOP

ENDIF

REPEAT

    c=ZIP_READ(2,0,0,1000,50)

UNTIL c<50

Function = 4

语法

value=ZIP_READ(4,index)

描述

这个函数返回内部缓冲区索引的值。

参数

value:指定索引的值。
index:0压缩缓冲区偏移量
1压缩缓冲区长度
2压缩文件偏移
3未压缩缓冲区偏移量
4未压缩缓冲区长度
5未压缩文件偏移量

例

IF ZIP_READ(0,"myfile.tfl",2048)=0 THEN

    PRINT "Error initialising reader"

    STOP

ENDIF

ZIP_READ(3,23)

REPEAT

    c=ZIP_READ(2,0,0,1000,50)

    PRINT "Compressed file indices: ";

    PRINT ZIP_READ(4,0),ZIP_READ(4,1),ZIP_READ(4,2)

    PRINT "Decompressed file indices: ";

    PRINT ZIP_READ(4,3),ZIP_READ(4,4),ZIP_READ(4,5)

UNTIL c<50

Function = 5

语法

value=ZIP_READ(5[,position])

描述

此函数设置解压文件的绝对位置。如果未指定可选的position参数，则使用默认值0。

参数

value:解压缩文件的绝对位置，如果有错误则为-1
position:解压缩文件中的绝对位置。

如果返回值小于length参数，那么我们已经到达文件的末尾，任何进一步的读取将导致TrioBASIC错误。

例

IF ZIP_READ(0,"myfile.tfl",2048)=0 THEN

    PRINT "Error initialising reader"

    STOP

ENDIF

ZIP_READ(3,23)

VR(100)=-1

REPEAT

    IF VR(100)>=0 THEN ZIP_READ(5,VR(100)):VR(100)=-1

    c=ZIP_READ(2,0,0,1000,50)

    PRINT "Compressed file indices: ";

    PRINT ZIP_READ(4,0),ZIP_READ(4,1),ZIP_READ(4,2)

    PRINT "Decompressed file indices: ";

    PRINT ZIP_READ(4,3),ZIP_READ(4,4),ZIP_READ(4,5)

UNTIL c<50
