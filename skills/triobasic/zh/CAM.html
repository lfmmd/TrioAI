CAM

类型

轴命令

语法

CAM(start point, end point, table multiplier, distance)

描述

CAM命令用于根据定义运动轮廓的位置表生成轴的运动。值表是用TABLE命令指定的。移动可以用从3到可用的最大表大小的任意数量的点来定义。控制器在表中的值之间执行线性插值，以允许少量的点来定义平滑轮廓。

通过将TABLE值偏移第一个值，然后将它们乘以multiplier参数，将它们转换为位置。这意味着一个非零的起始轮廓将被偏移，以便使得第一个点为零，然后所有值都使用乘法器进行缩放。然后将这些用作起始位置的绝对位置。

同时执行的两个或多个CAM命令可以使用表中的相同值。

CAM轮廓的速度是通过基轴的SPEED和距离参数来确定的。您可以使用这两个值来确定执行CAM轮廓所花费的时间。

与任何运动命令一样，SPEED可以在任何时候更改为任何正值。以ACCEL速度爬坡增加到当前SPEED值。

要获得不受ACCEL影响的CAM形状，该值应至少设置为SPEED值的1000倍(假设默认SERVO_PERIOD为1ms)。

当CAM命令执行时，ENDMOVE参数被设置为先前移动的结束

参数

start point:凸轮轮廓线在TABLE中的起始位置
end point:凸轮轮廓线在TABLE中的结束位置
multiplier:表值乘以这个值来生成位置。
distance:距离参数将轴的速度与完成凸轮轮廓所花费的时间联系起来。所花费的时间可以使用当前轴速和这个距离参数(以用户单位为单位)来计算。

CAM命令使用的TABLE值不使用单位。所有点都以编码器计数或步进脉冲计数进行缩放。

例

例1

一个系统正在被编译为一个花10秒完成的CAM轮廓。轴速度为200，因此计算和指定的距离为2000。距离参数不是轴移动的距离。仅用于确定CAM时间。

SPEED = 200 '轴速度

ACCEL = SPEED * 1000 '要在1ms内达到速度的轴

DECEL = SPEED * 1000  

time_value = 10 '完成轮廓的时间（s）

distance = SPEED * time_value 'CAM曲线的"distance" 参数

CAM(0, 100, 1, distance)

例2

运动需要遵循POSITION方程:

t(x) = x*25 + 10000(1-cos(x))

其中x的单位是度。这个示例表提供了一个简单的振荡叠加一个恒定的速度。要加载该表并连续循环，程序将是:

FOR deg = 0 TO 360 STEP 20 '循环填写表格

    rad = deg * 2 * PI/360 '将度转换为弧度

    x = deg * 25 + 10000 * (1 - COS(rad))

    TABLE(deg / 20, x) '将x的值放进表中

  NEXT deg



  WHILE IN(2) = ON '当输入2位ON时，重复凸轮运动

    CAM(0, 18, 1, 200)

    WAIT IDLE

  WEND

例3

一个带罩棘轮被用来制作一个模板，激光可以通过这个模板照射出来，用于打印十个数字的系统。所需的数字通过1号串口以ASCII文本的形式传输到控制器。

使用的编码器每次旋转有4000条沿，因此必须在每个位置之间移动400条沿。cam表从0到1，这意味着CAM乘法器需要是400的倍数才能在位置之间移动。

轮盘需要每0.25秒移动一次到预设位置。速度设置为10000沿/秒，我们希望在0.25秒内完成轮廓。所以用轴速乘以所需的完成时间

(10000 × 0.25)得到的距离参数等于2500。

 GOSUB profile_gen

  WHILE IN(2) = ON

    WAIT UNTIL KEY#1 '等待端口1上的字符

    GET#1, k

    IF k > 47 AND k < 58 THEN '检查有效ASCII字符

      position = (k - 48) * 400 '转换为绝对位置

      multiplier = position - offset '计算相对移动

      '检查反向移动是否更短

      IF multiplier > 2000 THEN

        multiplier = multiplier - 4000

      ELSEIF multiplier < -2000 THEN

        multiplier = multiplier + 4000

      ENDIF

      CAM(0, 200, multiplier, 2500) '设置CAM运动

      WAIT IDLE

      OP(15, ON) '触发激光闪光

      WA(20)

      OP(15, OFF)

      offset = (k - 48) * 400 '计算当前绝对位置

    ENDIF

  WEND



profile_gen:

  num_p = 201

  scale = 1.0

  FOR p = 0 TO num_p - 1

    TABLE(p, ((-SIN(PI * 2 * p / num_p) / (PI * 2)) + p / num_p) * scale)

  NEXT p

RETURN

例4

吸力拾取系统必须根据所承载的负载改变其速度。该机构有一个称重传感器，输入到控制器的模拟量通道(AIN)。

移动配置文件是固定的，但完成此移动所需的时间必须根据AIN而变化。AIN值从100到800不等，这导致1到8秒的移动时间。如果将速度设置为每秒10000个单位，所需时间为1到8秒，那么距离参数必须在10000到80000之间。(距离=速度x时间)

返回行程可以在0.5秒内完成，因此返回运动的距离值5000是固定的。乘数设置为-1，用于反向运动。

  GOSUB profile_gen '将凸轮轮廓加载到表中

  SPEED = 10000: ACCEL = SPEED * 1000: DECEL = SPEED * 1000

  WHILE IN(2) = ON

    OP(15, ON) '打开吸气

    load = AIN(0) '捕捉负载值

    distance = 100 * load '计算距离参数

    CAM(0, 200, 50, distance) '在计算时间内向前移动50mm

    WAIT IDLE

    OP(15, OFF) '关闭吸气 

    WA(100)

    CAM(0, 200, -50, 5000) '返回抓取位置

  WEND



profile_gen:

  num_p = 201

  scale = 400 '设置刻度，使乘数以mm为单位

  FOR p = 0 TO num_p - 1

    TABLE(p, ((-SIN(PI * 2 * p / num_p)/(PI * 2)) + p / num_p) * scale)

  NEXT p

RETURN

例5

在100Hz频率的轴上需要一个简单的正弦运动。请注意，这将需要比默认的1000µs更快的伺服周期，因为每个周期只需要0.01秒。

simple_camtable:

  i = 100

  cam_scale = UNITS AXIS(1)

  FOR theta = 0 TO INT(2 * PI * 100) STEP 10

    TABLE(i, cam_scale * SIN(theta / 100))

    PRINT i, theta, TABLE(i)

    i = i + 1

  NEXT theta

  TABLE(i, 0) '强制在表中的最后位置

  PRINT i, theta, TABLE(i)

  last_cam = i



  ' 然后CAM运行它：



  BASE(1)

  ' 100 Hz，每个周期是0.01秒

  '这部分设置周期时间不是运动的实际速度

  cycle_time = 0.01

  cam_factor = 10000

  SPEED = cam_factor / cycle_time

  ACCEL = SPEED * 4000

  DECEL = SPEED * 4000



  distance = 20 'mm

  WHILE TRUE

    CAM(100, last_cam, distance, cam_factor)

  WEND
