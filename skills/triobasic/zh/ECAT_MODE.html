ECAT_MODE

类型

插槽参数(MC_CONFIG)

语法

ECAT_MODE SLOT(slot_number) = value

描述

ECAT_MODE在固件的EtherCAT部分内设置各种操作。ECAT_MODE中的每个位用于打开或关闭下面描述的功能。该参数可以在MC_CONFIG中设置，在命令行或程序中使用。模式设置将在参数写入后立即激活。

参数

slot_number:运行EtherCAT的槽位。
在MC664(X)中，slot_number为EtherCAT模块的物理槽位。内置的EtherCAT端口槽位号固定为“-2”。
在MC4N-ECAT、MC6N-ECAT和Flex-6 Nano中，slot_number为0。
value:每个位的操作如下图所示。
位状态描述
00紧急消息响应。当从站接收到紧急消息并且控制器记录该错误时，EtherCAT协议将引发SYSTEM_ERROR (0x100000)。
1EtherCAT记录错误，但不引发SYSTEM_ERROR。
1标准EtherCAT启动。EtherCAT协议要等到所有sync0启动时间都到期后才请求安全操作状态更改。默认情况下(位1 = 0)
快速启动EtherCAT。EtherCAT协议不会等待所有sync0启动时间都到期。它要求在最早的时间内将安全操作到操作状态更改。
注意，一些从站可能需要用户程序在网络进入操作状态后清除从站错误。
2+供将来使用

位0和位1在固件V2.0291及以后版本中实现。

例

一个或多个从节点发送紧急消息以及在CoE状态字中设置故障位。在这种情况下，禁用SYSTEM_ERROR上的紧急消息操作是可以的，因此可以在MC_CONFIG中设置ECAT_MODE位0。EtherCAT网络运行在运动控制器的0号槽位。

ECAT_MODE SLOT(0) = 1
