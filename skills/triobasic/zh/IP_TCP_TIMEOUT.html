IP_TCP_TIMEOUT

类型

系统参数(仅限MC_CONFIG、MC464)

描述

IP_TCP_TIMEOUT以毫秒为单位定义TCP连接(Ethernet/IP、ModbusTCP、HMI、Token和Telnet)在没有任何活动的情况下保持打开状态的持续时间。当超过这个值时，控制器将关闭TCP连接。默认值为3600秒。该参数必须在MC_CONFIG中才有效。

将此值设置远离默认值可能会使Motion Perfect连接不稳定。

该值的每12位设置该部分以太网的超时时间(以秒为单位)。如果它保持0，那么它就变成了默认的3600秒。

在以太网堆栈中还有一个大约8秒的内置超时，因此当您将IP_TCP_TIMEOUT中的值设置为2秒时，总数为10秒。

例1

当主服务器没有活动时，设置以太网处理器在20秒后关闭Modbus TCP套接字，这使得主服务器能够在通信中断后重新打开并继续连接。请记住，指定的值是在内置的8秒超时时间之外的（见上面的注释），所以我们只需要在此基础上加上额外的时间（在本例中为12秒），就可以得到20秒的总超时时间。

'Modbus套接字将在20秒后关闭(12 + 8)

IP_TCP_TIMEOUT = $00C000000

例2

当主服务器没有活动时，设置以太网处理器在12秒后关闭以太网TCP套接字，这使得主服务器能够在通信中断后重新打开并继续连接。请记住，指定的值是在内置的8秒超时时间之外的（见上面的注释），所以我们只需要在此基础上加上额外的（在本例中为4秒），就可以得到12秒的总超时时间。

' TCP套接字将在12秒后关闭(4 + 8)

IP_TCP_TIMEOUT = $004000000000
