MODBUS

类型

系统功能

语法

MODBUS(function, slot[, parameters...])

描述

该功能允许用户配置以太网端口作为Modbus TCP和Modbus RTU客户端(Master)运行。使用MODBUS命令，用户可以打开与远程服务器的连接，使用Modbus功能号子集传输数据并检查错误。

最多可以同时打开10个Modbus TCP连接。

可能会打开一个Modbus RTU主连接。

通过串口建立Modbus连接时，必须先使用SETCOM命令配置串口参数。

参数

function:0打开Modbus客户端连接
1关闭连接
2检查连接状态
3发送命令(Modbus功能)
$10获取错误日志条目
$11获取错误日志计数

Function = 0;

语法

value = MODBUS(0,slot, ip address/network type 1...4[, port number/connection type[, vr_index[, unit identifier[, CmdTimeout[, CharTimeout]]]]])

描述

尝试打开一个Modbus TCP/RTU客户端连接到给定的远程服务器。

参数

value:TRUE =命令执行成功
FALSE=命令失败
slot:通信端口所在的模块槽位
ip address/network type:服务器的IP地址，用逗号分隔的4个字节(Modbus TCP)
1RS232 (Modbus RTU)
22线RS485 (Modbus RTU)
34线RS485 (Modbus RTU)
port number/connection type:可选端口号。如果没有给出，默认为端口502。(Modbus TCP)
0客户端(Modbus RTU)
1服务器(Modbus RTU) *用于未来的开发
vr_index:将写入连接句柄的VR的索引号。默认值为-1。-1表示打印到标准输出流。(通常为终端0)
unit identifier:请求头中使用的单元标识符字节。(仅限Modbus RTU *)
CmdTimeout:执行函数的超时时间(ms)。默认为5000 ms(5秒)。
CharTimeout:在触发错误之前，单个接收字符之间允许的最大时间间隔或消息中传输字符的能力。默认为100毫秒。

例1

'IP地址192.168.0.185，端口号502

IF MODBUS(0, -1, 192, 168, 0, 185, 502, 20) = TRUE THEN

  PRINT "Modbus port opened OK"

  modbus_handle = VR(20)

ELSE

  PRINT "Error, Modbus server not found"

ENDIF

例2

'Modbus RTU主控RS232

IF MODBUS(0, -1, 1, 0, 20) = TRUE THEN

  PRINT "Modbus port opened OK"

  modbus_handle = VR(20)

ELSE

  PRINT "Error, Modbus server not found"

ENDIF

例1

以16位模式读写保持寄存器。将保持寄存器数据映射到运动控制器中相同的VR地址。

my_slot = -1



open_modbus = $00

close_modbus = $01

get_status = $02

ex_modbus_func = $03

get_error_log = $10





'检查Modbus是否已经打开

MODBUS(get_status, my_slot, 100)

IF VR(100) = 1 THEN

    '关闭连接以便可以重新打开

    MODBUS(close_modbus, my_slot)

ENDIF



'打开modbus服务器(远程从站)并在VR(20)中放置句柄

MODBUS(open_modbus, my_slot, 192, 168, 0, 249, 502, 20)



REPEAT

    '从保持寄存器1000到1009获取10个值

    MODBUS(ex_modbus_func, my_slot, VR(20), 3, 1000, 10)

    '发送10个值到保持寄存器1010到1019

    MODBUS(ex_modbus_func, my_slot, VR(20), 16, 1010, 10)

    WA(200)

UNTIL FALSE

语法

MODBUS($10, slot, handle[, entry offset[, VR index]])

描述

返回错误日志条目。如果没有提供条目偏移量，则返回最后一个条目(偏移量= 0)。否则，1将返回前一个条目，2将返回除2以外的最后一个条目，等等。

参数

value:TRUE命令成功
FALSE命令不成功
slot:通信端口所在的模块槽位
handle:需要其错误日志条目的连接的句柄。如果-1则访问常规协议错误(例如打开连接失败)。
entry offset:错误日志中的条目。如果没有提供，则返回条目0。
VR index:保存返回值的VR数。如果设置为-1或不包含，则将该值打印到命令行终端。

例

例1

'获取错误日志条目0到4，输入VR(100)到VR(104)

FOR I = 0 to 4

  error_flag = MODBUS($10, -1, modbus_handle, i, 100 + i)

  IF error_flag = FALSE THEN

    PRINT "Error fetching error log entry "; i[0]

  ENDIF

NEXT i
