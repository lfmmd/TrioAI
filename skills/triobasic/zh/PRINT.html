PRINT

类型

文件函数

替代格式

?

语法

PRINT [#channel, ]expression-1[, expression-2[, ...]]

描述

PRINT命令允许TrioBASIC程序向通道输出一系列字符。通道可以是串行端口，也可以是连接到_运动控制器_的其他类型的连接。

通过用逗号或分号分隔多个表达式，可以在单个PRINT语句中打印多个表达式。每个表达式都可以使用HEX和CHR等函数修改，输出格式可以使用方括号内的格式说明符指定(见下文)。

在没有格式说明符的情况下，一个值被打印到小数点后4位。

任何大于1e19且小于1e-18的值都将使用科学记数法打印。

参数

#channel,完整通道列表参见#(如果省略，默认为0或OUTDEVICE)
Expression-n:任何有效的TrioBASIC表达式，数字或字符串、标量或数组，或任何TARGET类型

下列元素可以与表达式一起使用:
;对表达式进行标点，完全没有任何分隔，如果在最终表达式之后使用，则会抑制回车和换行的输出
,给表达式加标点，用制表符分隔每个表达式
[format-specifier]指定表达式应该输出的格式，可以是一个或两个数值，也可以是一个C格式字符串。格式说明符不允许使用IP、TARGET、TARGETJ或TARGETE表达式。
Values[d]要显示的小数位数
[w, d]要输出的总宽度(左对齐为负)和小数点后位数
C格式“%f”浮点输出(64位)
“%F”大写的浮点输出(64位)
“%e”科学记数法的浮点输出(64位)
“%E”大写科学记数法的浮点输出(64位)
“%L”十进制的整数输出(64位)
“%W”十进制的无符号整数输出(64位)
“%X”十六进制的无符号整数输出(64位)
“%R”字符串输出(size +字符数组)-原始字节输出
“%S”字符串输出(size +字符数组)- Unicode输出
“%C”字符输出

如果数字太大，无法符合指定的格式，则该字段将被问号填充，以表示没有足够的空间显示该数字。

如果输出的字符数小于宽度，则输出用空格填充，如果右对齐则在输出前面(正宽度)，如果左对齐则在输出后面(负宽度)。

最好使用分号分隔符将两个字符串显示为一个字符串，而不是将它们连接起来(例如，" PRINT str1;str2 "会产生与" PRINT str1 + str2 "相同的输出)，因为这不会产生必须执行连接的开销。

C格式字符串

这些非常强大，因为它们提供了比使用一个或两个值时更多的格式化可能性，但代价是一些复杂性。

一般的格式是百分比符号后面跟着一个字符，表示要输出的数据类型，这些在上表中有描述。这根本不提供格式化，因此为了定制所产生的输出，这两个符号之间可能会有标志、宽度和精度，下面将对此进行描述。

这些标志可以是任意顺序，并且可以识别以下内容:

" + " -强制输出标志

“-”-左对齐输出

" 0 " -输出导致零

宽度是一个十进制值，精度是一个小数点后面跟着一个十进制值。可以在不指定宽度的情况下指定精度。

字符串输出可以使用两种格式来控制，“%R”和“%S”，它们被用来代替“%S”(小写)，因为TrioBASIC字符串被存储为一个大小和单独的字符数组，而“%S”是指向以null结尾的字符数组的基数的指针。使用PRINT时的默认字符串格式是" %R "，因为这会以原始形式发出字符，即不进行任何解码，这对通信协议很有用，但如果您希望在字符串为Unicode时正确填充输出，则不太好，因为字符可能使用多个字节。对于Unicode字符串，最好使用“%S”，因为这将解码字符串的字符，并应用格式字符串中指定的正确填充和精度。“%R”和“%S”都将解析整个字符串，并且不会在第一个空字符处停止。

字符输出使用“%C”而不是“%C”，因为TrioBASIC整数长度为64位，而“%C”期望的8位值需要以不同的方式解码。字符输出在避免必须在输出之前将字符代码转换为字符串时非常有用。

除了格式方面，字符可能出现在前面的百分比符号之前和后面的' type '符号之后，这些将得到响应。在示例程序中使用此功能来显示将输出括在冒号字符内所产生的输出范围。

Motion Perfect终端

当PRINT用于输出到Motion Perfect终端(#0，#5，#6或#7)光标控制ESC序列可以使用。以下VT100型控制序列从MPv4版本4.4.1起支持。
光标控制
函数序列描述
Cursor Home<ESC>[{ROW};{COLUMN}H设置后续文本开始的光标位置。如果没有提供行/列参数(即<ESC>[H])，光标将移动到屏幕左上角的主位置。
Force Cursor Position<ESC>[{ROW};{COLUMN}f与光标Home相同。
Cursor Up<ESC>[{COUNT}A将光标向上移动COUNT行;默认计数为1。
Cursor Down<ESC>[{COUNT}B将游标向下移动COUNT行;默认计数为1。
Cursor Forward<ESC>[{COUNT}C将光标向前移动COUNT列;默认计数为1。
Cursor Backward<ESC>[{COUNT}D将光标向后移动COUNT列;默认计数为1。
删除文本
函数序列描述
Erase End of Line<ESC>[K从当前光标位置擦除到当前行的末尾。
Erase Start of Line<ESC>[1K从当前光标位置擦除到当前行的开头。

例1

打印字符串，使用引号和字符串变量组合。

DIM string1, string2 AS STRING(20)

string1 = "OPEN"

string2 = "CLOSED"

IF IN(4) = ON THEN

  PRINT "Guard status "; string2

ELSE

  PRINT "Guard status "; string1

ENDIF

例3

打印一个带有4个字符和1个小数点的VR，然后在下一个选项卡中打印一个带有2个小数点的局部变量。

VR(1) = 6

variable = 410.5

PRINT VR(1)[4, 1], variable[2]

打印输出将为:

6.0     410.50

打印一个直接后跟数值的字符串。注意在这个例子中分号分隔符是如何使用的。它不会切换到下一列，允许程序员更自由地选择打印项的位置。

>> PRINT " position = ";mpo

例5

在消息末尾打印回车，不换行。打印行末尾的分号抑制了通常在打印行末尾发送的回车。ASCII(13)在不换行的情况下生成CR。该字符串要从串口通道1输出。

PRINT #1, "ITEM "; total; " OF "; limit; CHR(13);

例9

下面的程序展示了" %R "和" %S " C格式字符串如何处理Unicode字符串，并说明虽然使用不带格式说明符的PRINT(相当于" %R ")可以正确输出Unicode字符串，但它不能准确地格式化它们。

DIM s1 AS STRING(100)

DIM s2 AS STRING(100)

DIM s3 AS STRING(100)



s1 = "The rain in Spain falls mainly on the plain"

s2 = "Der Regen fällt in Spanien hauptsächlich auf die Ebene"

s3 = "Η βροχή στην Ισπανία πέφτει κυρίως στην πεδιάδα"



PRINT

PRINT "=== No format string (%R) ===================================="

PRINT "'"; s1; "'"

PRINT "'"; s2; "'"

PRINT "'"; s3; "'"

PRINT



PRINT

PRINT "=== %60R ====================================================="

PRINT "'"; s1["%60R"]; "'"

PRINT "'"; s2["%60R"]; "'"

PRINT "'"; s3["%60R"]; "'"



PRINT

PRINT "=== %60.30R =================================================="

PRINT "'"; s1["%60.30R"]; "'"

PRINT "'"; s2["%60.30R"]; "'"

PRINT "'"; s3["%60.30R"]; "'"



PRINT

PRINT "=== %-60.30R ================================================="

PRINT "'"; s1["%-60.30R"]; "'"

PRINT "'"; s2["%-60.30R"]; "'"

PRINT "'"; s3["%-60.30R"]; "'"



PRINT

PRINT "=== %60S ====================================================="

PRINT "'"; s1["%60S"]; "'"

PRINT "'"; s2["%60S"]; "'"

PRINT "'"; s3["%60S"]; "'"



PRINT

PRINT "=== %60.30S =================================================="

PRINT "'"; s1["%60.30S"]; "'"

PRINT "'"; s2["%60.30S"]; "'"

PRINT "'"; s3["%60.30S"]; "'"



PRINT

PRINT "=== %-60.30S ================================================="

PRINT "'"; s1["%-60.30S"]; "'"

PRINT "'"; s2["%-60.30S"]; "'"

PRINT "'"; s3["%-60.30S"]; "'"
