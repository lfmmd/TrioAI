VP_OPTIONS

类型

轴参数

语法

VP_OPTIONS = value

VP_OPTIONS AXIS(n) = value

描述

允许设置选项来修改各种VP_MODE的行为，但主要是允许VP_MODE = 6的行为更像VP_MODE = 5，如果需要的话。

位 0控制VP_MODE = 6的最小移动缓冲区深度是否为5，这是VP_MODE = 5的默认行为。

位 1控制VP_MODE = 6是否将强制参数与轴参数绑定，即FORCE_SPEED、FORCE_ACCEL、FORCE_DECEL和FORCE_JERK分别与SPEED、ACCEL、DECEL和JERK绑定。当设置此位时，当加载带有强制参数的移动时，轴参数被覆盖，并且轴参数随后可用于改变其运动，这是VP_MODE = 5的默认行为。当此位未设置时，当使用强制要求的移动被加载时，轴参数保持不变，并且当移动被加载时，强制要求在整个移动中使用，这是VP_MODE = 0的默认行为。

位 2控制当VP_MODE设置为5时使用哪个VP_MODE。当未设置时，将使用VP_MODE = 5，而当设置时，将使用VP_MODE = 6。请注意，此设置仅改变执行速度轮廓计算的内容，不影响辅助操作(由位0和位1处理，如果需要完全兼容性，也应设置)。唯一的目的是允许在指定使用VP_MODE = 5(目前PLCopen运动功能块和RPS应用程序)的环境中测试VP_MODE = 6。

值

位描述值
0自动设置limit_buffers1
1将力需求与轴心需求绑定2
2VP_MODE 5将使用模式64

例

例1

使VP_MODE 6使用最小limit_buffers值5。

VP_OPTIONS.0 = 1

例2

使VP_MODE 6将力需求绑定到轴给定，以允许在轮廓调整运动。

VP_OPTIONS.1 = 1

例3

使用VP_MODE 6使VP_MODE 5轮廓运动(注意位0和位1也需要设置为完全兼容)。

VP_MODE = 5 '使用VP_MODE 5(被下面的位2覆盖)

VP_OPTIONS.0 = 1 '如果小于5，则假定LIMIT_BUFFERED为5

VP_OPTIONS.1 = 1 '将力需求绑定到轴需求

VP_OPTIONS.2 = 1 '调用VP_MODE = 6而不是5
