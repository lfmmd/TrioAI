CO_WRITE

类型

系统命令

语法

CO_WRITE(slot, address, index, subindex, type, vr_number[, value])

描述

该函数在远程驱动器或I/O设备中设置CANopen-over-EtherCAT (CoE)对象。对象的索引和子索引用于向该对象写入值。该值可以来自VR，也可以在VR编号为-1时直接输入命令。

参考远程设备的手册获取可用对象列表。如果对象值设置成功，该命令返回TRUE(-1)。否则，在写入值时出现错误的情况下，命令返回FALSE(0)。

参数

slot:EtherCAT模块槽位号。
address:远端设备在网络中的节点地址
index:CANopen对象索引
subindex:CANopen对象子索引
Type:1Boolean
2Integer 8
3Integer 16
4Integer 32
5Unsigned 8
6Unsigned 16
7Unsigned 32
8Real 32
17Real 64
21Integer 64
27Unsigned 64
vr_number:当VR值介于0和最大值之间时，结果将被存储。

(如果下一个参数包含要写入的值，则为-1)
value:直接设置对象的可选数据值

例：

例1

将EtherCAT地址为3的远程驱动器设置为回零模式。

>>CO_WRITE(0, 3, $6060, 0, 2, -1, 6)

>>

例2

将远程驱动比例增益和速度前馈增益设置为VR(21)和VR(22)中的值。

VR (21) = 2500

VR (22) = 1000

'两个对象都是无符号 16位(数据类型6)

CO_WRITE(0, 1, $60fb, 1, 6, 21)

CO_WRITE(0, 1, $60fb, 2, 6, 22)

警告:在通过EtherCAT写入CANopen对象之前，请务必参考制造商的用户手册。
