TEXT_FILE_LOADER

类型

命令

语法

TEXT_FILE_LOADER[(function [, parameter[,value]])]

描述

TEXT_FILE_LOADER命令控制控制器上的TEXT_FILE_LOADER_PROGRAM。这个函数允许从BASIC中控制和配置TEXT_FILE_LOADER。可以设置TEXT_FILE_LOADER_PROC来定义TEXT_FILE_LOADER_PROGRAM运行在哪个进程上。

TEXT_FILE_LOADER_PROGRAM是与PCMotion的文件加载功能通信的快速文件传输过程的控制器端。

如果不使用任何参数，则该函数为0。

参数

功能:描述:
0运行TEXT_FILE_LOADER程序
1读取TEXT_FILE_LOADER参数
2写入TEXT_FILE_LOADER参数

Function = 0

语法

TEXT_FILE_LOADER 

TEXT_FILE_LOADER (0)

描述

作为程序启动TEXT_FILE_LOADER通信协议。TEXT_FILE_LOADER程序将占用一个用户进程，无论它是自动运行还是手动运行。

TEXT_FILE_LOADER程序通常在打开文件加载连接时自动启动。如果你想指定它应该在哪个进程上运行，你可以手动调用它。

如果手动执行TEXT_FILE_LOADER，它所运行的程序将在TEXT_FILE_LOADER行挂起。因此，TEXT_FILE_LOADER应该是要执行的程序的最后一行。

Function = 1和Function = 2

语法

value = TEXT_FILE_LOADER(function, parameter[， value])

描述

函数1和函数2分别用于从TEXT_FILE_LOADER_PROGRAM读取和写入参数。

透明协议传输的默认目标应该在任何传输发生之前设置。

参数

参数:描述:
0传输状态参数(只读)
0无传输活动
1传输活动
1透明协议传输的默认目的地
0TEMP 文件
1FIFO 文件
2SDCARD

例

例1

等待传输开始，然后在字符到达控制器时处理它们。

'等待文件传输开始

WAIT UNTIL TEXT_FILE_LOADER(1, 0) = 1



'处理这个文件

WHILE KEY #fifo_channel

    GET #fifo_channel, k

    PRINT #echo_channel, CHR(k);

    IF k = 13 THEN PRINT #echo_channel, CHR(10);

    IF k >= 65 AND k <= 90 THEN 'A to Z

        ltflag = 0

        spflag = 0

        value = 0

        GOTO command_pro

    ENDIF

WEND

例2

将文件加载到FIFO中，然后将文件配置为读回BASIC中。

'将FIFO设置为透明协议的默认文件位置

TEXT_FILE_LOADER(2, 1, 1)



'初始化FIFO

OPEN #fifo_channel AS "TRANSFER_FILE" FOR FIFO_WRITE(fifo_size)

CLOSE #fifo_channel



'打开FIFO读取

OPEN #fifo_channel AS "TRANSFER_FILE" FOR FIFO_READ



'运行

WHILE running

    '等待文件传输开始

    WAIT UNTIL TEXT_FILE_LOADER(1, 0)

    WHILE KEY #fifo_channel 

        GET #fifo_channel, char

        PRINT #5, CHR(char)

    WEND

WEND
