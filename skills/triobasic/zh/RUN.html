RUN

类型

系统命令

语法

RUN ["program"[, process]]

描述

在控制器上运行指定程序。程序可以从另一个程序被运行。

一个程序可以在不同的进程中多次运行。你可以使用PROCNUMBER来帮助在程序中赋值。

程序将继续执行，直到没有更多的行要执行，在命令行中输入HALT，发出STOP或出现运行时错误。

当程序运行时，在程序的数据区初始化时有一个短暂的延迟。因此，运行程序总是比通过GOSUB调用子程序或调用FUNCTION要慢。

在使用GLOBAL时，这种延迟更为明显。

当程序运行时，将在GLOBAL和CONSTANT表中搜索已定义变量的名称。如果预定义了许多GLOBAL/CONSTANT名称，这可能会导致程序启动延迟。

从V2.0298开始，当IEC程序运行时，多任务是扁平的，没有快速进程。使用SCHEDULE_TYPE.8 = 1在MC_CONFIG中启用IEC 61131-3的快速进程。

参数

program:要运行的程序的名称。如果不存在，则运行所选程序
process:可选进程号。(默认最高可用)

例

例1

从程序中，在进程2上运行STARTUP程序并等待其完成:MAIN

RUN "STARTUP", 2

WA(50)  '等待程序开始

WAIT UNTIL PROC_STATUS PROC(2) = 0   '等待程序完成

WDOG = ON

例2

STARTUP完成后，MAIN程序将启动在最高可用进程中运行的其他程序。

RUN "IO_CONTROL"

RUN "HMI"

RUN "SAUSAGE_CHOPPER"

例3

SELECT程序STARTUP并在命令行上运行它。

>>SELECT "STARTUP"

STARTUP selected

>>RUN%[Process 21:Program STARTUP] - Running

>>%[Process 21:Line 238] (31) - Program is stopped

>>
