MC_PhasingAbsolute

在从轴的主轴位置创建一个绝对相位移动。

描述

MC_PhasingAbsolute用于使得从轴相对于主轴产生延迟或提前，以实现绝对相位移动。它类似于暂时打开母轴上的耦合。主轴无法获悉从轴的绝对相位移动。

相位移动的“Phase”、“Velocity”、 “Acceleration”、“Deceleration”和“Jerk”由FB控制。“Deceleration”是一个限制值，与扭矩和速度的符号相反。

作为比较: MC_MoveSuperimposed也可以作用于从轴。以从轴的视角来看，MC_PhasingAbsolute作用于主轴。所以相位移动会影响从轴上的运动。

当轴由以下同步控制指令控制时，可以执行MC_PhasingAbsolute: MC_CamIn, MC_GearIn, MC_GearInPos, MC_CombineAxes。

MC_PhasingAbsolute的“PhaseShift”输入是总位移。如果同一主/从轴上的第二个“PhaseShift_2”大于第一个“PhaseShift_1”，则主轴将移动更多(PhaseShift_2 - PhaseShift_1)。如果它们相等且前一次的移位已经完成，主轴将不会移位。否则，主轴位置的移位将小于之前的相位移动。

请确保输入和输出参数为下表所示的数据类型。

主轴位置相对于实际物理位置移位。

这类似于将主轴上的联轴器打开一会儿，并用于将轴向其主轴延迟或推进。

以从轴视角进行相位移动。主轴无法获悉从轴经历了相位移动(从轴视角的MasterPos = PhysicalMasterPos + PhaseShiftValueSlaveAxis)。相位移动值具有位置偏移的特征)。

相位移动一直保持，直到另一个“Phasing”命令再次改变它。

输入/输出

名称数据类型描述
输入
ExecuteBOOL从上升沿开始相位过程。
MasterAXIS_REF参考主轴
SlaveAXIS_REF参考从轴
PhaseShiftLREAL从轴与主轴位置的绝对相位差[u]。
VelocityLREAL达到相位差的最大速度[u/s]。
AccelerationLREAL达到相位差的最大加速度[u/s2]
DecelerationLREAL达到相位差的最大减速度[u/s2]
JerkLREAL达到相位差的最大加加速度[u/s3]
输出
DoneBOOL命令相位达到
BusyBOOLFB尚未完成，预计会有新的输出值。
ActiveBOOL表示FB对轴具有控制权
CommandAbortedBOOL“Command”被另一个命令终止
ErrorBOOL表示在功能块中发生了错误
ErrorIDUINT错误标识
AbsolutePhaseShiftLREAL在设置“Busy”时连续显示绝对相位移动[u]。

例

例1

例
(*例如MC_PhasingAbsolute *)
MasterRef.AxisNo := 0;
SlaveRef.AxisNo := 3;

inst_PhasingAbsolute ( Execute := bExecute,  
                        Master := MasterRef,
                        Slave := SlaveRef,
                        PhaseShift := lrPhaseShift, 
                        Velocity:= lrVeloShift,
                        Acceleration:= lrAccShift, 
                        Deceleration:=lrDecShift, 
                        Jerk:=lrJerkShift );

bDone := inst_PhasingAbsolute.Done; 
bBusy := inst_PhasingAbsolute.Busy; 
bActive := inst_PhasingAbsolute.Active; 
bCommAborted := inst_PhasingAbsolute.CommandAborted; 
bError := inst_PhasingAbsolute.Error; 
ErrorId := inst_PhasingAbsolute.ErrorID;
lrAbsPhaseShift := inst_PhasingAbsolute.AbsolutePhaseShift;
