MC_MoveVelocity

以指定的速度进行不断地受控制的运动。

描述

MC_MoveVelocity用于控制轴以指定的速度连续移动。当“Velocity”为零时，轴将保持速度为0或将速度减速到0，状态机将保持在“连续运动”状态。

如果输入的“Acceleration”，“Deceleration”和“Jerk”为零，则这些值将被设置为最大值。

输入参数“Jerk”可用于平稳加速或减速。

请确保输入和输出参数为下表所示的数据类型。

为了停止这个动作，FB必须被另一个发出新命令的FB打断。

当一个块被另一个块终止时，信号“InVelocity”必须被重置。

负速度*负方向=正速度。

结合MC_MoveSuperimposed，只要这个FB (MC_MoveVelocity)对设定速度的贡献等于这个FB的命令速度，输出“InVelocity”就被设置。

输入/输出

名称数据类型描述
输入
ExecuteBOOL从上升沿开始运动
AxisAXIS_REF参考轴
ContinuousUpdateBOOL令功能块使用当前的输入变量值，并将该值应用于正在进行的运动
VelocityLREAL最大速度值。它是一个带符号的值
AccelerationLREAL加速度值(电机能量增加)
DecelerationLREAL减速度值(电机能量减少)
JerkLREAL加加速度值
DirectionMC_DIRECTION枚举类型(1 -3个值:)
mcPositiveDirection,
mcNegativeDirection,
mcCurrentDirection。
注意:最短路径不适用
BufferModeMC_BUFFER_MODE定义FB的时间顺序。枚举类型:mcAborting, mcBuffered。
输出
InVelocityBOOL命令速度达到
BusyBOOLFB尚未完成，预计会有新的输出值
ActiveBOOL表示FB对轴具有控制权
CommandAbortedBOOL“Command”被另一个命令终止
ErrorBOOL表示FB中发生了错误
ErrorIDUINT错误标识

例
VAR
 Inst_MC_MoveVelocity : MC_MoveVelocity ;
 mv1 : MC_MoveVelocity ;
 Inst_MC_MoveVelocity1 : MC_MoveVelocity ;
 mv2 : MC_MoveVelocity ;
 Inst_MC_Power : MC_Power ;
 mcp : MC_Power ;
 refAxis0 : lib:AXIS_REF ;
(*$desc=轴引用*)
En: BOOL:= FALSE;
(*$desc=开启轴电源*)
bPowerSta: BOOL值:= FALSE;
(*$desc=轴的电源状态*)
 mDir : MC_DIRECTION := mcPositiveDirection ;
 mBufferMode : MC_BUFFER_MODE := mcAborting ;
(*$desc=缓冲区模式设置*)
Go: BOOL:= FALSE;
(*$desc=开始运动*)
Abort: BOOL:= FALSE;
(*$desc=命令终止状态*)
 Next : BOOL := FALSE ;
(*$desc=启用下一个FB运动*)
Test: BOOL:= TRUE;
(*$desc=开始下一个FB运动，如果next = TRUE*)
END_VAR

1、时序图的左侧部分说明了在第1个FB完成后调用第2个FB的情况。如果第1个达到命令的速度3000，那么输出' mv1.Invelocity”和信号“Next”导致第2个FB移动到速度2000。在下一个周期中，“mv1.Invelocity”被重置且“mv1.CommandAborted”被设置。因此，第2个FB的“Execute”被重置。一旦轴达到“Velocity”2000，设置“mv2.Invelocity”。

2、时序图的右边部分展示了第2个 FB执行时，第1个 FB尚未完成速度初始化的情况。下面的序列显示:第1个运动在输入“mv1.Execute”时通过“Go”再次启动。当第1个FB仍在加速达到速度3000时，第1个 FB将被中断并中止，因为测试信号开始了第2个FB的运行。现在第2个FB运行并减速到2000。
