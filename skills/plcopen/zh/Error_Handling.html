错误处理

描述

当一个功能块被执行时，输入参数和指令处理将被检查是否有错误。如果功能块中发生错误，函数块的' Error '输出变量将变为TRUE。当执行运动控制FB时，也将检查PLCopen状态图。输出' Error '将被设置为错误状态。输出' ErrorID '将给出错误原因的详细信息。

如果轴在运动时发生轴错误，'CommandAborted '将被设置。可以从MC_ReadAxisError /MC_ReadStatus/MC_ReadAxisInfo中获取错误信息。MC_ReadAxisError中的输出' AxisErrorID '将给出详细的错误信息。轴错误由运动内核生成，状态机变为' Errorstop '。

本文档简要介绍了错误ID的结构，并概述了应用程序中产生的所有可能的错误消息。它将帮助用户在使用PLCopen运动控制功能块时进行故障排除。

用户可以查询有关“ErrorID”的更多信息，包括短名称、原因、反应以及如何解决错误等。

错误信息的结构

每个错误信息包含一个唯一的错误ID和一个简短的名称，然后给出尽可能准确地描述错误原因的进一步信息。错误信息还包含错误发生后的反应和建议的解决方法。

ErrorID是一个WORD类型的值。它分为两部分，一部分是由FBs产生的错误(ErrorID < 0x8000)，另一部分是由轴产生的错误(ErrorID > 0x8000)。

为了简化诊断，根据原因对错误消息进行分类。每个相应的错误类型都被分配一个唯一的标识符和一个数据包，该数据包总是提供特定的错误类型。

错误类型概述

PLCopen第一部分运动控制功能块的误差分为轴错误和非轴错误。

轴错误一般需要调用函数块MC_Reset来清除错误。如果轴处于运动状态，这些错误大多会导致轴停止。轴错误的ErrorID值大于0x8000，并且可以包含多个错误状态，每个位表示一个错误。

FBs中产生非轴错误的主要原因是无效输入。当FB的“使能”输入为TRUE时，或者在FB的“执行”输入的上升沿，可以通过提供有效的输入来清除它。非轴错误的ErrorID值小于0x8000，即使发生多个错误，也只能表示一个错误状态。非轴误差不影响PLCopen状态图和当前运动。

两种类型或错误都有三部分描述:描述，反应和解决方案。
