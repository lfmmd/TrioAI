MC_VelocityProfile

指定一个时间-速度固定的运动轮廓。

描述

这个功能块指定一个时间-速度固定的运动轮廓。运动轮廓的最终速度保持在最后一个速度值。状态保持为“ContinuousMotion”。

这个功能并不意味着它会重复执行一个运动轮廓。该功能允许在不同的运动轮廓之间切换。

MC_VelocityProfile只能在以下任何一种状态下执行:“SynchronizedMotion”、“DiscreteMotion”、“ContinuousMotion”或“Standstill”。

在“Execute”的上升沿成功触发，即没有错误，状态更改为“ContinuousMotion”。

速度将保持在速度轮廓的末尾。

如果“CommandAborted”更改为TRUE， “profilcompleted”，“Busy ”， “Error”和“Active”则RESET。

如果“CommandAborted”为FALSE时“Error”变为TRUE，“profilcompleted”，“Busy ”和“Active”将RESET。

当BufferMode = mcAborting时，“Active”更改为TRUE同时“Busy ”更改为TRUE。

当BufferMode = mcBuffered时，“Busy ”变为TRUE， “Active”'保持FALSE，直到当前FB正在运行。“Active”和“Busy ”在“Done”时变为FALSE， “CommandAborted”或' Error '变为TRUE。

MC_TV_REF的定义如下:

Time/Velocity对的内容可以用DeltaTime/Velocity表示，其中DeltaTime是两个连续点之间的时间差。Velocity可以是带符号的值。

STRUCT

作为此FB的替代方案，可以使用与虚拟主机耦合的CAM FB。

名称数据类型描述
输入

bExecute := FALSE;

                        Axis := Axis0Ref,(*AXIS_REF*)
                        TimeVelocity := TimeVeloTable,(*MC_TV_REF*)
                        TimeScale := rTimeScale,(*REAL*)
                        VelocityScale := rVeloScale,(*REAL*)

bDone := inst_VelocityProfile.ProfileCompleted ;

bError := inst_VelocityProfile.Error;
