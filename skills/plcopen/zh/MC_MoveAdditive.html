MC_MoveAdditive

指定一个受控制的运动一个指定的相对距离，在轴状态“DiscreteMotions”中添加最近的命令位置。

描述

MC_MoveAdditive命令在轴状态“DiscreteMotion”的最近命令位置之外，进行指定相对距离的受控运动。最近命令的位置可能由先前的MC_MoveAdditive运动被终止产生。

当轴状态为“DiscreteMotion”时，新的命令位置将是当前命令位置加上“Distance”。

在下面的例子中，当MC_MoveAbsolute移动到目标位置2000时，MC_MoveAdditive将被执行。“Axis”将其速度更改为50，并在3000处停止。

如果设定位置在当前 “Acceleration”、“Deceleration”_和“Jerk”输入参数下停止距离较短，则重新计算运动参数“Acceleration”、“Deceleration”和“Jerk”_。下图中所示的虚线表示没有MC_MoveAdditve中断的路径。

如果距离太短，无法使用最大加速、减速度和加加速度停止运动，则会使用快速加速、快速减速度，忽略_“jerk”_。

蓝色虚线表示没有足够的距离达到目标速度输入参数。在到达目标速度之前，轴会降低速度。

粗线表示轴将没有加速阶段，因为离目标位置的距离很短。

绿色虚线表示没有足够的剩余距离来停止。停止时将在减速阶段将使用最大速度。

如果输入的“Distance”为负值(假设之前的速度为正值)，并且目标位置在当前运动方向的相反侧，则运动将停止并以相反的方向移动到目标位置。

达到“Velocity”是没有必要的。

当块被另一个块终止或轴状态由于一些错误而改变为其他状态时，“CommandAborted”更改为TRUE，当_“Error ”更改为_TRUE_或“ Execute”_更改为FALSE时，' CommandAborted '更改为FALSE。如果' Execute '为FALSE时“CommandAborted”变为TRUE， “CommandAborted”将保持TRUE至少一个周期。

确保输入和输出参数为下表所示的数据类型。

如果FB在轴状态“ContinuousMotion”下被激活，则在执行时将指定的相对距离添加到设置的位置。轴状态将变为“DiscreteMotion”。

输入/输出

名称数据类型描述
输入
ExecuteBOOL在上升沿开始动作
AxisAXIS_REF参考轴
ContinuousUpdateBOOLTRUE:当功能块被触发(“Execute”上升沿)时，只要它保持TRUE ，将使功能块使用输入变量的当前值并将其应用于正在进行的运动。
FALSE:使用“Execute”输入的上升沿，在整个移动过程中忽略输入参数的变化。
DistanceLREAL运动的相对距离
VelocityLREAL最大速度值(不一定达到)
AccelerationLREAL加速度值
DecelerationLREAL减速度值
JerkLREAL加加速度值
BufferModeMC_BUFFER_MODE定义FB的时间顺序。
输出
DoneBOOL到达命令距离
BusyBOOLFB尚未完成，预计会有新的输出值
ActiveBOOL表示FB对轴具有控制权
CommandAbortedBOOL“Command”被另一个命令终止
ErrorBOOL指示FB内部发生错误

例1

在这个例子中，运动由MC_MoveAdditive控制。它与“mcAborting”缓冲模式一起工作，在第一个动作完成后触发第二个执行触发器。时序图与MC_MoveRelative相同。
例1
VAR
 Inst_MC_Power : MC_Power ;
 Inst_MC_MoveAdditive : MC_MoveAdditive ;
 refAxis0 : lib:AXIS_REF ;
 bPowerOn : BOOL := FALSE ;
  (*$desc=启用轴0变量*)
 bExeAdditive : BOOL := FALSE ;
(*$desc=开始移动添加运动*)
 mBufferMode : MC_BUFFER_MODE := mcAborting ;
(*$desc= mcAborting缓冲模式*)
 PowerState : BOOL := FALSE ;
(*$desc=轴0电源状态*)
 bDone : BOOL ;
(*$desc=添加运动已完成*)
 bBusy : BOOL ;
 bActive : BOOL ;
 bAborted : BOOL ;
 bError : BOOL ;
 ErrorId : UINT ;
END_VAR
