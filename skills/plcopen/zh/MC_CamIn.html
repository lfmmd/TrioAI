描述

有两个常见的选项中止凸轮，在MC_CamIn被调用后，MC_CamOut和MC_Halt停止凸轮，尽管其他单轴FBs如MC_MoveAbsolute和MC_MoveRelative可以被调用。当调用MC_CamOut或MC_Halt时，两个轴之间的主从关系结束。

当MC_CamOut被调用时，它将以从轴的瞬时最终实际速度继续运动，并且轴运动继续以最终实际速度进行。

MC_Halt将轴运动的速度减速到0并停止运动。

“_MasterOffset”_输入参数可用于设置主轴对凸轮的偏移量。它是一个带符号的值。

指定凸轮表的起始点(0相位)为主轴的绝对位置。单位为命令单位。

'_MasterStartDistance'_用于指定从轴开始凸轮运动时主轴的位置。当使用' StartMode '指定绝对定位时，指定主轴的绝对位置。

'MasterSyncPosition '指定主轴在CAM轮廓中的绝对位置，从轴与主轴同步。它是一个带符号的值。如果该值超出范围，则会设置_'Error_ '。默认值为零。

‘StartMode’_输入参数可以设置为'_mcAbsolute'或'mcreative'。

请确保输入和输出参数为下表所示的数据类型。

“Acceleration”，“Deceleration”，“Jerk”不能作为MC_CamIn的输入。这些值可以使用MC_WriteParameter和MC_ReadParameter进行设置和读取。

主轴不要求是静止的。

当执行MC_CamIn时，如果实际的主从位置不对应于偏移值，则会发生错误，或者系统会自动处理差异。

Cam的位置相对于当前主从位置为Absolute或Relative。Absolute:主从之间的轮廓处于绝对模式。Relative:主从之间的轮廓处于相对模式。

MC_CamIn不与MC_CamTableSelect合并，因为这种分离允许随时调整。

从轴偏移量的机械类比是一个附加层厚度恒定的凸轮焊接工艺。因此，从轴位置具有恒定的偏移量，如果假设凸轮从轴的方向是线性的，则可以将偏移量解释为主轴的轴偏移量。

//取消MTYPE中所有之前的运动

Inst_TC_CANCEL(TRUE,0,0);

Inst_TC_CANCEL1(TRUE,1,0);



//凸轮轴配置

TCW_UNITS(0,1);

TCW_SPEED(0,10);

TCW_ACCEL(0,10);

TCW_DECEL(0,10);

Inst_TC_DEFPOS0(TRUE,0,0);

TCW_SERVO(0,TRUE);



//主轴配置

TCW_UNITS(1,1);

TCW_SPEED(1,1);

TCW_ACCEL(1,10);

TCW_DECEL(1,10);

Inst_TC_DEFPOS1(TRUE,1,0);

TCW_SERVO(1,TRUE);



TCW_WDOG(1);



//设置凸轮表

CamTable.TableStart:=200;

CamTable.TableFinish:=211;

Inst_MC_CamTableSelect(TRUE, CamTable, FALSE, FALSE, FALSE);

CamTableId:=Inst_MC_CamTableSelect.CamTableID;



// MC_CamIn配置

master.AxisNo:=1;

Slave.AxisNo:=0;

StartMode:=mcAbsolute;

MasterValueSource:=mcCommandedValue;

Inst_MC_CamIn(TRUE, Master, Slave, 0, 0, 1, 1, 0, 0, StartMode, MasterValueSource, CamTableID);



// MC_CamOut设置

bCamOutExecute:=FALSE;

Inst_MC_CamOut(bCamOutExecute, Slave);



//运行主轴

Inst_TC_FORWARD(TRUE,1);
