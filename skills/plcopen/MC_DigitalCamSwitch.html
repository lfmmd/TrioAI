MC_DigitalCamSwitch

Commands a group of discrete output bits to switch in analogy to a set of mechanical cam-controlled switches connected to an axis.

Description:

MC_DigitalCamSwitch is the analogy to switches on a motor shaft: it commands a group of discrete output bits to switch in analogy to a set of mechanical cam-controlled switches connected to an axis. Forward and backward movements are allowed.

A track is one cam that contains a set of switches related to one output. ‘TrackNumber’ corresponds to the cam number. When ‘Enable’ is TRUE, ‘EnableMask’ specifies whether each track is enabled or disabled. Bit 0 corresponds to track 0 and bit 15 corresponds to track 15. The corresponding track is enabled if a bit is set to 1 and disabled if the bit is set to 0. If the value of a bit changes from 1 to 0, the output of the corresponding track will turn OFF. This functionality is sometimes called PLS – Phase or Position or Programmable Limit Switch.

Upon rising edge of ‘Enable’, the parameters for the configured ‘Switches’, ‘Outputs’, and ‘TrackOptions’ are initialized. Internal switching begins immediately following the initialization.

Upon rising edge of "EnableMask", the digital output will be enabled. The output will be set and reset according to the configured ‘Switches’ and ‘TrackOptions’. When outputs are disabled, the outputs are reset.

‘EnableMask’ is used to enable and disable the corresponding track. If ‘FirstOnPositon[n]’ equal to ‘LastOnPosition[0]’, the corresponding output defined in ‘OutputNumber[n]’ will always be set.

‘AxisDirection’ is used to define the resource which the direction of digital cam is based on.

‘Duration’ is used when ‘CamSwitchMode’ is set to ‘Time based’.

The above timing diagram is ‘Position based’, and ‘AxisDirection’ is ‘Positive (1)’ or ‘Both (0)’. The axis is moving in constant velocity.

‘Duration’ is used when the ‘CamSwitchMode’ is time based. It specifies the output time after the corresponding ’OutputNumber’ is set to TRUE. It is a non-negative value.

The above diagram shows the outputs of the first three tracks with ‘OffCompensation’ for all tracks, and ‘OnCompensation’ for the second track.

‘OffCompensation’ is used to compensate output time delay for the outputs switching off. ‘OnCompensation’ is used to compensate output time delay for switching on. The index for ‘OffCompensation’ and ‘OnCompensation’ is used for corresponding track number. It can be positive or negative value. The time is delayed for a positive value and advanced for a negative value.

The negative time value will be calculated according to the actual velocity. ‘OnCompensation’ and ‘OffCompensation’ can be used in both ‘CamSwitchMode’ mode ‘Position based’ and ‘Time based’ and any direction of ‘AxisDircetion’. The output time for a track T track is calculated as follows:

 

T track = T out – T OnCompensation + T OffCompensation

 

T track: The output time for a track

T out: The output time for a track without compensation

T OnCompensation: The compensation time for ‘OnCompensation’

T OffCompensation: The compensation time for ‘OffCompensation’

 

 

Important

MC_DigitalCamSwitch is always used when the axis is moving in constant velocity.

If the axis is set as a rotation axis, the following operation will occur for ‘OnCompensiation’ or ‘OffCompensation’ and the axis changes abruptly.

Long time value for ‘OnCompensation’ or ‘OffCompensation’ may cause unstable output timing, such as the time is equivalent to the time for a rotation.

If the value of ‘OnCompensation’ results in exceeding ‘LastOnPosition’ point + ’OffCompensation’, the output timing will be unstable.

If the value of ‘OffCompensation’ results in execeeding ‘FirstOnPosition’ point + ‘OnCompensation’, the output timing will be unstable.

 

The above diagram provides an example of how the hysteresis function can be used. Specifying a hysteresis window prevents an unwanted series of pulses in the area around the switching position. The output corresponds to track 0 is set at the position ‘FirstOnPosition’ and ‘LastOnPosition’ are changed to ‘FirstOnPosition+Hysteresis’ and ‘LastOnPosition+Hysteresis’.

Example of an axis moving in the positive direction:

At point a, the axis position reaches the configured "FirstOnPosition". The output is turned on when the position "FirstOnPosition + Hysteresis" is reached.

At point b, the axis position reaches the turn-off position "LastOnPosition". The output is turned off when the position "LastOnPosition + Hysteresis" is reached.

‘Hysteresis’ is to avoid multiple switching around the switching point.

Make sure the input and output parameters are of data type indicated in the table below.

MC_CAMSWITCH_REF is a switch structure for use as ON/OFF switch pattern data.

STRUCT

   TrackNumber : ARRAY[0..15] of UINT; (*TrackNumber is    the reference to the track*)

   OutputNumber : ARRAY[0..15] of UINT; (* Output number    *)

   FirstOnPosition : ARRAY[0..15] of LREAL;

   LastOnPosition : ARRAY[0..15] of LREAL;

   AxisDirection : ARRAY[0..15] of MC_AXIS_DIRECTION;(* 0:    Both directions, 1:Positive direction, 2:Negative    direction *)

   CamSwitchMode : ARRAY[0..15] of MC_CAMSWITCH_MODE;(*    Default 0:Position based; 1:Time based *)

   Duration:  ARRAY[0..15] of TIME;

END_STRUCT

MC_TRACK_REF is a specific structure containing the track properties.

STRUCT

   OnCompensation: ARRAY[0..15] TIME;(* Compensation time    with which the switching on is delayed in time per    track.*)

   OffCompensation: ARRAY[0..15] TIME;(*Time compensation    the switching off is delayed per track.*)

   Hysteresis: ARRAY[0..15]LREAL; (*Distance from the    switching point (in positive and negative direction) in    which the switch is not executed until the axis has    left this area, in order to avoid multiple switching    aroundthe switching point.*)

END_STRUCT

Note: Every ‘Output[x]’ can be bound to any physical output port, even to the same one (The last track with the same output as the one before will cover the output result). In this example, they are bound to different ports.

The example below uses the values from the example above for ‘Switches’ (MC_CAMSWITCH_REF data type). It does not use ‘OnCompensation’ / ‘OffCompensation’ and ‘Hysteresis’. It is the behaviour of the outputs when the axis is moving continuously in the positive direction. The axis is a rotation axis with a modular length of 5000 units.

Set ‘EnableDigiCam’ to activate the FB ‘Inst_MC_DigitalCamSwitch’. Switching data and track options can be initialized while the FB is enabled. If no errors occur, the output in motion controller is activated and the machine enters STATE_READY state.

Axis0 can be made to move positive or negative. It is possible to observe the action of commanded outputs.


 

 

 Example

(* Defines for MC_DigitalCamSwitch example *)
#define STATE_READY      0
#define STATE_CAMIN      1
#define STATE_DIGITALCAM 2
#define NEXT_STEP        3
#define STATE_MOVE_POS   9
#define STATE_MOVE_NEG   6
#define STATE_STOP       5
#define ERROR_STEP       7
#define STATE_ERROR      8

VAR
 state : USINT ;
 RefTrack : lib:MC_TRACK_REF ;
 RefSwith : lib:MC_CAMSWITCH_REF ;
 EnableDigiCam : BOOL := FALSE ;
 Inst_MC_DigitalCamSwitch : MC_DigitalCamSwitch ;
 mcSource : MC_SOURCE ;
 Axis0Ref : lib:AXIS_REF ;
 MovePositive : BOOL := TRUE ;
   (*$desc=Move positive direction enable variable*)
 Inst_MC_MoveVelocity1 : MC_MoveVelocity ;
 mDirection : MC_DIRECTION ;
 BufferMode : MC_BUFFER_MODE ;
 ErrorID0 : UINT ;
 MoveNegative : BOOL := FALSE ;
   (*$desc=Enable variable for negative movement*)
 bStop : BOOL := TRUE ;
   (*$desc=To stop motion, it is bound to IO1*)
   (*$profile=IO*)
   (*$embed=(VERS=1    MODE=0    POINT=1)*)
 Inst_MC_Stop4 : MC_Stop ;
END_VAR

(* Variables initialization *)

(******************** SWITCH SETUP*********************)
RefSwith.TrackNumber[ 0 ] := UINT#0 ;
RefSwith.OutputNumber[ 0 ] := UINT#0 ;
RefSwith.FirstOnPosition[ 0 ] := LREAL#2000.0 ;
RefSwith.LastOnPosition[ 0 ] := LREAL#3000.0 ;
RefSwith.AxisDirection[ 0 ] := mcPositive;
RefSwith.Duration[ 0 ] := time#0 ;
RefSwith.CamSwitchMode[ 0 ] := mcPosition;

RefSwith.TrackNumber[ 1 ] := UINT#1 ;
RefSwith.OutputNumber[ 1 ] := UINT#1 ;
RefSwith.FirstOnPosition[ 1 ] := LREAL#2500.0 ;
RefSwith.LastOnPosition[ 1 ] := LREAL#3000.0 ;
RefSwith.AxisDirection[ 1 ] := mcNegative;
RefSwith.Duration[ 1 ] := time#0 ;
RefSwith.CamSwitchMode[ 1 ] := mcPosition;

RefSwith.TrackNumber[ 2 ] := UINT#2 ;
RefSwith.OutputNumber[ 2 ] := UINT#2 ;
RefSwith.FirstOnPosition[ 2 ] := LREAL#3500.0 ;
RefSwith.LastOnPosition[ 2 ] := LREAL#4000.0 ;
RefSwith.AxisDirection[ 2 ] := mcBoth;
RefSwith.Duration[ 2 ] := time#0 ;
RefSwith.CamSwitchMode[ 2 ] := mcPosition;

RefSwith.TrackNumber[ 3 ] := UINT#3 ;
RefSwith.OutputNumber[ 3 ] := UINT#3 ;
RefSwith.FirstOnPosition[ 3 ] := LREAL#5000.0 ;
RefSwith.LastOnPosition[ 3 ] := LREAL#1000.0 ;
RefSwith.AxisDirection[ 3 ] := mcBoth;
RefSwith.Duration[ 3 ] := time#0 ;
RefSwith.CamSwitchMode[ 3 ] := mcPosition;

(******************** TRACK SETUP*********************)
RefTrack.OnCompensation[ 0 ] := T#0 ;
RefTrack.OffCompensation[ 0 ] := t#0 ;
RefTrack.Hysteresis[ 0 ] := 0 ;

RefTrack.OnCompensation[ 1 ] := T#0 ;
RefTrack.OffCompensation[ 1 ] := t#0 ;
RefTrack.Hysteresis[ 1 ] := 0 ;

RefTrack.OnCompensation[ 2 ] := T#0 ;
RefTrack.OffCompensation[ 2 ] := t#0 ;
RefTrack.Hysteresis[ 2 ] := 0 ;

RefTrack.OnCompensation[ 3 ] := T#0 ;
RefTrack.OffCompensation[ 3 ] := t#0 ;
RefTrack.Hysteresis[ 3 ] := 0 ;

mcSource := mcCommandedValue;
Axis0Ref.AxisNo := 0 ;
mDirection := mcPositiveDirection;
BufferMode := mcAborting;

(******************* cyclic program*********************)
CASE State OF
     (*power and home axis before moving... *)
     (********************STATE_READY****************************)
    STATE_READY:
     IF (EnableDigiCam) THEN  
        Inst_MC_DigitalCamSwitch( Enable := TRUE (*BOOL*) , 
                                    Axis := Axis0Ref (*AXIS_REF*) , 
                                    Switches := RefSwith (*MC_CAMSWITCH_REF*) , 
                                    TrackOptions := RefTrack (*MC_TRACK_REF*) ,         
                                    EnableMask := UINT#15 (*UINT*) , 
                                    ValueSource := mcSource (*MC_SOURCE*) );
        
        State:=STATE_DIGITALCAM;
     END_IF ;
     (********************STATE_DIGITALCAM****************************)
    STATE_DIGITALCAM: 
     IF (Inst_MC_DigitalCamSwitch.InOperation) AND (MovePositive = TRUE ) THEN         
        mDirection := mcPositiveDirection;
        
        Inst_MC_MoveVelocity1( Execute := FALSE (*BOOL*) , 
                                Axis := Axis0Ref (*AXIS_REF*) ,         
                                ContinuousUpdate := TRUE (*BOOL*) , 
                                Velocity := LREAL#100.0 (*LREAL*) , 
                                Acceleration := LREAL#1000.0 (*LREAL*) , 
                                Deceleration := LREAL#1000.0 (*LREAL*) , 
                                Jerk := LREAL#500.0 (*LREAL*) , 
                                Direction := mDirection (*MC_DIRECTION*) , 
                                BufferMode := BufferMode (*MC_BUFFER_MODE*) );
        
        
        Inst_MC_MoveVelocity1( Execute := TRUE (*BOOL*) , 
                                Axis := Axis0Ref (*AXIS_REF*) ,         
                                ContinuousUpdate := TRUE (*BOOL*) , 
                                Velocity := LREAL#100.0 (*LREAL*) , 
                                Acceleration := LREAL#1000.0 (*LREAL*) , 
                                Deceleration := LREAL#1000.0 (*LREAL*) , 
                                Jerk := LREAL#500.0 (*LREAL*) , 
                                Direction := mDirection (*MC_DIRECTION*) , 
                                BufferMode := BufferMode (*MC_BUFFER_MODE*) );
        
        State := STATE_MOVE_POS;
     ELSIF (Inst_MC_DigitalCamSwitch.Error) THEN
         (*Store ErrorID in a variable*)
        ErrorID0:=Inst_MC_DigitalCamSwitch.ErrorID;
        State:= STATE_ERROR;
     END_IF ;
     (********************STATE_MOVE_POS****************************)
    STATE_MOVE_POS:    
     IF (Inst_MC_MoveVelocity1.InVelocity) AND (MoveNegative = TRUE ) THEN
        mDirection := mcNegativeDirection;
        
        Inst_MC_MoveVelocity1( Execute := FALSE (*BOOL*) ,
                                Axis := Axis0Ref (*AXIS_REF*) , 
                                ContinuousUpdate := TRUE (*BOOL*) , 
                                Velocity := LREAL#100.0 (*LREAL*) , 
                                Acceleration := LREAL#1000.0 (*LREAL*) , 
                                Deceleration := LREAL#1000.0 (*LREAL*) , 
                                Jerk := LREAL#500.0 (*LREAL*) , 
                                Direction := mDirection (*MC_DIRECTION*) , 
                                BufferMode := BufferMode (*MC_BUFFER_MODE*) );
        
        Inst_MC_MoveVelocity1( Execute := TRUE (*BOOL*) ,  
                                Axis := Axis0Ref (*AXIS_REF*) ,
                                ContinuousUpdate := TRUE (*BOOL*) , 
                                Velocity := LREAL#100.0 (*LREAL*) , 
                                Acceleration := LREAL#1000.0 (*LREAL*) , 
                                Deceleration := LREAL#1000.0 (*LREAL*) , 
                                Jerk := LREAL#500.0 (*LREAL*) , 
                                Direction := mDirection (*MC_DIRECTION*) , 
                                BufferMode := BufferMode (*MC_BUFFER_MODE*) );
        
        State:=STATE_MOVE_NEG;
     ELSIF (Inst_MC_MoveVelocity1.Error) THEN
         (*Store ErrorID in a variable*)
        ErrorID0:=Inst_MC_MoveVelocity1.ErrorID;
        State:= STATE_ERROR;
     END_IF ;
     (********************STATE_MOVE_NEG****************************)
    STATE_MOVE_NEG:
     IF ((Inst_MC_MoveVelocity1.InVelocity) AND (bStop = TRUE )) THEN
        Inst_MC_Stop4( Execute := FALSE (*BOOL*) , 
                        Axis := Axis0Ref (*AXIS_REF*) , 
                        Deceleration := LREAL#2000.0 (*LREAL*) , 
                        Jerk := LREAL#1000.0 (*LREAL*) );
        
        Inst_MC_Stop4( Execute := TRUE (*BOOL*) , 
                        Axis := Axis0Ref (*AXIS_REF*) , 
                        Deceleration := LREAL#2000.0 (*LREAL*) , 
                        Jerk := LREAL#1000.0 (*LREAL*) );
        
        State:= STATE_STOP;
     ELSIF (Inst_MC_MoveVelocity1.Error) THEN
         (*Store ErrorID in a variable*)
        ErrorID0:=Inst_MC_MoveVelocity1.ErrorID;
        State:= STATE_ERROR;
     END_IF ;
     (********************STATE_STOP****************************)
    STATE_STOP:   (*stop movement*)
     IF (Inst_MC_Stop4.Done = TRUE ) THEN
        Inst_MC_Stop4( Execute := FALSE (*BOOL*) ,
                        Axis := Axis0Ref (*AXIS_REF*) ,  
                        Deceleration := LREAL#2000.0 (*LREAL*) , 
                        Jerk := LREAL#1000.0 (*LREAL*) );
        State:=NEXT_STEP;
     ELSIF (Inst_MC_Stop4.Error = TRUE ) THEN
         (*Store ErrorID in a variable,*)
        ErrorID0:= Inst_MC_Stop4.ErrorID;
        Inst_MC_Stop4( Execute := FALSE (*BOOL*) , 
                        Axis := Axis0Ref (*AXIS_REF*) , 
                        Deceleration := LREAL#2000.0 (*LREAL*) , 
                        Jerk := LREAL#1000.0 (*LREAL*) );
        State:=ERROR_STEP;        
     END_IF ;

    ERROR_STEP:
     (* Adding code here, when motion got errors *)
    
    NEXT_STEP:
     (* Adding code here, after MC_Stop calling *)

END_CASE ;
