MC_MoveAdditive

Commands a controlled motion of a specified relative distance additional to the most recent commanded position in the axis state 'DiscreteMotion'.

Description:

MC_MoveAdditive commands a controlled motion of a specified relative distance additional to the most recent commanded position in the axis state 'DiscreteMotion'. The most recent commanded position may be the result of a previous MC_MoveAdditive motion which was aborted.

When the axis state is 'DiscreteMotion', the new command position will be current command position additional to ‘Distance’ .

In the example below, MC_MoveAdditive will be executed while MC_MoveAbsolute is ongoing to the target position 2000. The ‘Axis’ will change its speed to 50 and stop at 3000.

 Example

 

When the axis state is in 'ContinuousMotion', the new command position will be current set position at the time of execution additional to ‘ Distance’. MC_MoveVelocity will be aborted by MC_MoveAdditive. The PLCopen state will change from ‘ContinousMotion’ to ‘DiscreteMotion’.

In the following example, MC_MoveVelocity will be executed while MC_MoveAbsolute is ongoing to when the axis moving to position ‘P’. The ‘Axis’ will change its speed to 50 and stop at ‘P+1000’.

 Example

 

 

If the set position has shorter distance for stopping with inputs parameters the ‘Acceleration’, ‘Deceleration’ and ‘Jerk’, the motion parameters, ‘Acceleration’, ‘Deceleration’ or ‘Jerk’ will be recalculated. The dash line shows in the following picture means the path without MC_MoveAdditve interruption.

If the distance is too short for stopping using the maximum acceleration, deceleration and jerk, rapid acceleration, deceleration will be used, and ‘Jerk’ will be ignored.

The blue dotted line means there is not enough distance to reach the target velocity input parameter. The axis will decelerate the velocity before reaching the target velocity.

The bold line means that the axis will have no acceleration stage because the short distance left for the target position.

The green dotted line means that there is not enough distance left for stopping. The deceleration will use the maximum velocity for stopping.

If the input ‘Distance’ is a negative value (assumed previous velocity is positive value), and target position is in the reverse side of current motion direction, the motion will stop and move with the reverse direction to the target position.

‘Velocity’ is not necessary to reached.

‘CommandAborted’ changes to TRUE when the block is aborted by another block or the axis state changes to other state because of some errors, and changes to FALSE when one of ‘Error’ changes to TRUE or ‘Execute’ changes to FALSE. If ‘CommandAborted’ changes to TRUE when ‘Execute’ has been FALSE, ‘CommandAborted’ will stay TRUE for at least one cycle.

Make sure the input and output parameters are of data type indicated in the table below.

If the FB is activated in the axis state 'ContinuousMotion', the specified relative distance is added to the set position at the time of the execution. The axis state will change to ‘DiscreteMotion’.

Examples:

Example 1:

In this example, the motion is controlled by MC_MoveAdditive. It works with ‘mcAborting’ buffer mode and the second execution trigger is trigged after the first motion finished. The timing diagram is same as MC_MoveRelative .

 Example 1

VAR
 Inst_MC_Power : MC_Power ;
 Inst_MC_MoveAdditive : MC_MoveAdditive ;
 refAxis0 : lib:AXIS_REF ;
 bPowerOn : BOOL := FALSE ;
   (*$desc=Enable axis 0 variable*)
 bExeAdditive : BOOL := FALSE ;
   (*$desc=Start to move additive motion*)
 mBufferMode : MC_BUFFER_MODE := mcAborting ;
   (*$desc=Buffer mode mcAborting*)
 PowerState : BOOL := FALSE ;
   (*$desc=Axis 0 power status*)
 bDone : BOOL ;
   (*$desc=Additive motion is finished*)
 bBusy : BOOL ;
 bActive : BOOL ;
 bAborted : BOOL ;
 bError : BOOL ;
 ErrorId : UINT ;
END_VAR

Example 2:

In the following example, the motion is controlled by two MC_MoveAdditive FBs. They work with ‘mcAborting’ buffer mode. The 2 nd MC_MoveAdditive is triggered while the first motion is ongoing and has not finished. This is evident in the timing diagram below. ‘CommandAborted’ is set in the first FB.

 Example 2

VAR
 Inst_MC_Power : MC_Power ;
 refAxis : lib:AXIS_REF ;
   (*$desc=Axis reference*)
 bPowerEn : BOOL := FALSE ;
   (*$desc=To enable axis power*)
 bPowerSta : BOOL := FALSE ;
   (*$desc=The axis power status*)
 Inst_MC_MoveAdditive : MC_MoveAdditive ;
 exmove1 : BOOL := FALSE ;
   (*$desc=To start the first FB motion*)
 mBufferMode : MC_BUFFER_MODE := mcAborting ;
   (*$desc=FB buffer mode setting with default value mcAborting*)
 done1 : BOOL := FALSE ;
   (*$desc=Does first FB finish the motion*)
 busy1 : BOOL := FALSE ;
   (*$desc=Is the FB working*)
 active1 : BOOL := FALSE ;
   (*$desc=Is the FB actived*)
 abort1 : BOOL := FALSE ;
   (*$desc=Is the FB motion aborted*)
 error1 : BOOL := FALSE ;
   (*$desc=Is there an error in FB*)
 Inst_MC_MoveAdditive1 : MC_MoveAdditive ;
 exmove2 : BOOL := FALSE ;
   (*$desc=To start the second FB motion*)
 done2 : BOOL ;
   (*$desc=Does the second FB finish the motion*)
 busy2 : BOOL := FALSE ;
   (*$desc=Is the second FB working*)
 active2 : BOOL := FALSE ;
   (*$desc=is it actived*)
 abort2 : BOOL := FALSE ;
   (*$desc=Is it aborted*)
END_VAR

Example 3:

A sheet of glass is fed on a conveyor and is required to be stopped 250mm after the leading edge is sensed by a proximity switch.  The proximity switch is connected to the input 1.

 

 Example3

 mDir : MC_DIRECTION := mcPositiveDirection ;
   (*$desc=Motion direction*)

   (*$desc=Buffer mode for the FB*)
 bInVelo : BOOL ;
   (*$desc=The status of the axis motion*)
 Sensor_Input1 : INT := INT#3 ;
   (*$desc=The input check for the additive motion trigger*)
