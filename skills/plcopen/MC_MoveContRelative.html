MC_MoveContRelative

Controls a motion to a specified relative distance with a specified end velocity.

Description:

MC_MoveContRelative commands a controlled motion to a specified relative distance with a specified end velocity. If the commanded position is reached and no new motion command is put into the buffer, the axis continuous to run with the specified 'EndVelocity'.

You can set ‘EndVelocity’ to a negative value. The process is divided into two steps: ‘Deceleration’ and ‘Acceleration’ steps.

If ‘EndVelocity’ is set to 0, MC_MoveContRelative functions the same as MC_MoveRelative.

Make sure the input and output parameters are of data type indicated in the table below.

The axis will change to state ‘ContinuousMotion’ (meaning it will not stop by itself) after it is executed.

Negative velocity * negative direction = positive velocity.

‘mcShortedWay’ is used for modulo axis motion.

FALSE: with the rising edge of the ‘Execute’ input, a change in the input parameters is ignored during the whole movement.

Examples:

Example 1:

This example shows two MC_MoveContRelative and one MC_MoveRelative executing with ‘BufferMode = mcAborting’. The FBD program shows in below.

 Example

VAR
 Inst_MC_Power : MC_Power ;
 mcr1 : MC_MoveContRelative ;
 mcr2 : MC_MoveContRelative ;
 mr1 : MC_MoveRelative ;
 RefAxis : lib:AXIS_REF ;
 bPower : BOOL := FALSE ;
   (*$desc=Power on the refAxis*)
 Start : BOOL := FALSE ;
   (*$desc=To start the motion*)
 mBufferMode : MC_BUFFER_MODE := mcAborting ;
   (*$desc=Buffer mode variable with mcAborting init value*)
 Inst_MC_MoveRelative : MC_MoveRelative ;
 bPowerStatus : BOOL := FALSE ;
   (*$desc=Axis power  on status*)
END_VAR
