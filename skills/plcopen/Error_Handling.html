Error Handling

Description:

When a Function Block is executed, input parameters and instruction processing will be checked for errors and the Error output of a function block changes to TRUE if an error occurs in the function block. The PLCopen state diagram will be checked as well when a motion control FB is executed. The output Error will be set for the error state. The output ErrorID will give the detail information for error reason.

If an axis error occurs while the axis is in motion, the CommandAborted will be set. You can get the error information from MC_ReadAxisError / MC_ReadStatus / MC_ReadAxisInfo . The output AxisErrorID in MC_ReadAxisError will give the detail error information. Axis error is generated by motion kernel, the state machine changes to ‘Errorstop’.

This document briefly describes the structure of the error ID and provides overview of all possible error messages generated in application. It will help users in troubleshooting process when using PLCopen Motion Control function blocks.

Users to look up further information about ErrorID, including the short name, the cause, the reaction and how to solve the error and so on.

Structure of an Error Message:

Every error message contains a unique error ID and a short name, then given further information which describes the cause of the error as exactly as possible. The error message also contains the reaction after occurrence of the error and the recommended method of how to solve it.

ErrorID is a WORD type value. It is divided into two parts, one part for errors are generated from FBs ( ErrorID < 0x8000), the other from axis errors ( ErrorID > 0x8000).

To simplify diagnostics, the error messages are categorised based on causes. Every corresponding error type is assigned a unique identifier and a data packet that is always provided with the specific error type.

Overview of Error Types:

Concerning to errors in PLCopen part 1 motion control function blocks, error types are divided into axis errors and non-axis errors .

Axis errors generally need to call function block MC_Reset to clear the errors. These errors will mostly cause axis to stop if axis is in motion. ErrorID value of axis errors is greater than 0x8000 and can include more than one error state with each bit represents an error.

Non-axis errors are generated in FBs mostly because of invalid inputs. It can be cleared by providing valid inputs when a FB’s Enable input is TRUE, or at the rising edge of a FB’s Execute input. ErrorID value of non-axis errors is less than 0x8000 and can only represent an error state even though more than one error may have occurred. Non-axis errors do not affect the PLCopen state diagram and current motion.

Both types or error have three-part description: Description, Reaction and Solution.
