MC_PhasingAbsolute

Creates an absolute phase shift in the master position of a slave axis.

Description

MC_PhasingAbsolute is used to delay or advance an axis to its master for an absolute phase shift. It is analogous to opening the coupling on the master for a moment. The master does not know the absolute phase shift.

‘Phase’, ‘Velocity’, ‘Acceleration’, ‘Deceleration’ and ‘Jerk’ of a phase shift are controlled by the FB. ‘Deceleration’ is a limit and is applicable with opposite signs of torque and velocity.

For comparison: MC_MoveSuperimposed could also be used to act on the slave axis. MC_PhasingAbsolute acts on the master side, as seen from the slave. So phase shifting will affect the motion on slave axes.

MC_PhasingAbsolute can be executed when axis is controlled by the following synchronised control instructions: MC_CamIn, MC_GearIn, MC_GearInPos, MC_CombineAxes.

The ‘PhaseShift’ input for MC_PhasingAbsolute is the total shifting. If the second ‘PhaseShift_2’ is larger than the first ‘PhaseShift_1’ on the same master / slave axis, the master axis will shift more (PhaseShift_2 - PhaseShift_1). If they are equal, the master will not shift if the previous shift has been completed. Otherwise, the master position will be shifted less than the previous phase shift.

Make sure the input and output parameters are of data type indicated in the table below.

The master position is shifted in relation to the real physical position.

This is analogous to opening a coupling on the master shaft for a moment and is used to delay or advance an axis to its master.

The phase shift is seen from the slave. The master does not know that there is a phase shift experienced by the slave (MasterPos as seen from the SlaveAxis = PhysicalMasterPos + PhaseShiftValueSlaveAxis. The phase shift value has the character of a position offset).

The phase shift remains until another ‘Phasing’ command changes it again.

Inputs/Outputs

NameData TypeDescription
INPUTS
ExecuteBOOLStart the phasing process at the rising edge.

SlaveAXIS_REFReference to the slave axis

VelocityLREALMaximum Velocity to reach phase difference [u/s].
AccelerationLREALMaximum acceleration to reach phase difference [u/s2]

OUTPUTS
DoneBOOLCommanded phasing reached
BusyBOOLThe FB is not finished and new output value is to be expected.

CommandAbortedBOOL‘Command’ is aborted by another command
ErrorBOOLSignals that an error has occurred within the Function Block
ErrorIDUINTError identification
AbsolutePhaseShiftLREALDisplays continuously the absolute phase shift [u] while ‘Busy’ is set.

Examples

Example 1

Example
(* Example for MC_PhasingAbsolute *)
MasterRef.AxisNo := 0;
SlaveRef.AxisNo := 3;

inst_PhasingAbsolute ( Execute := bExecute,  
                        Master := MasterRef,
                        Slave := SlaveRef,
                        PhaseShift := lrPhaseShift, 
                        Velocity:= lrVeloShift,
                        Acceleration:= lrAccShift, 
                        Deceleration:=lrDecShift, 
                        Jerk:=lrJerkShift );

bDone := inst_PhasingAbsolute.Done; 
bBusy := inst_PhasingAbsolute.Busy; 
bActive := inst_PhasingAbsolute.Active; 
bCommAborted := inst_PhasingAbsolute.CommandAborted; 
bError := inst_PhasingAbsolute.Error; 
ErrorId := inst_PhasingAbsolute.ErrorID;
lrAbsPhaseShift := inst_PhasingAbsolute.AbsolutePhaseShift;
