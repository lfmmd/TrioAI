MC_GearInPos

Commands a gear ratio between the position of the slave and master axes from the synchronisation point onwards.

Description

If the gear ratio value (RatioNumerator/RatioDenominator) is positive, the slave axis and master axis will move in the same direction.

If the gear ratio value (RatioNumerator/RatioDenominator) is negative, the slave axis and the master axis will move in the opposite direction.

The MC Function Module calculates the velocity profile for linear acceleration and deceleration with the following three velocities using the ‘Acceleration’ (Acceleration Rate) and ‘Deceleration’ (Deceleration Rate).

• The velocity of the Slave (Slave Axis) when an instruction is executed is the initial velocity.

• The velocity of the Master (Master Axis) when an instruction is executed multiplied by the gear ratio is the final velocity.

• The velocity (Target Velocity) is the target velocity.

If the travel distance during the catching phase is too short, the target velocity will not be reached.

For the slave axis to catch up with the master axis for the ‘MasterSyncPosition’ and ‘SlaveSyncPosition’, the following condition must be met for the ‘Velocity’ input parameter of MC_GearInPos:

\[ Velocity > \frac{ RatioNumerator \times \text{Master axis velocity when MC_GearInPos is executed} }{ RatioDenominator } \]

If the slave axis is not able to catch up with the master axis for the ‘MasterSyncPosition’ and ‘SlaveSyncPosition’, an error will occur. If the axis is of modulo type, make sure that the synchronous operation starts within one cycle rotation.

If axes are linear axes, the input parameter ‘SyncMode’ will be ignored, because the catching phase has the same behaviour with different modes. If axes are modulo axes, the behaviours may be different with different ‘SyncMode’ inputs.

If axes are module axes, and the slave running in the same direction of the master axis, the slave axis will catch up with the master at the Synchronisation position in the same motion direction. The slave will get Synchronisation at the first ‘SlaveSyncPosition’ on the same direction of the master.

If axes are module axes, and the slave running in the inverse direction of the master axis, the slave axis will catch up with the master at the Synchronisation position in the inverse motion direction. The slave will get Synchronisation at the first ‘SlaveSyncPosition’ on the same direction of the master.

Make sure the input and output parameters are of data type indicated in the table below.

Any previous motion is continued until master crosses ‘MasterSyncPosition’ – ‘MasterStartDistance’ in the correct direction (according to the sign of ‘MasterStartDistance’). At that point in time the output ‘StartSync’ is set. When a ‘Stop’ command is executed on the ‘Slave’ axis before the synchronisation has happened, it inhibits the synchronisation and the function block issues ‘CommandAborted’.

The difference between the ‘SyncMode’ ‘mcCatchUp’ and ‘mcSlowDown’ is in the energy needed to synchronise ‘mcSlowDown’ costs the lowest energy vs ‘mcCatchUp’.

The ‘SyncMode’, ‘mcShortest’ will make the slave to get synchronisation at the first ‘SlaveSyncPosition’ on the same direction of the master for modulo axes.

If slave is a linear axis, ‘SyncMode’ has no effect on the catching phase.

The master axis should be in ‘ConstantVelocity’ motion state, otherwise an error will occur. And make sure the slave axis can catch up ‘MasterSyncPositon’ at its position ‘SlaveSyncPosition’.

Examples

Example 1

The master axis and slave axis are modulo axes in the range of [0,360]. The initial position of master axis is 0 degree and the initial position of the slave axis is 45 degree. The FB commands the slave axis to accelerate up to the gear ratio 1:1 during the master axis start distance in 90 degree and be synchronized with the master axis at the master and slave sync position.

As shown in the above FBD program, which simplify some operations to explain MC_GearInPos, it needs to add and optimise some effective operations to apply in the real production.

For example, if we want to produce the slab which has the special length 500mm,and the certain saw time is 2s, then we can set the relevant parameters, such as convey_velocity = 100mm/s, saw_syncposition = 100mm, saw_cutendposition = 200mm, convey_syncposition = 300mm and so on.

Then the program executes the operation sequentially:

  1. Enable the power
  2. Operate MC_MoveVelocity to move the conveyor axis in the special velocity
  3. Execute homing operation to start production
