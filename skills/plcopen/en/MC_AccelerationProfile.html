MC_AccelerationProfile

Commands a time-acceleration locked motion profile.

Description

This Function Block commands a time-acceleration locked motion profile. After finalising the acceleration profile, the acceleration goes to 0 (and typically the final velocity is maintained). It stays in the state ‘ContinuousMotion’.

This functionality does not mean it runs one profile repeatedly. It can switch between different profiles.

MC_AccelerationProfile can only be executed in any of these states: ‘SynchronizedMotion’, ‘DiscreteMotion’, ‘ContinuousMotion’, or ‘Standstill’.

On successful trigger at the rising edge of ‘Execute’, i.e. it has no errors, the state changes to ‘ContinuousMotion’.

All input parameters are valid until the next rising edge after it is finished or aborted.

When ‘BufferMode’ is set as ‘mcAborting’, ‘Active’ will change to TRUE at the same time as ‘Busy’ changes to TRUE.

When ‘BufferMode’ = ‘mcBuffered’, ‘Busy’ will change to TRUE, ’Active’ stays FALSE until the current FB is running. ’Active’ and ‘Busy’ will change to FALSE when one of ‘Done’, ‘CommandAborted’, or ‘Error’ changes to TRUE.

Make sure the input and output parameters are of data type indicated in the table below.

MC_TA_REF is defined as below:
The content of Time/ Acceleration pair may be expressed in DeltaTime/ Acceleration, where Delta could be the difference in time between two successive points. ‘Acceleration’ can be a signed value.

STRUCT
TableStart : UDINT;
NumberOfPairs : UDINT;
END_STRUCT

As an alternative to this FB, CAM FB coupled to a virtual master can be used.
For smooth motion, acceleration profile setting must be chosen with care as per the intended application.

Example
