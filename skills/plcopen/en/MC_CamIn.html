MC_CamIn

Engages the CAM.

Description

There are two common options for aborting camming, MC_CamOut and MC_Halt to stop camming after MC_CamIn has been called, although other single axis FBs such as MC_MoveAbsolute and MC_MoveRelative can be called. The master/slave relationship between the two axes is ended when MC_CamOut or MC_Halt is called.

MC_CamOut will continue motion at the instantaneous final actual velocity of the slave axis when it is called, and axis motion will continue at that final actual velocity.

MC_Halt will decelerate axis motion to speed 0 and stop motion.

‘MasterOffset’ input parameter can be used to set offset of the master shaft to cam. It is a signed value.

'SlaveOffset' input parameter is used to set offset of the slave table. It is a signed value.

'MasterScaling' input parameter is used to set the factor for the master profile. From the slave point of view, the master overall profile is multiplied by this factor. It must be a positive value. ‘Error’ is SET if the value is negative or zero.

The following picture shows the effect of different ‘MasterScaling’ values on slave axis. This example is based on the following assumed conditions:

  1. The black bold profile line is the slave axis.

  2. The master axis is moving in constant speed.

SlaveScaling' input parameter can be used to set the factor of the slave profile. The overall slave profile is multiplied by this factor. It must be a positive value. ‘Error’ is SET if the value is negative or zero.

The following picture shows the effect of different ‘SlaveScaling’ values on slave axis. This example is based on the following assumed conditions:

  1. The black bold profile line is the slave axis.

  2. The master axis is moving in constant speed.

  3. It is based on the above conditions. Only ‘SlaveScaling’ value is changed.

Specify the starting point of the cam table (0 phase) as an absolute position of the master axis. The unit is command unit.

'MasterStartDistance' is used to specify the position of the master axis when the following axis starts the cam motion. When absolute positioning is specified with ‘StartMode’, specify the absolute position of the master axis.

‘MasterSyncPostion’ specifies the absolute position of the master in the CAM profile where the slave is in-sync with the master. It can be a signed value. If the value is out of range, ‘Error’ will be SET. Default value is zero.

‘StartMode’ input parameter can be set to either ‘mcAbsolute‘ or ‘mcRelative’.

Make sure the input and output parameters are of data type indicated in the table below.
‘Acceleration’, ‘Deceleration’, ‘Jerk’ are not available as inputs to MC_CamIn. These values can be set and read using MC_WriteParameter and MC_ReadParameter.

The Master is not required to be stationary.
If the actual master and slave positions do not correspond to the offset values when MC_CamIn is executed, either an error occurs, or the system deals with the difference automatically.
The Cam is placed either Absolute or Relative to the current master and slave positions. Absolute: the profile between master and slave is seen as an absolute relationship. Relative: the relationship between master and slave is in a relative mode.

MC_CamIn is not merged with the MC_CamTableSelect because this separation enables changes on the fly.
A mechanical analogy to a slave offset is a cam welded with additional constant layer thickness. Because of this, the slave positions have a constant offset and the offset could be interpreted as axis offset of the master shaft if linear guided slave tappets are assumed.

Code:

// Cancel all previous moves in MTYPE

Inst_TC_CANCEL(TRUE,0,0);

Inst_TC_CANCEL1(TRUE,1,0);



// Cam axis configuration

TCW_UNITS(0,1);

TCW_SPEED(0,10);

TCW_ACCEL(0,10);

TCW_DECEL(0,10);

Inst_TC_DEFPOS0(TRUE,0,0);

TCW_SERVO(0,TRUE);



// Master axis configuration

TCW_UNITS(1,1);

TCW_SPEED(1,1);

TCW_ACCEL(1,10);

TCW_DECEL(1,10);

Inst_TC_DEFPOS1(TRUE,1,0);

TCW_SERVO(1,TRUE);



TCW_WDOG(1);



// Set Cam Table

CamTable.TableStart:=200;

CamTable.TableFinish:=211;

Inst_MC_CamTableSelect(TRUE, CamTable, FALSE, FALSE, FALSE);

CamTableId:=Inst_MC_CamTableSelect.CamTableID;



// MC_CamIn configuration

master.AxisNo:=1;

Slave.AxisNo:=0;

StartMode:=mcAbsolute;

MasterValueSource:=mcCommandedValue;

Inst_MC_CamIn(TRUE, Master, Slave, 0, 0, 1, 1, 0, 0, StartMode, MasterValueSource, CamTableID);



// MC_CamOut setup

bCamOutExecute:=FALSE;

Inst_MC_CamOut(bCamOutExecute, Slave);



// Run master axis

Inst_TC_FORWARD(TRUE,1);
