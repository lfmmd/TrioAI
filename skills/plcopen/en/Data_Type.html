Data Types

Trio PLCopen motion control can support all basic data types defined in Data Types of the “Standard IEC 61131-3 Language and Functions Help”. It also defined some complex data types, such as structures and enumeration.

All enumeration definitions for Trio PLCopen motion control function blocks are shown in following table.
EnumerationDescription
MC_BUFFER_MODEIt supports the following modes:
mcAborting: Default mode without buffering. The next FB aborts an ongoing motion and the command affects the axis immediately. The buffer is cleared.
mcBuffered:The next FB effects the axis as soon as the previous movement is ‘Done’. There is no blending.
mcBlendingLow: The next FB controls the axis after the previous FB has finished (equivalent to ‘Buffered’), but the axis will not stop between the movements. The velocity is blended with the lowest velocity of both commands (1 and 2) at the first end-position (1).
mcBlendingPrevious: The velocity is blended with the velocity of the first FB.
mcBlendingNext: The velocity is blended with velocity of the second FB.
mcBlendingHigh: The velocity is blended with highest velocity of both FBs.
MC_AXIS_DIRECTIONAxis moton direction selection, it supports the following modes:
mcBoth : Both directions
mcPositive: Positive direction
mcNegative: Negative direction
MC_CAMSWITCH_MODEDefines the type of cam digital switch work modes:
mcPosition: Position based
mcTime: Time based
MC_COMBINE_MODEDefines the type of combination applied to AxisOut :
mcAddAxes : Addition of the 2 input axes positions
mcSubAxes : Substractionof the 2 input axes positions
MC_DIRECTIONIt supports the following enumerated value:
mcPositiveDirection:Specifies the Positive direction of motion.
mcShortestWay: Specifies the Shortest way of motion.
mcNegativeDirection: Specifies the Negative direction of motion.
mcCurrentDirection: Specifies the Current direction of motion.
MC_HOME_MODESpecify the home method setting.
mcForwardZ:**Homing using encoder reference pulse “Zero Mark” in positive direction
**mcReverseZ: Homing using encoder reference pulse “Zero Mark” in negative direction
mcForwardS:Homing against an absolute switch in positive direction
mcReverseS: Homing against an absolute switch in negative direction
mcForwardZS: Homing against an absolute switch plus encoder reference pulse “Zero Mark” in positive direction
mcReverseZS: Homing against an absolute switch plus encoder reference pulse “Zero Mark” in negative direction
MC_SOURCEDefines the source for axis values (e.g. positions):
mcCommandedValue: Synchronization on commanded value.
mcSetValue: Synchronization on set value.
mcActualValue: Synchronization on actual value.
MC_START_MODESelect the travel method.
mcAbsolute: Absolute positioning
mcRelative: Relative positioning
MC_SYNC_MODEDefine the way to synchronize:
mcShortest: The shortest synchronization position for the slave axis
mcCatchUp: To catch up the master with input ‘Velocity’
mcSlowDown: To catch up the master with energy saving.
MC_TRIGGER_EDGETrigger edge selection for position regist:
mcRisingEdge: Trigged at the rising edge of the source input signal.
mcFallingEdge: Trigged at the falling edge of the source input signal.
MC_TRIGGER_MODEIt supports the following enumerated value:
mcHardware: same as Mode:21 in Trio Basic command REGIST()
mcTime: same as Mode:20 in Trio Basic command REGIST()

All structures definitions for Trio’ PLCopen motion control function blocks are shown in following table.
StructureDescription
AXIS_REFSTRUCT
AxisNo : UINT; (* Start from 0 *)
END_STRUCT
MC_CAM_REFSTRUCT
TableStart : UDINT;
TableFinish : UDINT;
END_STRUCT
MC_CAMSWITCH_REFSTRUCT
TrackNumber : ARRAY[0..15] of UINT; (* Output number is in the range of Outputs of MC_OUTPUT_REF*)
OutputNumber : ARRAY[0..15] of UINT; (* Output number is in the range of Outputs of MC_OUTPUT_REF*)
FirstOnPosition : ARRAY[0..15] of LREAL;
LastOnPosition : ARRAY[0..15] of LREAL;
AxisDirection : ARRAY[0..15] of MC_AXIS_DIRECTION;(* mcBoth : Both directions, mcPositive: Positive direction, mcNegative: Negative direction *)
CamSwitchMode : ARRAY[0..15] of MC_CAMSWITCH_MODE;(* Default mcPosition: Position based, mcTime: Time based *)
Duration : ARRAY[0..15] of TIME;
END_STRUCT
MC_HOME_REFSTRUCT
HomeMode: MC_HOME_MODE; (* Homing mode selection *)
HomeSpeed: LREAL; (*Value of the maximum velocity (not necessarily reached) to find homing signal (the absolute switch). *)
CreepSpeed: LREAL; (*Value of the maximum velocity (not necessarily reached) to find homing position (the absolute switch position/ encoder reference pulse “Zero Mark”) after finding homing signal (the absolute switch). *)
Acceleration: LREAL; (*Value of the maximum acceleration (not necessarily reached)*)
Deceleration: LREAL; (*Value of the maximum deceleration (not necessarily reached) *)
Jerk: LREAL; (*Value of the maximum jerk (not necessarily reached)*)
HomeSwitch: UINT; (* The input number which is used as home switch *)
END_STRUCT
MC_INPUT_REFSTRUCT
FirstInput: UINT;(* The start input number *)
LastInput : UINT;(* The end input number *)
END_STRUCT
MC_OUTPUT_REFSTRUCT
FirstOutput: UINT; (* The start output number *)
LastOutput : UINT;(* The end output number *)
END_STRUCT
MC_TP_REFSTRUCT
TableStart : UDINT;(* The start index value of table *)
NumberOfPairs : UDINT;
END_STRUCT
MC_TV_REFSTRUCT
TableStart : UDINT;(* The start index value of table *)
NumberOfPairs : UINT;
END_STRUCT
MC_TA_REFSTRUCT
TableStart : UDINT;(* The start index value of table *)
NumberOfPairs : UDINT;
END_STRUCT
MC_TRACK_REFSTRUCT
OnCompensation : ARRAY[0..15] of TIME; (*Compensation time with which the switching on is delayedin time per track. *)
OffCompensation: ARRAY[0..15] of TIME; (*Time compensation the switching off is delayed per track. *)
Hysteresis : ARRAY[0..15] of LREAL; (*Distance from the switching point (in positive and negativedirection) in which the switch is not executed until the axishas left this area, in order to avoid multiple switching aroundthe switching point.*)
END_STRUCT
MC_TRIGGER_REFSTRUCT
Mode : MC_TRIGGER_MODE; (* Trigger mode selection *)
Channel: UINT; (* Trigger signal channel selection *)
Source: UINT; (* Trigger signal source selection *)
Edge: MC_TRIGGER_EDGE;; (* Trigger signal edge selection *)
END_STRUCT
