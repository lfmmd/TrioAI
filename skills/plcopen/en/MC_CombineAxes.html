MC_CombineAxes

Combines the motion of 2 axes into a third axis with selectable combination method.

Description

MC_CombineAxes combines the motion of 2 axes into a third axis with selectable combination method. In essence, it is a calculation of a new position setpoint based on the 2 position setpoints of the input axes. This FB is reflected in the state diagram like a synchronized motion type.

MC_CombineAxes supports two combine modes: mcAddAxes and mcSubAxes.

  • mcAddAxes: Slave position = Master1 position (relative position) + Master2 position (relative position).
  • mcSubAxes: Slave position = Master1 position (relative position) - Master2 position (relative position).

MC_CombineAxes is not affected by errors in ‘Master_1’ or ‘_Master_2’. After the error is cleared and the Masters are back in motion, the _Slave will resume the combined positioning operation. Masters are not affected by errors in Slave.

As application example one can work with a separate profile synchronized to an object on a moving belt, or a rotating knife with flexible covered distance to be cut.

Make sure the input and output parameters are of data type indicated in the table below.

To stop the motion, MC_CombineAxes must be interrupted by another FB issuing a new command.

Inputs/Outputs

Examples

Example 1

Axis 1 is the master axis of MC_CamIn. Axis 0 is the slave. The master axis will move along the Cam table defined with sinusoid velocity profile. Axis 1 will move with MC_GearIn as axis 0 when it is synchronized with its master axis. When axis 1 is synchronized with axis 0, axis 3 will start with MC_MoveAbsolute. Axis 0 and axis 1 are combined with mcAddAxes mode to axis 3. The following picture shows the velocity profile for these axes.

(\* Defines for MC_CombineAxes example \*)

\#define STATE_READY     0

\#define STATE_CAMIN     1

\#define STATE_GEARIN    2

\#define STATE_COMBINE   3

\#define STATE_ERROR     8


(\* Defines for Variables used in MC_CombineAxes \*)

 Inst_MC_CamIn : MC_CamIn ;

  (\*\$desc=Axis 2 reference\*)

  (\*\$desc=Cam table ID\*)

  (\*\$embed=(VERS=1    MODE=0    POINT=7)\*)

 Axis1Ref : lib:AXIS_REF ;

  (\*\$desc=A slave axis which will move in Gear motion\*)

 bStartCombine : BOOL ;

  (\*\$desc=A slave axis which will move in combine motion\*)

  (\*\$profile=IO\*)

  (\*\$embed=(VERS=1    MODE=0    POINT=7)\*)

 Inst_MC_CombineAxes : MC_CombineAxes ;

 Axis3Ref : lib:AXIS_REF ;

 bStop : BOOL := FALSE ;

  (\*\$profile=IO\*)

  (\*\$embed=(VERS=1    MODE=0    POINT=1)\*)

 Inst_MC_Stop1 : MC_Stop ;

 Inst_MC_Stop2 : MC_Stop ;

 Inst_MC_Stop3 : MC_Stop ;

 bStart : BOOL := FALSE ;

  (\*\$desc=To start comine motion example\*)

  (\*\$profile=IO\*)

  (\*\$embed=(VERS=1    MODE=0    POINT=0)\*)

 SrcValue : MC_SOURCE := mcSetValue ;

 CombMode : MC_COMBINE_MODE ;

 Inst_MC_MoveAbsolute : MC_MoveAbsolute ;

 mDirection : MC_DIRECTION ;

 BufferMode : MC_BUFFER_MODE ;

END_VAR

(\* Variables initialization \*)

CamId := UINT\#25616; (\* This Cam table should be created before using \*)

Axis0Ref.AxisNo := 0; (\* The slave axis for Cam motion \*)

Axis2Ref.AxisNo := 2; (\* The master axis for Cam motion \*)

Axis3Ref.AxisNo := 3; (\* The slave axis for combine motion \*)

SrcValue := mcSetValue;

StartMode := mcAbsolute;

CombMode := mcAddAxes;

mDirection := mcPositiveDirection;

BufferMode := mcAborting;





(\* cyclic program \*)

        Inst_MC_GearIn( Execute := FALSE(\*BOOL\*),

                        Master := Axis2Ref(\*AXIS_REF\*), 

                        Slave := Axis0Ref (\*AXIS_REF\*),         

                        MasterOffset := LREAL\#0.0 (\*LREAL\*), 

                        SlaveOffset := LREAL\#0.0 (\*LREAL\*),
