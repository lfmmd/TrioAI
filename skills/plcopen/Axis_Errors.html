Axis Errors

Description:

Axis errors generally need to call function block MC_Reset to clear the errors. These errors will mostly cause axis to stop if axis is in motion.

Axis errors have 16-bit value (a WORD datatype) with each bit represents a different error state. Bit 15 is always set if an axis error occurs, therefore the ErrorID of Axis Errors is greater than 0x8000. See table below for summary of Axis Errors:

Details:

Bit 0

Speed limit active

Description: 

The actual speed is over speed limitation.

Reaction:

No action

Solution:

It may be dangerous, please check the speed setting and make sure them be available.

 

Bit 1

Communications error to remote drive

Description: 

The communication between controller and drives gets problem.

Reaction:

Disable axis

Solution:

Solving the communication error.

 

Bit 2

Remote drive error

Description: 

The controller finds remote drive getting error.

Reaction:

No action

Solution:

To solve the remote drive error.

 

Bit 3

In forward hardware limit

Description: 

The axis’ forward hardware limit was triggered.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it.

 

Bit 4

In reverse hardware limit

Description: 

The axis’ reverse hardware limit was triggered.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it.

 

Bit 5

Following error exceeds limit

Description: 

Following error of the axis exceeds limit because some problems.

Reaction:

The axis will be disabled.

Solution:

To check the reason and solve it. Make sure it is safe before next running.

 

Bit 6

FS_LIMIT active

Description: 

The axis position exceeds FS_LIMIT.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it. Make sure it is safe before next running.

 

Bit 7

RS_LIMIT active

Description: 

The axis position exceeds RS_LIMIT.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it. Make sure it is safe before next running.

 

Bit 8

Pulse output axis over speed

Description: 

The demand speed for the axis exceeds the capability of the step pulse hardware to produce pulses.  Typical maximum pulse rates are 2 MHz and 8 MHz depending on product.

Reaction:

The axis may work unexpected.

Solution:

Make sure the demand speed for the pulse output being in the capability of the pulse producer.

 

Bit 9

VOLUME_LIMIT active

Description: 

The axis has 3D volumetric limits set up using the VOLUME_LIMIT command and the robot end-effector position is outside the set limits.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it. Make sure it is safe before next running.

 

Bit 10

AXIS_FS_LIMIT active

Description: 

In a robotic FRAME , AXIS_DPOS is greater than AXIS_FS_LIMIT .  This usually refers to joint angle limits.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it. Make sure it is safe before next running.

 

Bit 11

AXIS_RS_LIMIT active

Description: 

In a robotic FRAME , AXIS_DPOS is less than AXIS_RS_LIMIT.  This usually refers to joint angle limits.

Reaction:

The axis motion is cancled.

Solution:

To check the reason and solve it. Make sure it is safe before next running.

 

Bit 12

Encoder power supply overload

Description: 

The 5 V supply to connected encoders or other hardware powered from the encoder connector has shut down due to overload.   There is one power source shared by all encoder ports. 

Reaction:

The axis will be canceled.

Solution:

To check the reason and solve it. Make sure it being safe before next running.

 

Bit 13

Reserved

 

Bit 14

Reserved

 

Bit 15

Axis error status

Description: 

It represents axis errors have occurred or not. It will be set if any axis errors have occurred, otherwise it will be reset.

Reaction:

The reaction is the same as axis errors have occurred.

Solution:

Solving all axis errors those have occured.
